## 개발 원칙

- **소리담 최종 버전만 이관**: 모든 레거시(v4/v6) 무시, 최종 버전(v7 등)만 가져옴
- **오픽톡닥에서 모두 v1**: 소리담 버전 접미사 제거 (예: `evaluations_v7` → `evaluations`)
- **인증/결제는 새로 구축**: 소리담 코드 복사 X → 업계 표준 방식
- **DB는 새로 설계**: 소리담 스키마를 참고하되 정리해서 새로 만듦
- **백엔드는 Server Actions** (T-8): 데이터 조회=Server Component, 데이터 변경=Server Action, 웹훅만 API Route
- **Edge Functions 사용 안 함**: AI API 호출 등 장시간 처리가 필요한 경우에도 Server Actions로 구현. 실제 구현 시 모듈별 판단
- **UI는 오픽톡닥 브랜드**: Tailwind 4.x + 테라코타 웜톤 디자인 시스템
- **모듈별 풀스택 이관**: DB → RLS → Server Actions → TypeScript 타입 → UI → 테스트

### Step 1: 시험후기 ← 다음 작업

> 시험 후기 제출 (3단계 위저드) + 출제 빈도 분석 + 공개 후기 열람 + 콤보 데이터 생성.
> 📖 상세 설계: `설계/시험후기.md`
> 📖 소리담 원본: `참조/소리담_분석결과.md` §6

#### 공개 정책 (M-4 확정)

**완료된 후기(status='complete')는 전체 공개.** 기출 문제 14개 + 설문 응답 9개 + 자유 후기 공개.

#### 위저드 3단계

| Step | 내용 |
|------|------|
| **Step 1** | 시험 기본 정보 (시험일, 등급) + 설문 7개 (시험 배경 4 + 체감 후기 3) |
| **Step 2** | 기출 문제 14개 입력 (2~15번) |
| **Step 3** | 자유 후기 (한줄 후기 필수 + 팁 선택) → 완료 → 콤보 생성 → 크레딧 보상 |

#### DB 테이블 (3개)

| 테이블 | 용도 | 소리담 대비 변경 |
|--------|------|------------------|
| `submissions` | 후기 마스터 + 설문 9개 컬럼 (exam_purpose, study_methods 등) | 서베이 8필드 삭제 → 통계용 설문 7개 신규, 포인트→크레딧 보상 |
| `submission_questions` | 14개 질문 기록 (question_number, combo_type, master_question_id) | combo_type 영어 통일 |
| `submission_combos` | 콤보 데이터 (custom_mode_questions 대체) | 2테이블 → 1테이블로 통합 (M-1) |

#### Server Actions

| Action | 용도 |
|--------|------|
| `createDraft` | Step 1 draft 생성 (시험 기본 정보 + 설문 7개) |
| `saveQuestions` | Step 2 14개 질문 저장 |
| `complete` | Step 3 완료 + 콤보 생성 + 크레딧 보상 |
| `listMy` / `getDetail` | 내 후기 조회 |
| `delete` / `updateGrade` | 후기 삭제, 등급 수정 |
| `getFrequency` | 빈도 분석 (미인증: 어드밴스만, 인증: 전체) |
| `getTopicQuestions` | 주제별 출제 문제 |
| `getPublicReviews` | 공개 후기 목록 (페이지네이션, 필터) |
| `getStats` | 전체 통계 (제출 수, 유니크 질문 수, 설문 집계) |

- 쉐도잉 4모드: Listen/Training(클라이언트) + Assessment/Real Test(API)

> 교정 스크립트는 `scripts` 테이블 공유 (`source='correct'`)

#### Server Actions

| Action | 용도 |
|--------|------|
| `startSession` | 세션 시작 + 약점 분석 |

| `generateCorrectedScript` | AI 교정 (scripts에 UPSERT) |
| `evaluateAttempt` | Level 1~4 훈련 평가 |
| `nextQuestion` | 다음 유형 활성화 |
| `getReviewSchedule` / `getSkillHistory` | 복습/성장 조회 |
