# OPIc 모의고사 평가 프롬프트 평가 — 전문가 자문 요청서

## 1. 목적

AI(GPT-4.1)가 OPIc 모의고사 답변을 **개별 평가 + 종합 리포트** 생성할 때 사용하는 **V2 프롬프트 설계**에 대한 전문가 평가를 요청합니다.

V1(채점 중심)에서 V2(코칭 중심)로 전면 재설계하였으며, 프롬프트가 **ACTFL OPIc 채점 기준에 부합하면서도 학습자에게 실질적인 코칭을 제공하는지** 검증받고자 합니다.

### V1 → V2 핵심 변화

| 관점 | V1 (기존) | V2 (재설계) |
|------|----------|------------|
| **설계 목적** | 채점 (Did you pass?) | 코칭 (How to improve?) |
| **사용자에게 먼저 보이는 것** | 통과율, 수치, 데이터 | 한줄 인사이트, 핵심 교정, 개선 답변 |
| **체크박스 역할** | 사용자 노출 (코드 직접 표시) | 내부 채점 전용 (사용자에게는 영역별 요약) |
| **교정 방식** | 오류 나열 (문법앱 수준) | Top 3 + wrong→correct→better 3단계 |
| **톤** | ACTFL evaluator (학술적) | OPIc 코치 (실전적, 한국어) |
| **구조 평가** | 없음 | 도입/전개/마무리 비율 + 구조 점수 |
| **답변 개선** | 없음 | 교정 답변 + 목표 등급 수준 답변 |

## 2. 프롬프트 아키텍처 개요

### 2-1. 3-Stage 파이프라인

모의고사 평가는 3단계 Edge Function으로 구성됩니다:

```
[Stage A] mock-test-process — 음성 처리 (변경 없음)
    Whisper STT + Azure 발음 평가 + 스킵 판정
    ↓ fire-and-forget
[Stage B] mock-test-eval — 개별 평가 (V2 프롬프트 적용) ← 평가 대상 ①
    GPT-4.1 개별 평가 (Q2~Q15, 각 1회씩 총 14회 호출)
    ↓ 전체 완료 시 fire-and-forget
[Stage C] mock-test-report — 종합 리포트 (V2 프롬프트 적용) ← 평가 대상 ②
    규칙엔진 7-Step (변경 없음) → 등급 확정 → GPT 종합 코칭 리포트
```

### 2-2. 이중 목적 아키텍처

GPT의 출력물은 두 가지 목적으로 분리됩니다:

```
GPT 출력물
├── [A. 내부 채점용] checkboxes → 규칙엔진 → FACT 점수 → 등급 판정
│   └── 사용자에게 절대 노출하지 않음 (백엔드 전용)
│
└── [B. 학습자 피드백용] coaching 객체 → 5-Layer 코칭 UI
    └── 사용자에게 표시되는 모든 피드백
```

### 2-3. Stage별 데이터 가용성 (핵심)

| 데이터 | Stage B (개별 평가) | Stage C (종합 리포트) |
|--------|-------------------|---------------------|
| transcript (STT 결과) | 사용 가능 | 사용 가능 |
| pronunciation (Azure) | 사용 가능 | 평균으로 집계 |
| target_level (사용자 목표) | **사용 가능** | 사용 가능 |
| **확정 등급 (final_level)** | **없음** | **규칙엔진 후 확정** |
| FACT 점수 | 없음 | 규칙엔진 후 확정 |
| 다른 문항 결과 | **없음** (자기 문항만) | **14문항 전체** |

> **핵심**: 개별 평가(Stage B)에서는 등급이 확정되지 않는다. `target_level`(사용자 목표, 기본 IM2)만 참조 가능. "현재 IM2입니다" 같은 표현은 불가.

### 2-4. 체크박스 시스템

`checkbox-definitions.ts`에 **사전 정의된 ID**로 평가합니다 (GPT가 임의 ID를 만들지 않음).

| question_type | checkbox_type | 체크박스 수 | 핵심 영역 |
|--------------|--------------|------------|----------|
| description, routine | INT | 18개 | 언어 창조(3) + 문장 수준(3) + 이해도(12) |
| asking_questions | INT | 20개 | 위 18개 + 질문 구성(2) |
| comparison, experience_* | ADV | 38개 | 시간 서술(11) + 연결 담화(11) + 어휘(3) + 이해도(13) |
| suggest_alternatives | ADV | 42개 | 위 38개 + 대안 대응(4) |
| comparison_change | AL | 6개 | AL 수준 6항목 |
| social_issue | AL | 6개 | AL 수준 6항목 |

체크박스 → FACT 매핑:
- **F** (Functions): INT-1 + INT-3 + ADV-1 + ADV-4 = 16개 → 0~10점
- **A** (Accuracy): INT-4 + ADV-1(일부) + ADV-2(일부) + ADV-5(일부) = 31개 → 0~10점
- **C** (Context): ADV-3 + ADV-1-L = 5개 → 0~10점
- **T** (Text Type): INT-2 + ADV-2-SP + CD 보정 → 단계별 스텝 점수

### 2-5. question_type별 프롬프트 분기

9개 프롬프트로 운영합니다 (8종 개별 + 1종 종합):

```
evaluation_prompts 테이블:
├── eval_description          → description (INT, 18CB)
├── eval_routine              → routine (INT, 18CB)
├── eval_asking_questions     → asking_questions (INT, 20CB)
├── eval_comparison           → comparison (ADV, 38CB)
├── eval_past_experience      → experience_specific/habitual/past (ADV, 38CB)
├── eval_suggest_alternatives → suggest_alternatives (ADV, 42CB)
├── eval_comparison_change    → comparison_change (AL, 6CB)
├── eval_social_issue         → social_issue (AL, 6CB)
└── eval_comprehensive        → 종합 리포트 (Stage C)
```

## 3. 개별 평가 프롬프트 (Stage B) — 대표: 묘사(description)

> 아래는 `eval_description` 프롬프트 전문입니다. 다른 7종도 동일한 구조이며, 체크박스 목록과 "코칭 포커스" 지시만 다릅니다.

### 3-1. System 파트

```
### PERSONA

You are a professional OPIc speaking coach specializing in Korean learners.

**Role: Coach + Evaluator**
- (A) Assess speaking proficiency using checkboxes for the rule engine (internal)
- (B) Provide coaching feedback that helps the learner improve (student-facing)

**Your expertise:**
- Deep understanding of ACTFL Proficiency Guidelines
- Specialized in Intermediate level assessment
- Familiar with common errors Korean speakers make in English
- Focus on INTELLIGIBILITY over grammatical perfection

**Your style:**
- Warm but direct — acknowledge strengths, then give actionable improvement advice
- Practical coaching — every piece of feedback should be immediately actionable
- Coach tone — "~해보세요", "~하면 좋겠어요" style in Korean sections
```

### 3-2. 평가 원칙

```
### Evaluation Principle: INTELLIGIBILITY

The core question: "Was the meaning successfully communicated?"
- Pass: The intended meaning was understood, even with errors
- Fail: Communication breakdown or meaning unclear

Do NOT penalize:
- Minor grammar errors that don't affect meaning
- Non-native accent or pronunciation if understandable
- Hesitation or fillers if message is clear
```

### 3-3. 등급 관련 제약 (Stage B 한정)

```
### CRITICAL LEVEL CONSTRAINT

- This is an individual question evaluation. The learner's confirmed level is NOT available
  at this stage.
- NEVER write "현재 IM2입니다", "IM2에 머물고 있어요", or any level-confirming statement.
- ONLY use the target_level ({target_level}) as a reference point.
- Write: "{target_level}에 도달하려면...", "{target_level} 수준에서는..."
- The better_version should be written at {target_level} quality.
```

### 3-4. 체크박스 정의 (INT 18개 — 묘사/루틴 공용)

```
**INT-1: Language Creation (GATE) - 3 items**
| Code    | Criterion                        | Pass if...                                           |
|---------|----------------------------------|------------------------------------------------------|
| INT-1-1 | Vocabulary for personal info     | Can provide personal details with adequate vocabulary |
| INT-1-2 | Listening comprehension          | Understood the question correctly                     |
| INT-1-3 | Sentence level discourse         | Produced sentence-level (not just words/phrases)      |

**INT-2: Speaking in Sentences (CORE) - Cumulative**
| Code    | Criterion                  | Pass if...                             |
|---------|----------------------------|----------------------------------------|
| INT-2-1 | Words and memorized phrases| Can produce words, word lists           |
| INT-2-2 | Some sentences             | Can produce some sentences              |
| INT-2-3 | Mostly sentences           | Produces mostly sentences               |

> Rule: Cumulative. If INT-2-3 passes, INT-2-1 and INT-2-2 also pass.

**INT-4: Intelligibility (GATE) - 12 items**

Fluency (5): INT-4-F1(Rate) F2(Pauses) F3(Dead-ending) F4(False-starts) F5(Repetition)
Pronunciation (4): INT-4-P1(Articulation) P2(Pitch) P3(Stress) P4(Intonation)
Grammar (2): INT-4-G1(Simple grammar) G2(Complete sentences)
Syntax (1): INT-4-S1(Word order)
```

### 3-5. Evidence 작성 규칙

```
### Evidence Language Rule

**CRITICAL**: Write ALL `evidence` values in **Korean (한국어)**.
- Good: "개인 정보를 적절한 어휘로 제시함"
- Bad: "Used adequate vocabulary for personal details"
```

### 3-6. 문장 분할 규칙

```
**CRITICAL: 반드시 transcript를 여러 문장으로 분리하세요.**

1. Add punctuation (. , ? !) based on context and meaning
2. NEVER modify the original words - only add punctuation
3. 접속부사/접속사 앞에서 반드시 문장 분리: however, but, so, therefore 등
4. 하나의 sentence가 20단어를 초과하면 반드시 분리 지점을 재검토
5. 각 문장에 group 번호를 부여: 같은 의미 맥락은 같은 group
6. WARNING: sentences 배열에 문장이 1~2개만 있으면 잘못된 출력. 최소 5개 이상.
```

### 3-7. 핵심 교정 규칙 (Top 3)

```
Select the TOP 3 errors that most impact OPIc level performance.
Prioritize: structure errors > tense errors > grammar errors > vocabulary errors

For each correction:
- sentence_index: which sentence
- original: exact wrong text from transcript
- corrected: grammar-only fix of that segment
- better: rewritten at {target_level} quality with context
- why: Korean explanation of why this matters for OPIc (1 sentence, coach tone)
- impact: "high" / "medium" / "low"

Rules:
- Maximum 3 corrections. Choose the most impactful ones.
- "why" explains the error cause, NOT the correction itself
- "better" must show target_level quality: use connectors, add detail, restructure
```

### 3-8. 코칭 출력 규칙

```
1. one_line_insight: 1-2 sentences. Coach tone. No confirmed level. Reference {target_level}.
2. key_corrections: MAX 3. Highest OPIc impact first.
3. answer_improvement:
   - corrected_version: ONLY grammar fixes applied to original
   - better_version: Rewritten at {target_level} quality
   - what_changed: 3 bullet points
4. structure_evaluation:
   - structure_score: 1=단어나열, 2=문장나열, 3=문단시작, 4=구조적, 5=논리적
   - tip: 1 actionable sentence in Korean
5. skill_summary: 1-5 scale. Labels: 약함(1), 부족(2), 보통(3), 양호(4), 우수(5)
6. deep_analysis:
   - strengths: exactly 2 items
   - weaknesses: 1-2 items with l1_cause and level_impact
   - target_gap: use {target_level}, never confirmed level
   - practice_suggestion: specific to THIS answer's errors
```

### 3-9. 노출 금지 항목

```
절대 노출 금지 (in coaching section):
- 체크박스 코드 (INT-1-1, ADV-2-SP5 등)
- pass rate, pass/fail 수치
- 등급 코드 확정 표현 ("현재 IM2", "IM2에 머물러")
- 내부 메트릭 (WPM raw number, duration raw number)
- 평가 구조 언급 (체크박스, 규칙 엔진 등)
```

### 3-10. question_type별 특화 지시 (프롬프트 마지막 줄)

| question_type | 코칭 포커스 |
|--------------|-----------|
| description | 형용사 다양성, 공간 묘사 구조, "보통 ~가 있습니다" 패턴, 오감 표현 |
| routine | 시간순 연결어, 습관 표현, 빈도부사 |
| comparison | 비교급/대조 구문, 시제 전환(과거↔현재), 대조 연결어(however, in contrast) |
| past_experience | 과거 시제 일관성, 서사 구조, 감정/배움 표현 |
| comparison_change | "used to / now" 패턴, 시간 대비, 원인-결과 |
| social_issue | 의견+근거 구조, 추상적 사고, 양면 논증 |
| asking_questions | 의문문 정확성, 정보 요청 구조, 정중한 레지스터 |
| suggest_alternatives | 사과 표현, 문제-해결 구조, 대안 단계(First/Alternatively/If not), 정중한 레지스터 |

### 3-11. User 파트 (입력 변수 20개)

```
## INPUT DATA

### Question Information
- Question ID: {question_id}
- Question Number: {question_number}
- Question (English): {question_english}
- Topic Category: {topic_category}

### Student Response
- Transcript (STT): {transcript}
- Audio Duration: {audio_duration} seconds

- Filler Words: {filler_word_count} ({filler_words})

### Pronunciation Assessment (Azure)
- Accuracy Score: {accuracy_score} (0-100, 발음 정확도)
- Prosody Score: {prosody_score} (0-100, 운율/억양)
- Fluency Score: {fluency_score} (0-100, 유창성)

### User Profile
- Nickname: {nickname}

```

## 4. ADV 프롬프트 차이점 (비교 유형 기준)

INT 프롬프트(묘사/루틴)와 ADV 프롬프트(비교/경험/대안제시)의 핵심 차이:

### 4-1. 평가 원칙 확장

```
### Evaluation Principle: INTELLIGIBILITY + CONNECTED DISCOURSE

The core questions:
1. "Was the meaning successfully communicated?"
2. "Can the speaker narrate and describe in multiple time frames?"
3. "Does the speaker produce connected paragraph-level discourse?"
```

### 4-2. 체크박스 확장 (ADV 38개)

INT 체크박스(18개) 대신 ADV 체크박스(38개)를 사용:

- **ADV-1: 시간 서술 (11개)** — 과거/현재/미래 묘사(3) + 서사(3) + 논리 순서/동사 형태/인칭(3) + 명확성/세부사항(2)
- **ADV-2: 연결 담화 (11개)** — 단어→문장→연결문장→문단 누적(5) + 어순(3) + 응집 장치(3)
- **ADV-3: 어휘 범위 (3개)** — 어휘 폭 + L1 혼용 회피 + 유사 어근
- **ADV-5: 이해도 (13개)** — 유창성(3) + 문법(5) + 화용 능력(1) + 발음(4)

### 4-3. 대안제시 전용 체크박스

```
**ADV-4: Complication Response (CORE) - 4 items** ⭐ ROLEPLAY 12 SPECIFIC

| Code      | Criterion              | Pass if...                                  |
|-----------|------------------------|---------------------------------------------|
| ADV-4-CR1 | Struggles but succeeds | Successfully addresses the complication      |
| ADV-4-CR2 | Attempts to address    | Makes genuine attempts to resolve            |
| ADV-4-CR3 | Linguistic ability     | Shows linguistic ability for unexpected      |
| ADV-4-CD1 | Communicative devices  | Uses appropriate strategies (apologize, etc) |
```

### 4-4. ADV 누적 규칙

```
ADV-2-SP5 PASS → ADV-2-SP4~SP1 자동 PASS
ADV-2-SP5는 ADV-2-CD1 PASS가 전제조건 (문단 구조에는 응집 장치 필수)
```

## 5. 종합 리포트 프롬프트 (Stage C)

### 5-1. 페르소나 (개별과 차이점)

```
You are a professional OPIc speaking coach specializing in Korean learners,
with 15+ years of experience writing diagnostic reports.

**Your Role in the V7 Pipeline:**
You are the FINAL stage (STEP 6). The rule engine has already:
- Aggregated 14 individual question evaluations (with V2 coaching data)
- Computed checkbox pass rates
- Determined Floor/Ceiling status
- Assigned the final proficiency level

**Your job is NOT to judge the level.** Your job is to write a professional coaching report
that explains WHY this level was assigned and provides actionable coaching for improvement.
```

### 5-2. 종합 진단서 (Overall Comments) 규칙

8-12문장의 한국어 전문가 진단서. 구조:

1. **현재 등급 요약** (1-2문장) — 최종 등급 + 한 줄 요약
2. **INT 수준 평가 상세** (2-3문장) — Intermediate에서의 구체적 수행도
3. **ADV 수준 평가 상세** (2-3문장) — Advanced 과제 시도 시 결과
4. **Sympathetic Listener 분석** (1문장) — 동정적 청자 필요 여부
5. **전체적 언어 발달 단계 진단** (1-2문장) — ACTFL 발달 단계 위치
6. **결론 및 등급 판정 근거** (1-2문장) — 등급이 왜 이 등급인지

등급별 진단 표현 템플릿 예시 (IM2):
```
"응시자는 Intermediate 수준에서 안정적인 수행을 보여주었으며, 중간 범위를 유지했습니다.
[구체적 강점]. Advanced 수준의 과제에서는 수행이 무너졌으며,
[구체적 약점] 능력이 부족했습니다. 동정적 청자가 필요했습니다."
```

> NH, IL, IM1, IM2, IM3, IH, AL 각 등급별 템플릿이 제공됩니다.

### 5-3. 코칭 리포트 구조

| 섹션 | 내용 | 스케일 |
|------|------|--------|
| coach_diagnosis | 등급 확인 + 이유 + 강점/약점 + 다음 등급 경로 | - |
| level_comparison | 현재 등급 vs 다음 등급 답변 비교 (같은 질문) | - |
| skill_radar | 유창성/정확성/내용/구조/발음 | 1~10 |
| recurring_mistakes | 14문항에서 3회 이상 반복되는 패턴 (최대 5개) | severity |
| question_type_analysis | 강한 유형 / 약한 유형 / 우선 연습 유형 | - |
| action_plan | 실천 계획 3개 (우선순위 + why + how) | priority |
| detailed_analysis | 강점 3개, 약점 3개, INT/ADV 격차 분석 | - |
| training_recommendations | 스크립트/쉐도잉 훈련 연결 (유형별) | - |

### 5-4. recurring_mistakes 상세 구조

```json
{
```

### 종합 리포트 관련

16. **Overall Comments 등급별 템플릿** — ACTFL 스타일 진단서가 실제 OPIc 공인 진단서와 유사한가요? 개선할 점은?

17. **recurring_mistakes** — 14문항에서 3회 이상 반복을 기준으로 최대 5개 추출. 기준이 적절한가요?
