## 1. 권한 체계

### 역할 구분
- `app_metadata.role = 'admin'` — JWT 기반, DB 조회 불필요
- role 부여 후 재로그인 필요 (JWT 갱신)

### 인증 헬퍼 (`lib/auth.ts`)

| 함수 | 용도 | 반환 |
|------|------|------|
| `getAdminUser()` | 레이아웃 서버 검증 | `User \| null` |
| `requireAdmin()` | SA 권한 검증 | `{ supabase, userId, userEmail }` |

### 보안 구조
1. **미들웨어**: `/admin` 인증 필수 (protectedRoutes)
2. **레이아웃**: `getAdminUser()` → 비관리자 redirect
3. **Server Actions**: `requireAdmin()` 이중 검증
4. **RLS**: admin 전체 읽기 정책 (`user_credits`, `orders`, `submissions`, `mock_test_*`)
5. **감사 로그**: 모든 관리자 작업 `admin_audit_log` 기록

| **평가 설정** | 규칙엔진 threshold 13개, GPT 모델 설정 4종, 유형별 토글 8개, 과제충족 체크리스트 10종 |

**SA**: `getMockExamStats()`, `getMockExamSessions()`, `retriggerEvaluation()`, `getAdminSessionDetail()`, `getTaskChecklists()`, `updateTaskChecklist()`
**컴포넌트**: `eval-pipeline-view.tsx`, `eval-settings-tab.tsx`

#### 평가 설정 탭 상세 (4섹션)

**섹션 A — 규칙엔진 Threshold** (`re_*` 13개):
- Floor 4개: `re_floor_nh/il/im1/im2`
- Ceiling 2개: `re_ceiling_broke_down/respond`
- Sympathetic 4개: `re_sympathetic_low/mid/at_times/pron_weight`
- 기타 3개: `re_checkbox_pass_threshold`, `re_al_pass_threshold`, `re_q12_gatekeeper_threshold`
- "기본값 복원" + "저장" 버튼

**섹션 B — GPT 모델 설정** (4×3=12개):
- Judge: 모델/온도/토큰 (기본: gpt-4.1-mini, 0.2, 6000)
- Coach: 모델/온도/토큰 (기본: gpt-4.1, 0.4, 8000)
- Report: 모델/온도/토큰 (기본: gpt-4.1, 0.4, 12000)
- Growth: 모델/온도/토큰 (기본: gpt-4.1-mini, 0.5, 3000)

**섹션 C — 유형별 활성화 토글** (8개):
묘사, 루틴/습관, 질문하기 RP, 비교/변화, 과거 경험, 대안 제시 RP, 사회 비교/변화, 의견 제시/주장

**섹션 D — 과제충족 체크리스트** (`task_fulfillment_checklists` 10행):
- 10개 아코디언 (question_type별)
- 각 항목: 필수/심화 항목, Ideal Flow, Common Mistakes, Core Prescription, Feedback Tone, Start Template
- DB 데이터가 EF에서 하드코딩 대신 사용됨 (폴백: `TYPE_CHECKLISTS`)
