## 출제 플로우 (F-1)

```
1. 기출 풀 조회: submissions WHERE exam_approved='approved' AND 사용자 미사용
2. 랜덤 3개 추출 → 미리보기 카드 표시
3. 사용자가 1개 선택
4. Q1 자기소개(시스템 고정) + Q2~Q15 기출 순서 그대로 출제

기출 3개 미만 시: 사용 이력 리셋 후 재추출
모든 기출 소진 시: 전체 이력 리셋
```

```
┌─ 기출 A ──────────────────┐
│ 일반 1: 🏠 집              │
```

## 훈련 / 실전 2모드 (F-5)

> **핵심**: 소리담 현재 UI = 훈련 모드 (이관). 실전 모드가 신규 개발.

| 항목 | 훈련 모드 (training) — 소리담 이관 | 실전 모드 (test) — **신규** |
|------|---------------------|-----------------|
| 컨셉 | 코치가 옆에서 봐주는 연습 | 실제 OPIc 시뮬레이션 |
| 세션 유지 | **72시간** | **90분** |
| 이어하기 | 가능 (중간부터 계속) | 가능 (90분 내, 네트워크 끊김 등 대비) |
| 문제 간 전환 | 사용자 Next 클릭 (pause 자유) | 사용자 Next 클릭 (실제 OPIc 동일) |
| 개별 평가 확인 | 문제 완료 후 즉시 확인 가능 | 전체 완료 후에만 확인 |
| 타이머 | 경과 시간 표시 (제한 없음) | **40분 카운트다운** |
| 40분 경과 시 | — | 경고 알림 + 남은 문제 계속 진행 허용 |
| 질문 리플레이 | 소리담 동일 (5초 내 1회) | 소리담 동일 (5초 내 1회) |
| 녹음 리플레이 | 가능 (자기 녹음 재생) | **불가** (녹음 끝 → 바로 제출) |
| 이전 문제 | 가능 | **불가** (되돌아가기 없음) |

### 40분 경과 처리 (실전 모드)

```
[40분 경과]
  → 경고 모달: "실제 OPIc 시험이라면 여기서 종료됩니다.
     하지만 훈련 목적이므로 남은 문제를 마저 풀 수 있습니다."
  → [남은 문제 마저 풀기] ← 권장 (15문제 완료 후 평가가 정확함)
  → [여기서 종료하기] ← 현재 진행 문항까지만 평가
```

**남은 문제 마저 풀기를 권장하는 이유**:
- 불완전 문항(예: 12문제)으로 평가하면 규칙엔진 경우의 수가 복잡해짐
- 14문제 기준으로 설계된 체크박스/FACT 점수 계산이 불완전해짐
- 기술적으로 중간 종료 평가가 더 어려움 → 15문제 완료를 유도

### 모드 선택 UX

- 기출 선택 후, 모드 선택 화면 표시
- 훈련 모드: "중간에 개별 평가를 확인하며 학습할 수 있어요. 72시간 내 이어하기 가능"
- 실전 모드: "실제 OPIc 시험(40분)처럼 진행됩니다. 40분 경과 시 알림이 표시됩니다. 네트워크 끊김 시 90분 내 이어하기 가능"
- 실전 모드 선택 시 확인 다이얼로그 필수

### 기술 차이

- **백엔드**: 동일 (같은 평가 파이프라인)
- **프론트엔드**: `mode` 값으로 분기
  - 훈련 (소리담 이관): 개별 평가 즉시 표시, pause 허용, 72h 세션 유지, 이전 문제 가능, 녹음 리플레이 가능
  - 실전 (신규): 개별 평가 숨김, 40분 카운트다운, 이전 문제 불가, 녹음 리플레이 불가, 40분 경고, 90분 세션 유지, 이어하기 가능
- **DB**: `mock_test_sessions.mode` = `'training' | 'test'`

## Q1 자기소개 (F-6)

```
Q1: "Tell me something about yourself." (항상 동일 고정)
Q2~Q15: 기출 세트의 14문제 (평가 대상)
총 15문제, 실제 평가 14문제
```

- Q1은 녹음/저장하지만 **평가 큐에 등록하지 않음**
- 진행률 계산: 14개 기준 (Q2~Q15만)
- 규칙엔진, 체크박스, FACT 점수 모두 Q1 제외
- Q1 질문 텍스트 + 오디오: questions 테이블의 시스템 질문 (`SELF_INTRO` ID) 사용

## 크레딧 정책 (F-5)

- **차감 시점**: 세션 생성 시 1크레딧 차감 (두 모드 동일)
- **차감 순서**: `plan_mock_exam_credits` → `mock_exam_credits` (만료분 우선)
- **환불 조건**: 시스템 오류(EF 장애, 평가 3회 재시도 실패)로만 환불
- **자발적 포기**: 환불 없음
- **만료 시**: 답변한 문제까지만 평가 완료 처리, 크레딧 환불 없음
- **RPC**: `consume_mock_exam_credit` / `refund_mock_exam_credit` (스크립트 패턴 재활용)

## 테이블 설계 (4개 + 설정 1개)

### mock_test_sessions (세션 관리)

```sql
CREATE TABLE mock_test_sessions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id      TEXT UNIQUE NOT NULL,        -- 'mt_xxxxxxxx'
  user_id         UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  submission_id   UUID NOT NULL REFERENCES submissions(id),  -- F-1: 기출 세트 원본
  mode            TEXT NOT NULL CHECK (mode IN ('training','test')),  -- F-5: 2모드
  status          TEXT NOT NULL DEFAULT 'active'
                  CHECK (status IN ('active','completed','expired')),
  question_ids    TEXT[] NOT NULL,              -- Q2~Q15 question_id 배열 (14개)
  current_question SMALLINT DEFAULT 1,
  total_questions  SMALLINT DEFAULT 15,
  holistic_status TEXT DEFAULT 'pending'
                  CHECK (holistic_status IN ('pending','processing','completed','failed')),
  report_retry_count SMALLINT DEFAULT 0,       -- 최대 3회 재시도
  report_error    TEXT,
  started_at      TIMESTAMPTZ DEFAULT NOW(),
  completed_at    TIMESTAMPTZ,
  expires_at      TIMESTAMPTZ NOT NULL          -- training: +72h, test: +90m (40분은 프론트 타이머용)
);
```

**인덱스**: `(user_id, status)`, `expires_at`, `submission_id`

### mock_test_answers (답변 + 평가 큐 통합)

```sql
CREATE TABLE mock_test_answers (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id      TEXT NOT NULL REFERENCES mock_test_sessions(session_id) ON DELETE CASCADE,
  question_number SMALLINT NOT NULL,            -- 1~15
  question_id     TEXT,                          -- Q1은 NULL 가능 (시스템 질문)
  audio_url       TEXT,
  audio_duration  NUMERIC(8,2),
  transcript      TEXT,
  word_count      INTEGER,
  filler_word_count INTEGER DEFAULT 0,
  long_pause_count  INTEGER DEFAULT 0,
  pronunciation_assessment JSONB,               -- Azure Speech SDK 결과
  eval_status     TEXT DEFAULT 'pending'
                  CHECK (eval_status IN ('pending','processing','stt_completed',
                         'evaluating','completed','failed','skipped')),
  eval_retry_count SMALLINT DEFAULT 0,
  eval_error      TEXT,
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(session_id, question_number)
);
```

- 소리담의 `mock_test_queue` 별도 테이블을 `answers.eval_status`로 통합
- **Q1은 eval_status='skipped'으로 즉시 처리** (평가 큐 미등록)

### mock_test_evaluations (개별 문항 평가)

```sql
CREATE TABLE mock_test_evaluations (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id      TEXT NOT NULL,
  user_id         UUID NOT NULL REFERENCES auth.users(id),
  question_number SMALLINT NOT NULL,            -- 2~15 (Q1 제외)

  fail_count      SMALLINT,
  pass_rate       NUMERIC(5,4),
  sentences       JSONB,
  corrections     JSONB,                        -- eval_only에서도 문장별 교정 포함
  deep_analysis   JSONB,                        -- 5섹션: overall/linguistic/communicative/gap/recommendation
  transcript      TEXT,
  wpm             NUMERIC(6,2),

  filler_count    SMALLINT,
  long_pause_count SMALLINT,

  tokens_used     INTEGER,
```

## 백엔드 구조 (F-3: 3개 EF + SA)

T-9 원칙 "CRUD = Server Actions, AI = Edge Functions" + **오디오 Client→Storage 직통** 업로드.

> Vercel Pro Body 제한 4.5MB → WAV 4분(~7.5MB) SA 경유 시 실패. Client에서 Supabase Storage에 직접 업로드.

### Server Actions (`lib/actions/mock-exam.ts`)

| SA | 역할 |
|----|------|
| `getExamPool` | 기출 3개 미리보기 (승인 submissions 조회) |
| `createSession` | 세션 생성 + 크레딧 차감 RPC |
| `submitAnswer` | audioUrl + 메타데이터 저장 + fire-and-forget → EF |
| `skipQuestion` | 스킵 처리 |
| `getSession` | 세션 상세 조회 |
| `getHistory` | 이력 목록 |
| `expireSession` | 만료 처리 |

> 상태 폴링은 SA 아님 — Client에서 Supabase 직접 조회 (F-8)

### Edge Functions (3개 — AI 처리만)

```
supabase/functions/
├── mock-test-process/                  ← EF 1: Stage A (~70초)
│   └── index.ts                        ← Whisper STT + Azure 발음 평가
│
├── mock-test-eval/                     ← EF 2: Stage B (~100초)
│   └── index.ts                        ← GPT-4.1 체크박스 평가
│
├── mock-test-report/                   ← EF 3: Stage C (~45초)
│   └── index.ts                        ← 규칙엔진 실행 + GPT 리포트 생성
│
└── _shared/                            ← 공유 유틸
    ├── rule-engine.ts                  ← V7 규칙엔진 (~1,437줄, 소리담 이관)
    ├── checkbox-definitions.ts         ← INT(18)/ADV(38)/AL(6) 체크박스 정의
    ├── azure-pronunciation.ts          ← Azure Speech SDK 발음 평가
    └── skip-detector.ts               ← 3단계 스킵 판정 (15초/15자/환청)
```

## 평가 플로우 (Fire-and-Forget 체인)

```
[Client] → Supabase Storage `mock-test-recordings` 버킷 직접 업로드 (WAV, 프록시 없음)
         → audioUrl 수신
[Client] → SA submitAnswer (URL + 메타데이터만, 수 KB)
  → answers INSERT (eval_status='pending')
  → Q1이면: eval_status='skipped' (평가 제외, F-6)
  → Q2~Q15: fire-and-forget HTTP → mock-test-process/

[Stage A] mock-test-process/ (~70초)
  → 1차 스킵: audio_duration ≤ 15초 → 전체 생략
  → Whisper STT (word-level timestamps)
  → 2차 스킵: transcript ≤ 15자 또는 환청 → Azure/GPT 생략
  → Azure 발음 평가 (accuracy/prosody/fluency)
  → answers UPDATE (transcript, pronunciation, eval_status='stt_completed')
  → fire-and-forget → mock-test-eval/

[Stage B] mock-test-eval/ (~100초)
  → GPT-4.1 체크박스 평가 (question_type 기반, F-15)
  → evaluations INSERT
  → answers UPDATE (eval_status='completed')

[Stage C] mock-test-report/ (~45초)
  → Q15 eval 완료 시 자동 트리거 (mock-test-eval/ 에서 fire-and-forget)
  → 규칙엔진 7-Step 실행 → 최종 등급 판정
  → FACT 점수 계산
  → GPT 리포트 생성 (종합 피드백, 훈련 권장)
  → mock_test_reports INSERT

[재시도 로직]
  → 실패 시 eval_retry_count++ (최대 3회)
  → backoff: 1초 → 2초 → 4초
  → 최종 실패: eval_status='failed' + eval_error 기록
  → 3회 실패 시 크레딧 환불 트리거
```

### 훈련 모드 특수 플로우

```
[문제 완료] → 답변 제출 → 평가 시작
  → 프론트: "평가 중..." 스피너 + 단계별 진행 표시
  → eval_status 변화를 5초 폴링으로 감지
    processing → "음성 분석 중..."
    stt_completed → "체크박스 평가 중..."
    completed → "평가 완료!" → 개별 평가 결과 즉시 표시
  → 사용자가 평가 확인 후, 준비되면 다음 문제로 이동
```

## 규칙 엔진 등급 판정 (rule-engine.ts)

소리담 V7 로직 그대로 이관 (~1,437줄):

```
Step 1: 체크박스 집계 (INT/ADV/AL 통과율, 누적 로직)
Step 2: Floor 판정 (INT 통과율 → NH/IL/IM1/IM2/IM3)
Step 3: Ceiling 판정 (ADV 통과율 → BROKE_DOWN/RESPOND/SUSTAIN)
Step 4: Sympathetic Listener (발음 + ADV-5 복합 점수)
Step 5: Q12 게이트키퍼 (롤플레이 12번 ADV-4 통과율)
Step 6: AL 판정 (AL 게이트키퍼 + 통과율)
Step 7: 최종 등급 결합
```

### FACT 점수

```
F (Functions): 15개 체크박스 → 0-10점
A (Accuracy): 31개 체크박스 → 0-10점
C (Context):  5개 체크박스 → 0-10점
T (Text Type): 11개 체크박스 → 0-10점
총점 = (F+A+C+T) × 2.5 = 100점 만점
```

### 체크박스 구조 (F-12)

> 네이밍: `question_type` 기반 (questions 테이블의 `question_type_eng` 컬럼 일치)

| question_type (오픽톡닥) | 소리담 answer_type | checkbox_type | 개수 | 적용 문항 |
|---|---|---|---|---|
| `description` (묘사) | description | INT | 18 | 설명형 |
| `routine` (루틴) | routine | INT | 18 | 루틴/습관 |
| `asking_questions` (질문하기) | roleplay_11 | INT | 20 | 11-13번 롤플레이 (+INT-3) |
| `comparison` (비교) | comparison | ADV | 38 | 비교 |
| `experience_*` (경험 3종) | past_experience | ADV | 38 | 과거 경험 (specific/habitual/past 공유) |
| `suggest_alternatives` (대안제시) | roleplay_12 | ADV | 42 | 12번 특수 롤플레이 (+ADV-4) |
| `comparison_change` (비교변화) | advanced_14 | AL | 6 | 14번 어드밴스 |
| `social_issue` (사회적이슈) | advanced_15 | AL | 6 | 15번 어드밴스 |

**question_type → 프롬프트 그룹 매핑** (10종 → 8그룹):
- `experience_specific`, `experience_habitual`, `experience_past` → 모두 `eval_past_experience` 프롬프트 공유
- 나머지 7종은 1:1 매핑

## 세션 정책

### 동시 세션 제한

- 활성 세션 **1개만** 허용
- 기존 활성 세션 있을 때:
  - 훈련 모드: "이어하기" 안내 → 기존 세션 계속 또는 포기
  - 실전 모드: "이어하기" 안내 → 기존 세션 계속 또는 포기 (90분 내)

### 만료 처리

- 훈련 모드: 72시간 후 `status='expired'`
- 실전 모드: **40분 경과 = 경고 모달** (강제 종료 아님)
  - "남은 문제 마저 풀기" 선택 → 계속 진행 (타이머 빨강, 초과 시간 표시)
  - "여기서 종료하기" 선택 → 현재까지 문항으로 평가 + `status='completed'`
  - 이어하기: 90분 내 재접속 시 진행한 데까지 이어서 가능
  - 세션 자체 만료: 90분 후 `status='expired'`
- 만료 시: 답변한 문제까지만 평가 완료 처리
- 죽은 세션 정리: Q1 외 평가 0건인 세션 자동 삭제

### 기출 사용 이력

- `mock_test_sessions.submission_id`로 추적
- 해당 사용자가 이미 사용한 기출은 풀에서 제외
- 모든 기출 소진 시: 사용 이력 리셋, 처음부터 다시

### 소리담 질문 오디오 재생 시스템

**2-Play 시스템** (코드 line 656-803):

```
1. AVA 클릭 → 1차 오디오 재생 시작
2. 1차 재생 완료 → 자동으로 startRecording() + canReplay=true + 5초 카운트다운
3. 5초 내 다시 듣기 클릭 가능:
   → 2차 재생 → 완료 시 다시 자동 startRecording() + canReplay=false
4. 5초 만료 시: 리플레이 없이 녹음만 계속
```

- **Primary**: `new Audio(audioUrl)` + `requestAnimationFrame` 프로그레스
- **Fallback**: `window.speechSynthesis` (Web Speech API, rate: 0.9, lang: 'en-US')
- **핵심**: 오디오 재생 완료 → **자동 녹음 시작** (수동 녹음 시작 버튼 아님)
- 리플레이 1회만. 더블 플레이 방지 `cleanupAudio` 헬퍼.

## UX 개선 (F-17 확정 — 전수 구현)

> 소리담 대비 사용자 경험 개선 13개 항목. 전수 구현 확정 (2026-03-02).

### UX-1. 시험 전 — 심리적 부담 완화

#### 1-1. 기출 카드에 난이도 힌트 (신규)

현재 기출 카드는 주제만 표시. 사용자가 "어떤 기출이 쉬운지" 판단할 정보가 없음.

```
┌─ 기출 A ──────────────────────┐
│ 난이도: ★★☆ (보통)            │ ← question_type 분포 기반 자동 계산
│                               │
│ 일반1: 🏠 집 (묘사+루틴)       │ ← question_type 표시
│ 일반2: 🎵 음악 (묘사+경험)     │
│ 일반3: ♻️ 재활용 (비교+경험)    │
│ 롤플: 🏦 은행 (질문+대안)      │
│ 어드: 📱 기술 (비교변화+이슈)   │
│                               │
│ 묘사 3 | 경험 4 | 비교 2 | ... │ ← question_type 분포 뱃지
└───────────────────────────────┘
```

- 난이도 계산: 어드밴스/롤플레이 비중 + question_type 다양성 기반
- 목적: 초보자가 묘사/루틴 위주 기출을 선택하도록 유도

#### 1-2. 모드 선택 시 확인 다이얼로그 개선 (기존 확정 + 보강)

실전 모드 선택 시 단순 확인이 아닌, **체크리스트 형태**:

```
┌─ 실전 모드로 시작할까요? ────────────────┐
│                                         │
│ ✅ 조용한 환경에서 진행합니다              │
│ ✅ 40분 동안 중단 없이 진행할 수 있습니다  │
│ ✅ 크레딧 1개가 차감됩니다                │
│                                         │
│ [시작하기]              [취소]           │
└─────────────────────────────────────────┘
```

- 3개 체크박스 모두 체크해야 "시작하기" 활성화
- 실전 모드의 진지함을 사전에 인지시킴

### UX-2. 시험 중 — 진행 안내 강화

#### 2-1. 키보드 단축키 (쉐도잉 패턴 재활용)

쉐도잉 모듈이 1~4 키보드 단축키를 지원. 모의고사도 동일 패턴 적용:

| 키 | 동작 | 비고 |
|----|------|------|
| `Space` | 녹음 시작/중지 | 가장 자주 사용 |
| `R` | 질문 리플레이 | 5초 내 1회 |
| `→` / `Enter` | 다음 문제 | 녹음 완료 후 |
| `←` | 이전 문제 | 훈련 모드만 |
| `S` | 건너뛰기 | Skip |
| `Esc` | 세션 나가기 확인 | 오조작 방지 다이얼로그 |

- 하단에 `?` 아이콘으로 단축키 안내 팝오버

#### 2-2. 문제 전환 시 카테고리 안내 (신규)

현재 소리담은 문제 번호만 표시. 사용자가 "지금 일반인지 롤플레이인지" 알기 어려움.

```
┌─ 상단 바 ─────────────────────────────────────────┐
│ Q11/15  롤플레이: 🏦 은행                [32:15]  │
│ ████████████████████░░░░░░                        │
│                                                   │
│ 💡 "상황극입니다. 은행 직원에게 질문하세요"           │ ← 카테고리별 한줄 가이드
└───────────────────────────────────────────────────┘
```

- 일반: "주제에 대해 자유롭게 이야기하세요"
- 롤플레이: "상황극입니다. {역할}에게 {행동}하세요"
- 어드밴스: "심화 질문입니다. 구체적인 의견을 제시하세요"
- 훈련 모드에서만 표시 (실전에서는 숨김 — 실제 OPIc에 없으므로)

#### 2-3. 녹음 품질 실시간 피드백 (신규)

녹음 중 볼륨 바만으로는 부족. **음성 감지 피드백** 추가:

```
녹음 중 상태:
- 소리 없음 3초 이상: "마이크를 확인해주세요" 토스트 (상단 경고)
- 너무 작음 (볼륨 20% 이하): 볼륨 바 빨강 + "좀 더 크게 말해주세요"
- 정상: 볼륨 바 초록
- 4분 임박 (3:30 경과): "녹음 종료 30초 전" 카운트다운
```

- Web Audio API AnalyserNode 활용 (이미 볼륨 바에 사용)
- 임계값 기반 단순 판정, 추가 라이브러리 불필요

#### 2-4. 업로드 상태 인디케이터 (신규)

녹음 완료 → Storage 업로드 → EF 호출까지 네트워크 의존. 사용자에게 상태 표시:

```
[녹음 완료] → [업로드 중...] → [제출 완료 ✓] → [다음 문제]
                ↑ 진행 바 표시     ↑ 0.5초 후 자동 전환
```

- 업로드 실패 시: "업로드 실패. 재시도 중..." + 자동 재시도 (최대 3회)
- 네트워크 끊김 시: 로컬 임시 저장 + "네트워크 복구 후 자동 업로드" 안내

### UX-3. 훈련 모드 전용 — 학습 효과 극대화

#### 3-1. 개별 평가 대기 중 다음 문제 프리뷰 (신규)

현재: 평가 완료까지 대기 → 결과 확인 → 다음 문제
개선: 평가 대기 중 **다음 문제 미리보기** 표시

```
┌─ 평가 중 ─────────────────────────────────────────┐
│ Q3 평가 진행 중... (체크박스 평가 중)                │
│ ████████████░░░░░░░░                              │
│                                                   │
│ ┌─ 다음 문제 미리보기 ──────────────────┐           │
│ │ Q4: 롤플레이 — 🏦 은행                │           │
│ │ "질문하기" 유형                       │           │
│ │                                      │           │
│ │ 💡 팁: 롤플레이에서는 상황에 맞는       │           │
│ │ 자연스러운 질문을 던지세요.             │           │
│ └──────────────────────────────────────┘           │
│                                                   │
│ [평가 건너뛰고 다음으로 →]  [평가 완료 대기]         │
└───────────────────────────────────────────────────┘
```

- "평가 건너뛰고 다음으로" 선택 시: 다음 문제 시작, 평가 결과는 나중에 질문 그리드에서 확인
- 평가 완료 시: 자동으로 결과 패널 표시
- 목적: 대기 시간 (~100초)을 학습 시간으로 전환

#### 3-2. 질문 그리드에서 이전 평가 재확인 (신규)

훈련 모드에서 질문 그리드의 완료된 문제 클릭 시 → 해당 평가 결과 다시 보기:

```
질문 그리드:
[1:건너뜀] [2:✓] [3:✓ ← 클릭] [4:녹음중] [5] [6] ...

→ Q3 평가 결과 팝오버:
  통과율 78% | INT | 발음 82
  [상세 보기] [닫기]
```

- 팝오버로 간결하게 표시 (전체 패널 열지 않음)
- "상세 보기" 클릭 시 전체 평가 패널 표시

### UX-4. 평가 대기 — 이탈 방지

#### 4-1. 예상 소요 시간 표시 (신규)

평가 대기 화면에 남은 예상 시간을 표시:

```
개별 평가: 8/14 완료 (약 10분 남음)
종합 분석: 대기 중 (개별 완료 후 약 2분)
```

- 산출: 남은 문항 × 평균 처리 시간 (Stage A ~70초 + Stage B ~100초)
- 처음 1~2문항 실측 후 동적 조정

#### 4-2. 브라우저 백그라운드 알림 (신규)

평가 대기 중 사용자가 다른 탭으로 이동했을 때:

```
[브라우저 탭 제목]: "🎯 평가 완료! — 오픽톡닥 모의고사"
+ Notification API (권한 허용 시): "모의고사 평가가 완료되었습니다. 결과를 확인하세요."
```

- `document.hidden` 감지 → 평가 완료 시 Notification
- 탭 제목 업데이트 (기본 "평가 진행 중..." → "평가 완료!")

#### 4-3. OPIc 팁 + 약점 예고 (신규)

평가 대기 중 이미 완료된 문항 기반으로 **중간 피드백** 힌트:

```
💡 지금까지의 경향:
```

### UX-6. 네트워크 복원력

#### 6-1. 오프라인 감지 + 로컬 보관 (신규)

```
[네트워크 끊김 감지]
  → 녹음 데이터: IndexedDB에 임시 저장
  → 상단 배너: "오프라인 상태입니다. 녹음은 계속됩니다."
  → 세션 타이머: 계속 진행 (프론트엔드)

[네트워크 복구]
  → IndexedDB에서 미업로드 파일 감지
  → 자동 업로드 재시도 (순차적)
  → "업로드 복구 중... (2/3)"
```
