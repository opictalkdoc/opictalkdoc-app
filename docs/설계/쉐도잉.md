### A-2. Edge Functions (2개)

#### ① shadowing-test-create (세션 생성)

**입력:** `{ package_id, script_id }`
**처리:**
1. 인증 확인 (Bearer 토큰)
2. scripts에서 질문 정보 조회
3. 세션 ID 생성: `st_{timestamp}_{random}`
4. 기존 active 세션 만료 처리
5. 새 세션 INSERT (24시간 유효)

**출력:** `{ success, session_id, session }`

#### ② shadowing-test-evaluate (음성 평가)

**입력:** `{ session_id, package_id, script_id, audio_data (Base64 webm), audio_duration }`
**처리 (9단계):**
1. 권한 확인 + 포인트 결정 (10P)
2. scripts에서 english_text, key_expressions 조회
3. Base64 → Blob 변환
4. **Whisper STT** (model: whisper-1, language: en)
5. 답변 길이 검증 (< 5단어 → 환불 + 거부)
6. `admin_prompts` 테이블에서 평가 프롬프트 로드 + 변수 치환
7. **GPT-4o-mini** 평가 (temperature: 0.3, max_tokens: 1500, json_object)
8. `shadowing_test_evaluations` INSERT
9. 세션 status='completed' 업데이트

**에러 시:** 포인트 자동 환불 (포인트 사용한 경우만)

**GPT 평가 출력:**

### A-3. 4가지 모드 (프론트엔드)

#### Listen Mode (클라이언트 완결)
- 전체 오디오 재생 + 타임스탬프 기반 문장 하이라이트
- 속도 조절 (0.5x ~ 2.0x), 스크립트 표시 전환 (영문/한글/양쪽/없음)
- DB/API 호출 없음

#### Training Mode (클라이언트 완결)
- 3단계 루프: Shadow(듣기) → Record(녹음) → Evaluate(비교)
- 문장별 오디오 구간 재생 + 사용자 녹음 + 파형 비교
- 자동 정지: 원본 길이 + 2초
- DB 호출 없음, localStorage로 진도 추적

#### Assessment Mode (클라이언트 완결 — 소리담에서는 API, 오픽톡닥에서 변경 검토 필요)
- 전체 스크립트 1회 녹음 → 평가
- 녹음팁 랜덤 표시, 3초 카운트다운, 결과 5영역 + 등급

#### Real Test Mode (API 호출)
- 스크립트 숨김, 질문만 표시
- 세션 생성 → 2분 녹음 → Whisper STT → GPT 평가 → DB 저장
- 결과: 5영역 + OPIc 등급 + 스크립트 활용도 + 강점/약점/개선안

## Part B-2: 쉐도잉 훈련 BP 연구 (2026-02-25 정리)

> 소리담 4모드 구조의 학술적 타당성 평가 + 업계 BP 사례 조사 결과

### 학술 근거

#### Kadota(2019) — 쉐도잉 4대 인지 메커니즘

| 메커니즘 | 설명 | 소리담 대응 |
|----------|------|------------|
| **Input Effect** | 청각 입력의 지각·이해 능력 향상 | ✅ Listen Mode (전체 듣기) |
| **Practice Effect** | 반복을 통한 음성 처리 자동화 | ✅ Training Mode (문장별 반복) |
| **Output Effect** | 발화 능력 직접 향상 | ✅ Assessment/Real Test (실제 발화) |
| **Monitoring Effect** | 자기 발화 모니터링 능력 향상 | ⚠️ 부분적 (AI 피드백은 있으나 실시간 모니터링 없음) |

- **결론**: 소리담 4모드는 Kadota의 4대 메커니즘을 **약 80% 수준**으로 커버

#### ELT School 5단계 쉐도잉 모델 (Whitworth 2025)

```
Step 1: Listening (내용 파악 — 1~2회 전체 듣기)
Step 2: Meaning Check (의미 확인 — 모르는 단어/표현 점검)
Step 3: Overlapping (오버래핑 — 텍스트 보면서 원본과 동시 읽기)
Step 4: Prosody Shadowing (쉐도잉 — 텍스트 없이 따라 말하기)
Step 5: Record & Review (녹음 후 원본과 비교 검토)
```

#### 소리담 vs 학술 모델 대응 분석

| 학술 단계 | 소리담 모드 | 커버 여부 | 비고 |
|-----------|------------|----------|------|
| Step 1: Listening | Listen Mode | ✅ 완전 대응 | |
| Step 2: Meaning Check | (없음) | ⚠️ 간접 | 한글 번역은 있으나 명시적 단계 아님 |
| **Step 3: Overlapping** | **(없음)** | **❌ 누락** | **가장 중요한 갭** — 텍스트 보면서 동시 읽기 |
| Step 4: Prosody Shadowing | Training Mode | ⚠️ 부분 | 쉐도잉이라 부르지만 실제로는 문장별 따라 읽기 |
| Step 5: Record & Review | Assessment/Real Test | ✅ 상세 평가 | GPT 5영역 평가로 오히려 학술보다 상세 |

#### 핵심 발견: Overlapping ≠ Shadowing

- **Overlapping**: 텍스트를 **보면서** 원본 음성과 **동시에** 발화 (발음·리듬·억양 습득에 핵심)
- **Shadowing**: 텍스트 **없이** 0.5~1초 뒤에 따라 말하기 (자동화·유창성 향상)
- 학술적으로 Overlapping → Shadowing 순서가 **필수** (갑자기 텍스트 없이 따라 말하면 효과 반감)
- 소리담은 Listen → 바로 Training(문장별 따라 읽기)으로 건너뛰어 **인지 부하 급상승** 문제

### 업계 BP 사례

| 서비스 | 핵심 기능 | 참고할 점 |
|--------|----------|----------|
| **ELSA Speak** | 음소(phoneme) 레벨 발음 분석 + 실시간 피드백 | 세밀한 피드백이 학습 효과에 큰 차이 |

| **Speechling** | 원어민 코치 녹음 피드백 | AI로 대체 가능 (비용 절감) |

### OPIc 맥락에서 쉐도잉의 위치

```
[OPIc 말하기 훈련 3단계]

Step 1: 스크립트 작성 (Script) — 답변 내용 구조화
Step 2: 쉐도잉 훈련 (Shadowing) — 발화 기초 체력 훈련

```

[하루오픽 4단계] ← 최종 확정 (2026-02-26)

텍스트  ✅           토글(전체/첫단어/숨김)  ❌            ❌

- **혼자 말하기** (Step 3) — OPIc 고유 "2분 답변 시간 관리" 훈련
- **Assessment/Real Test 통합** (Step 4) — 질문만 보고 자유 발화 1가지로 통합
- **크레딧**: Step 1~3 무료 (클라이언트 완결), Step 4만 크레딧 소비

## Part C: 하루오픽 구현 설계 (4단계 점진 훈련 모델)

### 쉐도잉의 위치

쉐도잉은 **스크립트 모듈의 하위 기능**이다.
- 네비게이션: 스크립트 탭 → "쉐도잉 훈련" 탭
- 스크립트 생성 → 패키지 생성(WAV+JSON) → **쉐도잉에서 소비**
- AI 튜터 Level 2도 쉐도잉이지만, 튜터링 컨텍스트(교정 답변 기반)이므로 별도 흐름

### 쉐도잉 vs 튜터링 Level 2 차이

| 구분 | 쉐도잉 (스크립트 모듈) | 튜터링 Level 2 |
|------|----------------------|---------------|
| 진입 | 내 스크립트 선택 → 쉐도잉 | 모의고사 → 진단 → 교정 답변 생성 후 |
| 소스 | 사용자가 만든 스크립트 | AI가 교정한 template_answer |
| 훈련 | 4단계 점진 훈련 (듣기→따라읽기→혼자말하기→실전) | 쉐도잉만 (텍스트 없이 따라 말하기) |
| 평가 | GPT 5영역 상세 평가 (Step 4) | 유사도 평가 (score ≥ 70 통과) |
| DB | shadowing_sessions + shadowing_evaluations | tutoring_attempts (training_level='level2') |
| 결과 | OPIc 등급 추정 + 스크립트 활용도 | 통과/불통과 → Level 3 진행 |

### 선행 조건 (쉐도잉 구현 전 완료되어야 할 것)

쉐도잉은 스크립트 모듈의 결과물을 **소비**하는 기능이므로, 다음이 먼저 구현되어야 한다:

#### 1. 스크립트 모듈 테이블 (스크립트 설계 문서 참조)

| 테이블 | 쉐도잉에서 사용하는 데이터 |
|--------|------------------------|
| `scripts` | `english_text`, `korean_text`, `sentence_data` (JSONB), `key_expressions` (TEXT[]), `question_text`, `question_korean`, `topic` |
| `script_packages` | `wav_url` (TTS 오디오), `json_url` (타임스탬프 JSON), `status` ('completed'만 쉐도잉 가능) |

#### 2. Storage 파일 (패키지 생성 시 자동 생성)

| 파일 | 생성 시점 | 쉐도잉에서 사용 |
|------|----------|----------------|
| `audio/{package_id}.wav` | 패키지 Step 1 (ElevenLabs TTS) | Step 1~3에서 재생 |
| `json/{package_id}.json` | 패키지 Step 2 (Whisper 타임스탬프) | 문장별 구간 재생, 하이라이트 |

#### 3. 공통 인프라 (이미 존재하거나 스크립트 모듈에서 생성)

| 인프라 | 상태 | 비고 |
|--------|------|------|

### 4단계 상세 설계

#### Step 1: 듣기 (Listen) — 클라이언트 완결

**목적**: Kadota Input Effect — 내 스크립트의 음성 패턴을 귀에 익히기

```
[데이터 로드]
  script_packages에서 wav_url + json_url 조회
  JSON에서 shadowing_sentences 파싱
  scripts에서 english_text, korean_text 조회

[재생]
  WAV 전체 재생 (HTMLAudioElement)
  currentTime 기반으로 현재 문장 하이라이트
  0.75x ~ 1.5x 속도 조절
  영문/한글/둘 다 표시 전환
  자동 스크롤
  구간 반복 (A-B repeat)
```

- DB/API 호출 없음
- 1~2회 듣기면 충분 (내가 쓴 글이므로)
- Zustand 로컬 상태 관리

**UI 요소:**
- 재생/정지 버튼 + 속도 조절
- 문장 리스트 (현재 문장 하이라이트)
- 표시 모드 전환 (영문/한글/둘 다)
- 진행바 (전체 오디오 위치)

---

#### Step 2: 따라읽기 (Shadow) — 클라이언트 완결

**목적**: Kadota Practice Effect + ELT School Overlapping 통합 — 텍스트 힌트를 조절하며 따라 말하기

```
[문장 순차 진행]
  1. 현재 문장 표시 (힌트 레벨에 따라)
  2. 해당 구간 오디오 재생 (start~end)
  3. 사용자가 음성과 함께 따라 읽기
  4. 재생 완료 → 완료 표시
  5. 수동으로 다음 문장 이동

[텍스트 힌트 토글 (3단 세그먼트 컨트롤)]
  전체 보기 (Eye 아이콘): 텍스트 전체 표시 — 오버래핑 효과
  첫단어만 (EyeOff 아이콘): 첫 단어만 보이고 나머지 ___ — 점진 난이도
  숨김 (EyeClosed 아이콘): 텍스트 완전 숨김 — 쉐도잉 효과

[진행 표시]
  "3/8 문장" 형태 진행률
  완료한 문장 체크 표시 (프로그레스바)

[힌트별 안내 문구]
  전체: "텍스트를 보며 음성과 함께 읽어보세요"
  첫단어: "첫 단어만 보고 나머지를 떠올려 보세요"
  숨김: "음성만 듣고 따라 읽어보세요"
```

- DB/API 호출 없음, 크레딧 소비 없음
- 기존 5단계의 오버래핑(Step 2)과 쉐도잉(Step 3)을 하나로 통합
- 사용자가 자신의 수준에 맞게 자유롭게 힌트 레벨 전환
- 진도: localStorage (Zustand persist)

---

#### Step 3: 혼자 말하기 (Recite) — 클라이언트 완결

**목적**: OPIc 전용 — TTS/텍스트 없이 전체 답변 + 시간 관리 훈련

```
[실행]
  1. 질문 텍스트만 표시 ("Tell me about your favorite park.")
  2. TTS 재생 없음 — 사용자가 혼자 처음부터 끝까지 말하기
  3. 타이머 표시 (경과 시간)
  4. 전체 녹음

[타이머 시각 피드백]
  0:00 ~ 1:29  →  회색 (아직 짧음)
  1:30 ~ 2:00  →  초록 (적정 구간 ✅)
  2:00 ~ 2:30  →  노랑 (약간 김)
  2:30+        →  빨강 (너무 김)

[Peek 기능]
  막힐 때 "힌트 보기" 버튼 → 다음 문장 영문 표시 (3초간)
  peek 사용 횟수 카운트 (의존도 체크)

[자기 검토]
  녹음 재생 + 원본 TTS 재생 비교
  총 시간, peek 횟수 표시
```

- DB/API 호출 없음, 크레딧 소비 없음
- **OPIc에 꼭 필요한 이유**: 문장별로는 잘 하는데 전체를 이어서 말하면 막히는 문제 해결, 2분 타이밍 연습
- 녹음: MediaRecorder API → Blob('audio/webm')

---

#### Step 4: 실전 (Speak) — Edge Function, 크레딧 소비

**목적**: Kadota Output+Monitoring Effect — 시험 시뮬레이션 + AI 종합 평가

```
[준비]
  크레딧 잔여 확인 (user_credits)
  영어 질문만 표시 — 스크립트 텍스트 완전 숨김

[녹음]
  3초 카운트다운 → 자유 답변 녹음 (최대 3분)
  타이머 표시 (Step 4와 동일 색상 피드백)

[평가 요청] Edge Function: scripts/handlers/shadowing-evaluate.ts
  1. 크레딧 차감 (user_credits)
  2. 녹음 Base64 → Whisper STT (model: whisper-1, language: en)
  3. 발화 길이 검증 (< 5단어 → 크레딧 환불 + 거부)
  4. ai_prompt_templates에서 'evaluate_shadowing' 프롬프트 로드
  5. 변수 치환: {{question_text}}, {{transcript}}, {{training_script}}, {{key_expressions}}
  6. GPT-4.1 평가 (temperature: 0.3, max_tokens: 2000, json_object)
  7. shadowing_sessions INSERT (status='completed')
  8. shadowing_evaluations INSERT
  9. 결과 반환

[결과 화면]
  5영역 레이더 차트 (발음/유창성/문법/어휘/내용)
  OPIc 등급 추정 (IL~AL)
  스크립트 활용도 (0~100%)
  강점/약점/개선 제안
  이전 평가 대비 변화 (해당 스크립트 기록 있을 때)
```

**에러 처리:**
- 5단어 미만 → 크레딧 환불 + "답변이 너무 짧습니다" 메시지
- API 실패 → 크레딧 환불 + 재시도 안내

### 타임스탬프 매칭 알고리즘 (패키지 생성 시)

패키지 생성 Step 2 (`generate-shadowing.ts`)에서 실행. 쉐도잉 Step 1~3이 이 결과물에 의존.

```
[입력]
  - WAV 오디오 파일 (Storage에서 다운로드)
  - sentence_data (scripts 테이블에서 조회)

[Step 1] Whisper word-level STT
  → response_format='verbose_json', timestamp_granularities=['word']
  → {text, words: [{word, start, end}, ...]}

[Step 2] 문장-단어 매칭 (sentence-matcher.ts)
  → 각 GPT 문장을 Whisper 단어 배열에서 슬라이딩 윈도우로 검색
  → 텍스트 정규화: 소문자 + 구두점 제거 + 아포스트로피 제거
  → Levenshtein Distance 유사도 계산
  → 가장 높은 유사도 위치의 타임스탬프 추출

[Step 3] 갭 보정 (fillTimestampGaps)
  → 문장 간 갭 > 0.5초 → 중간값으로 조정
  → 오버랩 → 경계를 중간값으로 조정

[Step 4] JSON 생성 + Storage 업로드
  → shadowing_sentences JSON 구조 (위 참조)
  → script_packages 업데이트 (json_url, status='completed')
```

  playbackRate: number;
  displayMode: 'both' | 'english' | 'korean';

  // === Step 2: 따라읽기 ===
  shadowIndex: number;

  shadowCompleted: number[];

  reciteTimer: number;
  recitePeekCount: number;
  reciteShowPeek: boolean;

async function startRecording(): Promise<MediaRecorder> {
  const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
  const recorder = new MediaRecorder(stream, { mimeType: 'audio/webm' });
  const chunks: Blob[] = [];
  recorder.ondataavailable = (e) => chunks.push(e.data);
  recorder.start();
  return recorder;
}

function stopRecording(recorder: MediaRecorder): Promise<Blob> {
  return new Promise((resolve) => {
    recorder.onstop = () => resolve(new Blob(chunks, { type: 'audio/webm' }));
    recorder.stop();
  });
}
