## 핵심 철학

### 튜터링의 목적은 딱 하나: 다음 시험에서 등급을 올리는 것

- 개별평가가 "뭐가 틀렸는지" 알려줌
- 종합평가가 "뭘 해야 하는지" 알려줌
- **하지만 "아는 것"과 "할 수 있는 것"은 다름**
- 튜터링은 이 간극을 메움 — "알고 있는 것"을 "실제로 할 수 있는 것"으로 전환

### 3개 모듈 역할 (의사 → 처방전 → 물리치료)

| 모듈 | 역할 | 비유 |
|------|------|------|
| 개별 평가 (Stage B) | 진단 + 교정 | 의사의 진찰 |
| 종합 평가 (Stage C) | 진단서 + 처방전 | 의사의 소견서 |
| **튜터링** | **치료 + 재활 + 추적** | **물리치료사** |

### 핵심 원칙: "피드백은 1회, 수행은 5회"

- 한 문항을 평가하고 끝내면 "아는 것"에서 멈춤
- 한 문항/한 능력을 **형태를 바꿔 5번 수행**해야 "할 수 있는 것"이 됨

## 기존 설계 대비 근본적 변화

| 항목 | 기존 v2 (4레벨) | 새 v3 (세션+프로토콜) |
|------|---------------|-------------------|
| 훈련 구조 | answer_type별 L1→L2→L3→L4 | **단일 세션 6 Screen + 5 프로토콜** |
| 훈련 단위 | "교정 답변 하나를 외우기" | **"능력을 여러 문항으로 체화"** |
| 새 질문 연습 | 없음 | **매 세션 471개 DB에서 새 질문 제시** |
| 오류 패턴 교정 | 없음 | **Oral Transformation + Self-repair** |
| 시간 압박 훈련 | 없음 | **15-60-15 타이머** |
| 변형 대응력 | 없음 | **Forced Variation (강제 변주)** |
| 자기수정 습관 | 없음 | **Self-repair 의무 수행** |
| 과제 생성 | pass_rate 기반 수동 | **종합평가 처방에서 자동 변환** |
| 등급별 차이 | 동일 경로 | **파라미터만 자동 조절** |
| 스크립트/쉐도잉 | 별도 모듈 | **인라인 호출 (필요 시점에만)** |

## 탭 구조: 진단 | 처방 | 훈련

### 진단 탭

**목적**: 현재 레벨/약점/우선순위 확인

**데이터 소스**:
- 모의고사 종합평가: 확정 등급, FACT 점수, 반복실수 패턴, 유형별 강/약
- 개별평가 6-Layer: 문항별 영역점수, 구조분석, L1 패턴, 말하기습관
- 이전 튜터링 세션: 성장 추이, KPI 달성률

**표시 항목**:
- 연결된 모의고사 선택 (드롭다운)
- 확정 등급 + 목표 등급
- 핵심 약점 Top 3 (행동 지표로 변환된 형태)
  - 예: "과거시제 일관성 60% 미만", "예시 디테일 2개 미만", "침묵 총합 5초 초과"
- 성장 추이 그래프 (모의고사별 변화)

### 처방 탭

**목적**: "이번 주/이번 세션에서 무엇을 어떤 순서로 훈련할지" 자동 설계

**표시 항목**:
- 목표 등급 & 목표 유형
- Top 3 약점 (우선순위 / 원인 / 교정 규칙)
- 추천 훈련 세션 목록 (자동 생성)
  - 각 세션: 유형 + 토픽 + 핵심 프로토콜 + 예상 시간
  - 우선순위: 🔴 높음 / 🟡 권장 / 🟢 추가
- 세션 생성 버튼: "30분 1회" / "20분 압축" / "시험 전 워밍업 10분"

**처방 자동 생성 데이터 흐름**:
```
종합평가 coaching_report (GPT 출력)
  ├─ top3_priorities (3개) → 약점 Top2 추출
  ├─ recurring_patterns (3~5개) → 오류 패턴 드릴 포함
  ├─ question_type_map (10개) → 약한 유형 Top 2~3
  └─ 개별평가 (14문항) → 구조점수 ≤ 2 문항 추출
      │
      ▼
  tutoring_prescription (EF 자동 변환, §모의고사-평가-v3.md §3-8a)
  → priority_weaknesses + error_drill_tags + weak_types + training_order
      │
      ▼
  처방 엔진: 약점 Top2 + 유형 1개 → 세션 자동 구성
```

### 훈련 탭

**목적**: 실제 훈련 수행 — "말이 바뀌는" 반복 루프

**내부 모드 3가지** (상단 토글):
1. **코치드 훈련**: Screen 0~6 기반, 프로토콜 내장 (기본)
2. **자율 연습**: 문항/유형 선택 후 반복 녹음 (시간 제한 옵션)
3. **실전 시뮬**: 미니 실전(3문항/12분) ~ 풀 실전(14문항/90분)

## 코치드 훈련 세션: Screen 0~6 상세 설계

### 세션 시간 구성 (30분 표준)

| Screen | 이름 | 시간 | 핵심 프로토콜 |
|--------|------|------|-------------|
| 0 | 세션 브리프 | 1분 | — |
| 1 | 워밍업 입력 | 4분 | 쉐도잉 |
| 2 | EPP 패턴 고정 | 5분 | **EPP** |
| 3 | 변형 훈련 | 7분 | **Forced Variation + Oral Transformation** |
| 4 | 타임드 실전 | 7분 | **15-60-15** |
| 5 | Self-repair | 4분 | **Self-repair** |
| 6 | 리캡 | 2분 | — |

### Screen 0: 세션 브리프 (1분)

**사용자에게 보이는 것**:
- 오늘 세션 목표 1문장 (예: "과거경험 2개를 연결어로 묶어 40초 이상 유지")
- 오늘의 약점 2~3개 + 금지 습관 1개 (예: "음…/and and 반복 금지")
- 성공 기준 체크리스트 3개 (예: "EPP 3문장 완성 / 변형 2회 성공 / Self-repair 1회 이상")
- 오늘 사용할 프로토콜 아이콘 + 예상 소요시간

**사용자 행동**: 확인 → "시작" 버튼

**시스템**:
- GPT: 처방 데이터(top3_priorities + recurring_patterns + question_type_map) → 목표/성공기준 생성

### Screen 1: 워밍업 입력 (4분)

**사용자에게 보이는 것**:
- 오늘 유형의 모범 답변 (등급별 텍스트 정책 적용)
- 문장별 TTS 재생 버튼
- 쉐도잉 2회 (따라 말하기)
- 리듬/속도 간단 지표 (정교한 채점보다 "리듬 맞추기" 중심)

**사용자 행동**: 모범 답변 듣기 → 2회 따라 말하기

**시스템**:
- TTS: Gemini 문장별 타임스탬프 (기존 스크립트 패키지 활용)
- Azure Speech SDK: 발음 accuracy/fluency (간단 표시)
- 쉐도잉 모듈 인라인 호출 (기존 구현 재활용)

**등급별 텍스트 정책**:
- IL~IM1: 전체 텍스트 ON + 키워드 하이라이트
- IM2~IH: 키워드만 (문장 전체 OFF), 필요 시 1회 힌트
- AL: 텍스트 OFF, 음성만

### Screen 2: EPP 패턴 고정 (5분)

**EPP = Essential Pattern Pack**
오늘 유형에서 반드시 나오는 뼈대를 구어체로 고정 (암기 아닌 슬롯화)

**사용자에게 보이는 것**:
- 질문 1개 (해당 question_type)
- EPP 뼈대 카드 2~3개 (문장 템플릿)
  - 예: "To be honest, [주제] is something I [동사]..."
  - 예: "The main reason is that [이유]..."
  - 예: "For example, last [시간], I [구체적 행동]..."
- 오늘의 필수 연결어/문법 2~3개
- 오늘의 금지 습관 (recurring_patterns에서 추출)

**사용자 행동**:
- EPP 뼈대를 **내 문장으로** 3회 반복 (짧게, 각 15~20초)

**시스템**:
- GPT: 사용자 레벨/오류 기반으로 EPP 선정 + 필수 요소 결정
- Whisper: STT
- GPT: "한국인식 직역" 경고 (예: "I think so" 남발, "There are many people" 공허문장)
- 스크립트 모듈: 슬롯 구조 활용 (기존 scripts 테이블의 slots 데이터)

### Screen 3: 변형 훈련 (7분)

**Forced Variation (강제 변주) + Oral Transformation (구두 변환)**

**사용자에게 보이는 것**:

*Forced Variation 파트* (4~5분):
- 동일 question_type에서 변형 카드 2~3장 자동 제시
  - 시간 변화: now vs used to
  - 조건 변화: alone vs with friends
  - 비교: A vs B
  - 이유 반전: 좋아하지만 단점 1개
- 각 카드마다 "연결어 1개 + 디테일 1개" 강제 미션
- questions DB(471개)에서 같은 유형의 다른 질문도 활용

*Oral Transformation 파트* (2~3분):
- 짧은 문장 카드 6~10개
- 즉석 변환 지시: 시제 바꾸기 / 이유 추가 / 비교 / 가정법
  - "I go there on weekends."
  - → "I used to go there..."
  - → "If I had more time, I would..."
  - → "Compared to eating out, cooking is..."

**사용자 행동**:
- Variation: 30~45초씩 2~3회 짧게 말하기 (매번 새로운 디테일 필수)
- Transformation: 카드 1개씩 즉시 말로 변환 (2~4초 내)

**시스템**:
- GPT: 변형 프롬프트 생성 (471 DB 태그 + 처방 기반)
- Whisper: STT + 중복 표현/내용 반복 탐지
- Azure: fluency 변화 측정 (변형 시 끊김 확인)
- GPT: Oral Transformation 정답 판정 ("의도 충족" 기준, 대체 정답 허용)

### Screen 4: 타임드 실전 (7분)

**15-60-15 프레임**
- 15초: 구조(블록) 잡기
- 60초: 본론 수행
- 15초: 마무리(요약/느낌/추천/교훈)

**사용자에게 보이는 것**:
- 실제 시험 UI처럼: 질문 + 준비시간 카운트다운 + 답변시간 카운트다운
- 2라운드 (같은 유형, 다른 질문)
- 필수 블록 체크리스트 (도입/배경/사건/디테일/감정/마무리)

**사용자 행동**:
- 15초 준비 → 60초 말하기 (녹음) → 15초 즉시 마무리
- 2번 반복 (다른 질문)

**시스템**:
- MediaRecorder: 녹음 + 무음 감지 (침묵 구간 표시)
- Whisper: STT
- Azure: 발음/유창성/억양 + 단어별 점수
- GPT: (a) 필수 블록 충족 여부 (b) 문법/표현 교정 (c) 시간 분석
  - 누락 블록 → 다음 라운드에 강제 미션으로 부여

**등급별 타이머 차이**:
- IL→IM: **15-30-15** (60초 강제 X, 짧게)
- IM→IH: **15-60-15** (정규)
- IH→AL: **15-75-15** 또는 **즉답(0-60-0)**

### Screen 5: Self-repair (4분)

**사용자에게 보이는 것**:
- Screen 4 STT 텍스트에 하이라이트 3종:
  1. 🔴 문법/형태 오류
  2. 🟡 L1 간섭 (의미는 통하지만 어색)
  3. 🔵 더 고급 연결/확장 포인트
- Self-repair 스크립트 버튼:
  - "I mean..." / "Let me rephrase that." / "What I'm trying to say is..."

**사용자 행동**:
- 하이라이트 중 2개 선택 → **Self-repair 문장을 넣어서 직접 고쳐 재녹음** (10~20초)
- IM→IH: 최소 1회 Self-repair 수행 필수 (게이트)
- IH→AL: 최소 2회, "자연스러운 리프레이즈"로 품질 상승

**시스템**:
- GPT: 오류 분류 (문법 vs L1 간섭 vs 확장) + 수정 우선순위 제안
- TTS: Self-repair 예시 발화 (짧게)
- Whisper + Azure: 재발화 평가 (수정 전/후 비교 저장)

### Screen 6: 리캡 + 다음 세션 연결 (2분)

**사용자에게 보이는 것**:
- 오늘의 결과 3줄:
  1. 가장 좋아진 점
  2. 다음에 고칠 1개
  3. KPI 달성 여부 (체크리스트 3개 기준)
- 다음 세션 자동 추천 (같은 유형 반복 vs 약점 전환)
- 버튼 2개:
  - "같은 타입 다른 토픽으로" → 바로 다음 세션
  - "미니 실전 3문항" → Simulation 모드

**시스템**:
- GPT: 오늘 수행 데이터 분석 → 3줄 요약 + 다음 처방 업데이트
- 처방 엔진: 약점 점수 갱신 + 다음 세션 자동 생성

## 5대 훈련 프로토콜 상세

### 1. EPP (Essential Pattern Pack) — "뼈대 자동화"

**목적**: 유형별 필수 답변 구조를 반사적으로 출력하게 만들기
**OPIc 등급 연결**: 구조 점수 직결. IM 이상은 구조가 있어야 함.

**프로토콜**:
- 유형별 필수 뼈대 2~3개를 슬롯 형태로 제공
- 학습자가 자기 내용으로 슬롯을 채워 반복 발화
- 암기가 아닌 "뼈대는 고정, 내용은 자유"

**유형별 EPP 예시**:

| question_type | 필수 뼈대 |
|---------------|----------|
| description | 전체 소개 → 개인 연결 → 구체적 예시 → 마무리 느낌 |
| routine | 빈도/시간 → 구체 행동 → 이유 → 좋아하는 점 |
| past_experience_* (3종) | 시점 설정 → 에피소드 → 문제/전환 → 감정/교훈 |
| comparison | A 설명 → B 설명 → 차이점 → 내 선택 + 이유 |
| roleplay_12 | 사과 → 이유 → 옵션 2개 → 상대 선택 유도 |
| advanced_15 | 입장 → 이유 1 + 예시 → 이유 2 + 설명 → 결론 |

### 2. Forced Variation (강제 변주) — "암기 깨기"

**목적**: 외운 답 깨기 + 변형 질문에도 동일 품질 유지
**OPIc 등급 연결**: IH 판정 핵심. 같은 유형 다른 질문에서도 수행 유지.

**프로토콜**:
- 같은 뼈대로 조건만 바꿔 4~6회 변주
- 변경 요소: 시간/장소/대상/감정/비교/이유반전
- 룰: **같은 첫 문장 금지, 같은 디테일 중복 금지**
- 시스템이 중복 감지 → 새 디테일 강제

**등급별 차이**:
- IL→IM: 변경 요소 1개만
- IM→IH: 변경 요소 2개
- IH→AL: 즉흥 랜덤 카드 (가정/반전/예외/트레이드오프)

### 3. Oral Transformation (구두 변환) — "문법 말로 자동화"

**목적**: 문법을 "문제풀이"가 아닌 "말로 자동 출력"으로 체화
**OPIc 등급 연결**: IM→IH 가장 빠른 길. 정확성 + 유창성 동시 상승.

**프로토콜**:
- 문장 1개 제시 → 즉석 변환 조건 → 2~4초 내 말로 출력
- 변환 유형: 시제/주어/이유 추가/비교/가정법/강조

**등급별 차이**:
- IL→IM: 현재↔과거, because 1개 붙이기
- IM→IH: 비교/이유절/간단 관계절
- IH→AL: 완곡표현/조건부 주장/부분동의 ("I partially agree, but...")

**한국인 L1 간섭 집중 교정 대상**:
1. 문장 골격 부재 (주어-동사 늦게 나옴)
2. 시제 일관성 붕괴 (과거 중 현재로 튐)
3. 과도한 겸손/회피 ("I'm not sure" 반복)
4. 직역형 연결 ("그래서… 근데…" 루프)
5. 발화 단위 짧음 (Chunking 실패)

### 4. 15-60-15 (타임드 압축-확장-압축) — "시간 운영"

**목적**: 시간 압박 하 실전 감각 + 길이 확보/정리 능력
**OPIc 등급 연결**: IM→IH 탈락 70%는 시간 구조 실패

**프로토콜**:
- 15초: 구조(블록) 잡기 — 키워드 3개만 머릿속으로
- 60초: 본론 수행 — 끊기지 않고 완주
- 15초: 마무리 — "방금 내 답변에서 1문장만 더 좋게"

**EPP 드릴 연계 (같은 소재 3회전)**:
- 30초 버전: 핵심만 (배경1 + 사건2 + 감정1 + 결론1)
- 60초 버전: 디테일 2개 추가
- 90초 버전: 문제-해결/교훈/추천까지 확장
→ 같은 소재를 길이만 바꿔 말할 수 있으면 시간 운영 안정

**등급별 타이머**:
- IL→IM: 15-30-15 또는 10-40-10
- IM→IH: 15-60-15 (정규)
- IH→AL: 15-75-15 또는 0-60-0 (즉답)

### 5. Self-repair (자기수정 루프) — "IH→AL 분기점"

**목적**: 실수해도 자연스럽게 고치고 이어가는 능력
**OPIc 등급 연결**: AL 채점에서 강하게 반영. "완벽한 문법"보다 "자연스러운 수정"이 중요.

**프로토콜**:
- 본인 발화에서 시스템이 오류 2~3개 하이라이트
- Self-repair 템플릿을 넣어서 **즉시 고쳐 재녹음**
  - "I mean..." / "Let me rephrase that." / "What I'm trying to say is..."
  - "Actually, I should say..." / "Well, not exactly..."
- 수정 전/후 비교 저장 → 성장 추적

**3초 규칙**: 실수 인지 → 3초 내 복구 못 하면 그냥 진행 (멈추지 않기)

**등급별 차이**:
- IL→IM: 끊기지 않고 다시 말하기 (재진술)
- IM→IH: 연결어 업그레이드 + 직역 제거
- IH→AL: 리프레이즈로 품질 상승 (정정이 아닌 업그레이드)

## 등급별 파라미터 종합표

| 파라미터 | IL→IM | IM→IH | IH→AL |
|---------|-------|-------|-------|
| **텍스트** | 전체 ON | 키워드만 | OFF |
| **EPP** | 3개 고정표현, 짧게 | 2~3개 + 확장 1개 | 1~2개 (템플릿 티 방지) |
| **Variation 변경 요소** | 1개 | 2개 | 즉흥 랜덤 |
| **Transformation** | 현재/과거 | 비교/이유절 | 완곡/조건부/부분동의 |
| **타이머** | 15-30-15 | 15-60-15 | 15-75-15 또는 즉답 |
| **Self-repair** | 끊기지 않고 재진술 | 연결어 업그레이드 (1회 필수) | 메타발화 자연화 (2회 필수) |
| **GPT 피드백** | 오류 1~2개만 | L1 간섭 강하게 | 논리/깊이 평가 |
| **필수 블록** | 3개 (도입/본론/마무리) | 5개 (+디테일/감정) | 6개 (+논리전개/대안) |
| **목표 답변 길이** | 30~45초 | 60~75초 | 75~90초 |

### 등급별 병목과 뚫는 훈련

**NH/IL → IM**:
- 병목: 문장 생성 느림, 침묵, 기본 시제 불안
- 훈련: 고정 블록 20개 생존력 확보, 30초 답변만 완주, Oral Transformation 쉬운 버전

**IM → IH** (핵심 타겟, 가장 많은 구간):
- 병목 ①: 에피소드 없음 (경험→루틴으로 도망)
- 병목 ②: 디테일 부족 (추상적, general)
- 병목 ③: 시간 운영 실패 (중간 멈춤/급마무리)
- 훈련: EPP 30/60/90 드릴, 디테일 체크리스트 6개 강제 (when/where/who/what/how/feeling), 15-60-15 타이머

**IH → AL**:
- 병목: 예측 가능/반복적, 돌발/가정/비교에서 깊이 부족
- 훈련: Forced Variation 고난도 5연타, 주장-근거-예시-반론 자동화, Self-repair 고급화

## 처방 → 세션 자동 생성 규칙

### Rule 1: "약점 Top2 + 유형 1개"로 세션 구성
- 한 세션에 교정 포인트를 2개 넘기지 않음 (3개 이상이면 체화율 급락)
- 문항 유형은 처방 우선순위 + 모의고사 취약 유형 반영

### Rule 2: 약점을 "행동 지표"로 변환
- "문법 약함" → "과거시제 일관성 60% 미만"
- "할 말 없음" → "예시 디테일(when/where/who) 2개 미만"
- "시간 관리 실패" → "답변 시간 30초 미만 또는 90초 초과"
- 이 행동 지표가 Screen 4/5의 피드백 기준

### Rule 3: 등급별 파라미터 자동 세팅
- 확정 등급 → 텍스트 정책/타이머/반복횟수/EPP 개수 자동 결정
- 등급 파라미터 테이블 참조 (위 종합표)

### Rule 4: "성공하면 확장, 실패하면 분해"
- Screen 4에서 목표 길이/구조 달성 → 다음 세션: Variation 난이도↑ 또는 유형 확장
- 실패 → 다음 세션: EPP/Transformation 비중↑ (실전 비중↓)

### Rule 5: 시험 D-day 근접 시 Simulation 비율 자동 상향
- D-7: 미니 실전(3문항) 세션당 1회 삽입
- D-3: 6문항/25분 중간 실전 삽입
- D-1: 워밍업 10분 + 실전모드(90분) 권장

### 세션 우선순위 점수 계산

```typescript
priority = (약점도 * 0.5) + (시험출현빈도 * 0.2) + (최근훈련공백 * 0.2) + (사용자선호토픽 * 0.1)
```

- IM→IH는 특히 routine/past_experience_memorable/comparison/roleplay_12를 자주 돌림 (점수 효율 높음)
- 토픽: 약한 토픽 우선. 단, 초보는 친숙 토픽 비중↑ (말이 나와야 훈련이 됨)

## 실전 시뮬레이션 (Simulation)

### 기존 모의고사와의 차이

| 항목 | 모의고사 | 실전 시뮬 |
|------|---------|----------|
| 목적 | 진단 + 등급 확인 | 훈련 성과 확인 + 실전 감각 |
| 문항 구성 | 표준 세트 (고정) | **약점 유형 비중 높인 적응형** |
| 길이 | 15문항/90분 | 미니(3문항/12분) ~ 풀(14문항/60분) |
| 중간 개입 | 없음 | 없음 (동일) |
| 종료 후 | 개별+종합 평가 | **"이 문항을 Training으로" 버튼** → 바로 세션 생성 |

### 재활용 vs 별도 구현

**재활용 (기존 모의고사 모듈 70%)**:
- 녹음/타이머 UI
- 문항 플레이/진행 로직
- 기본 결과 표시

**별도 구현 (30%)**:
- Adaptive Set 구성: 약점 유형 TOP3 비중 높여 세트 재구성
- Post-Sim Review: 문항별 "Training으로 다시" 버튼 → 해당 type+topic으로 세션 자동 생성
- 미니 실전 모드 (3문항/6문항 옵션)

```

  │      ├── 개별평가 6-Layer + v3 task_fulfillment

  │      ├── priority_weaknesses → 약점 Top 2 추출
  │      ├── error_drill_tags → 오류 패턴 드릴 포함
  │      ├── weak_types + training_order → 유형/순서 결정
  │      └── must_fix_for_next_grade (drill_tag 기반)
  │
  └──→ [훈련 탭] 오늘 세션 자동 구성 + Simulation 추천
         │
         ├── Screen 1: 쉐도잉 모듈 인라인 호출 (워밍업)
         │     └── 텍스트 ON/OFF는 등급 파라미터가 자동 제어
         │
         ├── Screen 2: 스크립트 모듈 활용 (EPP 슬롯 구조)

         │
         ├── Screen 5: 스크립트 모듈 활용 (대체 표현 후보)
         │
         └── Screen 6 → "미니 실전" / "풀 실전" 바로 연결
               │
               └──→ 실전 결과 → 다시 진단/처방 업데이트 (폐루프)

핵심: 진단 → 처방 → 훈련 → (미니)실전 → 진단 업데이트의 폐루프
```

### tutoring_prescriptions (처방 과제 — 신규)

```sql
CREATE TABLE tutoring_prescriptions (

  priority            SMALLINT NOT NULL,       -- 1=높음, 2=권장, 3=추가
```

### ai_prompt_templates + script_specs (스크립트와 공유)

## 세션 시간 변형

### 20분 압축 (시간 부족/시험 직전)

- Oral Transformation 생략, Variation 2회로 축소

### 30분 표준 (기본)
- 위 설계 그대로

### 40분 확장 (여유 있을 때)
- 변형 4회 + 미니실전 2문항 + 리뷰 강화

### 10분 워밍업 (시험 당일)
- 쉐도잉 3분 + EPP 3분 + 미니실전 1문항 4분

## 절대 빼지 마라 TOP 3

1. **타임드 실전 + 15-60-15** (Screen 4)
   - 시간 압박 없으면 실전형으로 안 바뀜. IM→IH 탈락 70%는 시간 구조 실패.

2. **Self-repair** (Screen 5)
   - IH→AL 분기점. "틀려도 자연스럽게 고치고 이어가기"가 실제 채점에서 강하게 반영.

3. **처방 → 세션 자동 생성**
   - 사용자가 매번 "뭘 할지" 고민하면 이탈. 자동 연결이 제품 성과의 엔진.

## 나중에 해도 된다 TOP 3

1. **정교한 발음 채점 (phoneme-level)**
   - 멈춤/속도/강세 거친 지표로 충분. OPIc은 전달력/구조/유지시간이 우선.

2. **마이크로 태스크 초개인화**
   - 유형별 템플릿 + 변주 카드만으로 커버. 데이터 쌓인 뒤 고도화.

3. **스크립트 완전 자동 생성의 고급화**
   - EPP 슬롯 + 키워드 + self-repair 대체표현이면 충분. 오히려 템플릿 과의존 위험.
