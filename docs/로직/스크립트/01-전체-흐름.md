# 01. 전체 흐름

> 스크립트 생성부터 쉐도잉 훈련까지의 전체 동작 흐름

---

## 사용자 시나리오

### Phase 1: 스크립트 생성

```
① 주제·질문 선택 (위저드 Step 1)
   카테고리(일반/롤플레이/어드밴스) → 주제(빈도순) → 질문 선택
   → TopicPagination + QuestionSelector 컴포넌트
   → 기존 스크립트가 있으면 바로 "보기" 가능

② 경험 입력 (위저드 Step 2)
   사용자가 한국어로 경험/스토리 입력
   목표 등급(NH~AL) 선택
   → 입력 없으면 "Level 3 확장 정책" 적용 (GPT가 자체 생성)

③ AI 생성 대기 (위저드 Step 3)
   createScript SA → 크레딧 차감 → draft UPSERT → fire-and-forget EF
   EF가 Two-Pass 생성 (Pass 1: 스크립트, Pass 2: 학습 분석)
   3초 간격 폴링으로 완료 감지 → english_text 있으면 자동 전환

④ 결과 확인 (위저드 Step 4)
   4가지 보기 모드 (영한병기/영어만/한글만/분리)
   핵심 정리 7가지 학습 콘텐츠 (인터랙티브 하이라이트)
   수정 요청 (최대 3회, 크레딧 무료)
   → 확정 → 패키지 생성 시작

⑤ 완료 (위저드 Step 5)
   패키지 생성 진행률 표시
   패키지 완료 → 쉐도잉 훈련 CTA
```

### Phase 2: 패키지 생성

```
⑥ Phase 1: TTS 음성 생성
   확정된 스크립트 → Gemini TTS API 호출
   PCM(24kHz) → 리샘플링(44.1kHz) → WAV 변환
   → Supabase Storage 업로드 (script-packages/audio/)

⑦ Phase 2: 타임스탬프 생성
   WAV 파일 → Whisper STT (word-level timestamps)
   슬라이딩 윈도우 + Levenshtein 유사도로 문장-단어 매칭
   → 갭/오버랩 보정 → JSON Storage 업로드

   Phase 2 실패 시 → "partial" 상태 (음성은 사용 가능, 문장 하이라이트 불가)
```

### Phase 3: 쉐도잉 훈련

```
⑧ Step 1: 전체 듣기
   원어민 TTS 음성 재생 + 문장별 하이라이트
   속도 조절 (0.75x/1.0x/1.25x/1.5x)
   표시 모드 (영한/영어만/한국어만)
   문장 클릭 → 해당 위치로 점프
   문장 반복 재생 토글

⑨ Step 2: 따라읽기
   문장별로 듣기 → 따라 말하기
   텍스트 힌트 3단계 (전체/첫 단어만/숨김)
   완료한 문장 체크 + 진행 추적

⑩ Step 3: 혼자 말하기
   텍스트 없이 혼자 스크립트 암기 말하기
   타이머 + 엿보기 기능 (횟수 카운트)
   OPIc 학습 팁 표시 (TanStack Query)

⑪ Step 4: 실전 녹음 + AI 평가
   질문만 표시, 텍스트 숨김
   녹음 (최대 3분) → 스크립트 크레딧 1개 소모
   Whisper STT → GPT-4.1 평가 (5영역 + OPIc 등급)
   환불 조건: STT 실패, 5단어 미만, API 오류
```

## 전체 데이터 흐름도

```

   ▼

└──────────────────────┬───────────────────────┘
                       │ fire-and-forget
        ┌──────────────▼──────────────┐
        │    EF: scripts/generate     │
        │  Pass 1: GPT-4.1 스크립트   │
        │  (등급 규격 + 슬롯 구조)     │

        └──────────────┬──────────────┘
                       │
   ┌───────────────────▼───────────────────┐
   │         위저드 Step 3~4               │
   │  폴링(3초)으로 완료 감지 → 결과 표시   │
```

## 상태 전이 다이어그램

### 스크립트 상태 (scripts.status)

```
    ┌────────┐
    │ draft  │ ← UPSERT 생성
    └───┬────┘
        │ confirmScript
        ▼
    ┌───────────┐
    │ confirmed │ → 패키지 생성 가능
    └───────────┘
```

### 패키지 상태 (script_packages.status)

```
processing → completed (Phase 1+2 성공)
          → partial   (Phase 1 성공, Phase 2 실패 — 음성만 사용 가능)
          → failed    (Phase 1 실패)
```

### 쉐도잉 세션 상태 (shadowing_sessions.status)

```
    ┌─────────┐
    │ started │ ← 세션 생성 (Step 4 녹음 제출 시)
    └────┬────┘
         │ 평가 완료
         ▼
    ┌───────────┐
    │ completed │
    └───────────┘
```

## 크레딧 시스템

### 소모 시점

| 작업 | 크레딧 종류 | 소모량 |
|------|-----------|--------|
| 스크립트 생성 (generate/correct) | 스크립트 크레딧 | 1개 |
| 쉐도잉 실전 평가 (Step 4) | 스크립트 크레딧 | 1개 |
| 스크립트 수정 (refine) | — | 무료 (최대 3회) |
| 패키지 생성 | — | 무료 |

### 소진 우선순위

```
1순위: plan_script_credits  (플랜 구매 → 만료됨)
2순위: script_credits       (횟수권 → 영구)
```

### 환불 조건

| 상황 | 환불 |
|------|------|
| 스크립트 생성 실패 (EF 오류) | ❌ (draft는 이미 생성됨) |
| 쉐도잉 평가: Whisper STT 실패 | ✅ 자동 환불 |
| 쉐도잉 평가: 5단어 미만 발화 | ✅ 자동 환불 |
| 쉐도잉 평가: GPT API 오류 | ✅ 자동 환불 |

## Server Actions 전체 목록

| # | 함수 | 역할 |
|---|------|------|
| 1 | `createScript` | 크레딧 차감 + draft UPSERT + fire-and-forget EF |
| 2 | `createCorrectScript` | 교정 모드 (사용자 영어 답변 교정) |
| 3 | `refineScript` | 수정 요청 (최대 3회, 무료) |
| 4 | `confirmScript` | 스크립트 확정 (draft → confirmed) |
| 5 | `deleteScript` | 스크립트 삭제 (패키지 + Storage 포함) |
| 6 | `getMyScripts` | 내 스크립트 목록 |
| 7 | `getScriptDetail` | 스크립트 상세 (폴링 대상) |
| 8 | `getScriptSpec` | 등급별 규격서 조회 |
| 9 | `checkScriptCredit` | 크레딧 잔량 확인 |
| 10 | `getOpicTips` | OPIc 학습 팁 (Step 3 대기 화면) |
| 11 | `createPackage` | 패키지 생성 (Phase 1 + Phase 2) |
| 12 | `getShadowingData` | 쉐도잉 데이터 로드 |
| 13 | `startShadowingSession` | 쉐도잉 세션 생성 |
| 14 | `getShadowingHistory` | 쉐도잉 이력 조회 |
| 15 | `getShadowableScripts` | 훈련 가능 스크립트 목록 |
| 16 | `getShadowingEvaluation` | 쉐도잉 평가 결과 조회 |

## Edge Function 전체 목록

| # | 함수 | 엔드포인트 | 역할 |
|---|------|-----------|------|
| 1 | `scripts` | `/generate` | Two-Pass 스크립트 생성 |
| 2 | `scripts` | `/correct` | Two-Pass 교정 생성 |
| 3 | `scripts` | `/refine` | 기존 스크립트 수정 |
| 4 | `scripts` | `/evaluate` | 쉐도잉 실전 평가 (STT + GPT) |
| 5 | `scripts-package` | `/generate-package` | Phase 1: Gemini TTS → WAV |
| 6 | `scripts-package` | `/generate-shadowing` | Phase 2: Whisper → 타임스탬프 |

## 타이밍 요약

| 단계 | 소요 시간 | 비고 |
|------|----------|------|
| 스크립트 생성 (Pass 1+2) | ~30초 | GPT-4.1 × 2회 |
| 스크립트 수정 | ~30초 | Pass 1+2 재생성 |
| 패키지 Phase 1 (TTS) | ~20초 | Gemini TTS |
| 패키지 Phase 2 (타임스탬프) | ~10초 | Whisper STT |
| 쉐도잉 평가 | ~15초 | Whisper + GPT-4.1 |
