# 03. 패키지 생성

> 확정된 스크립트를 TTS 음성 + 타임스탬프로 변환하는 2-Phase 파이프라인

---

## 개요

패키지 = TTS 음성 파일(WAV) + 문장별 타임스탬프(JSON)

```
확정된 스크립트 → Phase 1: TTS → Phase 2: 타임스탬프 → 쉐도잉 훈련 가능
```

패키지 생성은 SA(createPackage)에서 **순차적으로 await** 호출한다 (fire-and-forget이 아님).

---

## Server Action: createPackage

```
createPackage(scriptId) :
   │
   ├─ ① 스크립트 확인 (status === 'confirmed')
   │
   ├─ ② Phase 1 — generate-package EF (await)
   │   → Gemini TTS → PCM → WAV → Storage
   │   → 성공: { package_id, wav_file_path }
   │   → 실패: status='failed', 에러 반환
   │
   ├─ ③ Phase 2 — generate-shadowing EF (await)
   │   → WAV → Whisper STT → 문장 매칭 → JSON
   │   → 성공: status='completed'
   │   → 실패: status='partial' (음성은 사용 가능)
   │
   └─ ④ 결과 반환
       → completed: 풀 기능 쉐도잉
       → partial: 음성 재생만 가능 (문장 하이라이트 불가)
       → failed: 전체 실패
```

## Phase 1: TTS 음성 생성 (scripts-package/generate-package)

### 동작 흐름

```
스크립트 조회
   │
   ├─ 권한 확인 (script.user_id === user_id)
   ├─ 상태 확인 (status === 'confirmed')
   │
   ├─ 기존 패키지 삭제 (재생성 시)
   │   → Storage 파일 삭제
   │   → script_packages 레코드 삭제
   │
   ├─ 패키지 레코드 생성 (status: 'processing', progress: 10)
   │
   ├─ Gemini TTS API 호출
   │   │
   │   ├─ 모델: gemini-2.5-pro-preview-tts
   │   ├─ 보이스: Zephyr (기본) / 사용자 선택 가능
   │   ├─ 프롬프트: "Read aloud as if you are talking to a friend:\n{영어 텍스트}"
   │   ├─ 응답: base64 인코딩된 PCM 오디오 (24kHz, 16bit, 모노)
   │   └─ 타임아웃: 180초
   │
   ├─ 오디오 변환 파이프라인
   │   │
   │   ├─ base64 → Uint8Array
   │   ├─ PCM 24kHz → 리샘플링 44.1kHz (선형 보간)
   │   └─ PCM → WAV (44바이트 헤더 + PCM 데이터)
   │
   ├─ Storage 업로드
   │   → 경로: script-packages/audio/{package_id}.wav
   │   → content-type: audio/wav
   │
   └─ 패키지 업데이트
       → wav_file_path, wav_file_size, progress: 60
```

### 리샘플링 로직

```
입력: 24kHz PCM (Gemini TTS 출력)
출력: 44.1kHz PCM (표준 WAV)

선형 보간:
  sampleRateRatio = 24000 / 44100 = 0.544
  newLength = round(inputLength / ratio)
  각 출력 샘플 = 인접 두 입력 샘플의 가중 평균
```

### WAV 헤더 구조

```
Offset  Size  Field
0       4     "RIFF"
4       4     파일 크기 - 8
8       4     "WAVE"
12      4     "fmt "
16      4     16 (PCM fmt 크기)
20      2     1 (PCM 포맷)
22      2     1 (모노)
24      4     44100 (샘플레이트)
28      4     88200 (바이트레이트)
32      2     2 (블록 정렬)
34      2     16 (비트 깊이)
36      4     "data"
40      4     PCM 데이터 크기
44      ...   PCM 데이터
```

## Phase 2: 타임스탬프 생성 (scripts-package/generate-shadowing)

### 동작 흐름

```
패키지 + 스크립트 조회
   │
   ├─ Storage에서 WAV 다운로드
   │
   ├─ Whisper STT (word-level timestamps)
   │   │
   │   ├─ 모델: whisper-1
   │   ├─ 응답 형식: verbose_json
   │   ├─ 타임스탬프 단위: word
   │   ├─ 언어: en
   │   │
   │   └─ 출력: WhisperWord[]
   │       { word: "Hello", start: 0.0, end: 0.42 }
   │       { word: "my",    start: 0.45, end: 0.62 }
   │       ...
   │
   ├─ 문장 추출 (extractSentences)
   │   → paragraphs.paragraphs[].slots[].sentences[]
   │   → 전체 스크립트 기준 연속 인덱스 (1, 2, 3, ...)
   │
   ├─ 문장-단어 매칭 (matchSentencesToWords)
   │   → 슬라이딩 윈도우 + 유사도 계산
   │   → 각 문장의 시작/종료 시간 결정
   │
   ├─ 갭/오버랩 보정 (fillTimestampGaps)
   │
   ├─ JSON Storage 업로드
   │   → 경로: script-packages/json/{package_id}.json
   │
   └─ 패키지 완료 업데이트
       → json_file_path, timestamp_data, status: 'completed', progress: 100
```

### 문장-단어 매칭 알고리즘

```
각 문장에 대해:

1. 문장을 단어 배열로 분리 + 정규화 (소문자, 구두점 제거)
2. Whisper 단어 시퀀스에서 슬라이딩 윈도우 탐색
   - 윈도우 크기 = 문장 단어 수
   - 탐색 범위 = searchStart ~ searchStart + windowSize × 3
3. 각 윈도우 위치에서 유사도 점수 계산
   - 위치별 단어 비교 (완전 일치: 1.0, Levenshtein 근접: 0.7)
   - 전체 유사도 = 매칭 점수 합 / max(a.length, b.length)
4. 최고 점수 윈도우의 첫/끝 단어 시간으로 문장 시간 결정
5. 유사도 < 0.3이면 → 이전 문장 끝 + 추정 시간 (0.4초/단어)
6. searchStart를 윈도우 끝으로 이동 (순차 매칭 보장)
```

### 유사도 계산 상세

```
calculateSimilarity(sentenceWords, windowWords):

  각 위치 i에 대해:
    완전 일치 (a[i] === b[i]) → +1.0
    Levenshtein 거리 ≤ max(1, a[i].length/3) → +0.7
    불일치 → +0.0

  return 매칭 합 / max(a.length, b.length)
```

### 갭/오버랩 보정

```
인접 문장 사이:
  gap > 0.5초 → 중간점으로 양쪽 조정
  gap < 0 (오버랩) → 중간점으로 양쪽 조정
  각 문장의 duration 재계산
```

## 패키지 상태와 에러 처리

### 상태별 의미

| 상태 | 의미 | 사용 가능 |
|------|------|----------|
| `processing` | 생성 중 | ❌ |
| `completed` | Phase 1+2 성공 | ✅ 전체 기능 |
| `partial` | Phase 1 성공, Phase 2 실패 | ⚠️ 음성만 가능 |
| `failed` | Phase 1 실패 | ❌ |

### Phase 2 실패 시 partial 처리

```
Phase 2 실패 원인:
  - Whisper 음성 인식 실패
  - 음성에서 단어 미감지
  - 타임스탬프 저장 실패

partial 상태:
  → WAV 파일은 Storage에 저장됨
  → 쉐도잉 Step 1(전체 듣기)은 가능
  → 문장별 하이라이트/따라읽기는 불가
  → UI에서 "듣기만 가능" 표시
```

### 진행률 (progress) 단계

```
10 → 패키지 레코드 생성
20 → TTS API 호출 시작
40 → PCM → WAV 변환 완료
60 → Storage 업로드 완료 (Phase 1 완료)
65 → Phase 2 시작
70 → WAV 다운로드 완료
80 → Whisper STT 완료
90 → 문장 매칭 완료
100 → JSON 업로드 + 패키지 완료
```
