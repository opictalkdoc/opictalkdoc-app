# 02. 스크립트 생성

> 위저드 5단계 흐름과 Two-Pass EF 생성 로직 상세

---

## 위저드 5단계 (script-wizard.tsx, ~700줄)

### Step 1: 주제·질문 선택

```
카테고리 선택 (일반/롤플레이/어드밴스)
   │
   ├─ TopicPagination 컴포넌트
   │   → getTopicsByCategory() 조회 (빈도순)
   │   → 페이지네이션: 모바일 6개/PC 12개
   │
   ├─ QuestionSelector 컴포넌트
   │   → 선택된 주제의 질문 목록 표시
   │   → 기존 스크립트가 있으면 "보기" 버튼 노출
   │
   └─ 질문 선택 → Step 2로 이동
       → SelectedQuestion → QuestionOption 변환
       → queryClient 캐시에서 question_type_eng 추출
```

### Step 2: 경험 입력

```
사용자 입력 영역:
   │
   ├─ 목표 등급 선택 (NH~AL, 기본: IM2)
   │   → 프로필 targetGrade가 있으면 기본값으로 사용
   │
   ├─ 경험/스토리 입력 (textarea)
   │   → 한국어로 자유 입력
   │   → 비워두면 "Level 3 확장 정책" 적용
   │       (GPT가 해당 등급에 맞는 범용 콘텐츠 자동 생성)
   │
   └─ "스크립트 생성" 버튼 클릭 → handleGenerate()
```

### Step 2 → Step 3 전환 (handleGenerate)

```
createScript SA 호출
   │
   ├─ ① 크레딧 확인 + 차감 (consume_script_credit RPC)
   │
   ├─ ② scripts 테이블 UPSERT
   │   → onConflict: "user_id,question_id" (질문당 1개)
   │   → status: 'draft'
   │   → english_text, korean_translation, paragraphs = null (리셋)
   │   → EF가 채울 필드를 비워둠 (폴링 감지용)
   │
   ├─ ③ fire-and-forget EF 호출
   │   → POST /scripts/generate (or /correct)
   │   → { script_id }
   │   → .catch()로만 에러 처리 (await 하지 않음)
   │
   └─ ④ script_id 반환 → generatedScriptId 설정 → Step 3 이동
```

### Step 3: 생성 대기 (자동 전환)

```
refetchInterval 폴링 (3초 간격):
   │
   ├─ getScriptDetail(scriptId) 호출
   │   → english_text가 비어있으면 → 계속 폴링
   │   → english_text가 있으면 → 자동 Step 4 전환
   │
   └─ UI: 스피너 + "AI가 스크립트를 생성하고 있습니다" 메시지
       → OPIc 학습 팁 표시 (getOpicTips, staleTime: Infinity)
```

### Step 4: 결과 확인

```
스크립트 뷰어:
   │
   ├─ 보기 모드 4가지 (ScriptViewMode)
   │   ├─ "both"  — 영한 병기 (기본)
   │   ├─ "en"    — 영어만
   │   ├─ "ko"    — 한글만
   │   └─ "split" — 영어 블록 + 한글 블록 분리
   │
   ├─ 본문 (ScriptRenderer)
   │   → paragraphs 구조 기반 렌더링
   │   → Introduction / Body / Conclusion 섹션 분리
   │   → 슬롯별 문장 표시
   │
   ├─ 핵심 정리 (ScriptSummaryView)
   │   → 7가지 학습 콘텐츠 카드
   │   → 인터랙티브 하이라이트 (클릭 → 본문에서 위치 강조)
   │
   ├─ 수정 요청 (refine)
   │   → 수정 프롬프트 입력
   │   → 최대 3회, 크레딧 무료
   │   → Step 3 → Step 4 재전환 (동일 폴링 메커니즘)
   │
   └─ 확정 (confirm)
       → status: 'draft' → 'confirmed'
       → 패키지 생성 시작
```

### Step 5: 완료

```
패키지 생성 현황:
   │
   ├─ createPackage SA 호출 (await)
   │   → Phase 1: generate-package EF (TTS 생성)
   │   → Phase 2: generate-shadowing EF (타임스탬프)
   │
   ├─ 진행률 표시 (0% → 60% → 100%)
   │
   └─ 완료 → "쉐도잉 훈련 시작" CTA
```

## UPSERT 패턴

### 핵심: 질문당 1개 스크립트

```sql
UNIQUE (user_id, question_id)
```

같은 질문에 대해 다시 생성하면:
1. 기존 레코드를 **덮어씀** (UPDATE)
2. 기존 패키지는 **자동 삭제** (deleteScript or 재생성 시)
3. 크레딧은 **새로 차감** (UPSERT도 새 생성으로 취급)

### UPSERT 시 리셋되는 필드

```typescript
{
  english_text: null,         // 폴링 감지 트리거
  korean_translation: null,
  paragraphs: null,
  word_count: null,
  total_slots: null,
  key_expressions: null,
  generation_time: null,
  ai_model: null,
  refine_count: 0,
  status: 'draft',
}
```

## Two-Pass EF 아키텍처 (scripts/index.ts)

### 라우터 구조

```
POST /scripts/generate  → handleGenerate  (Service Role Key 인증)
POST /scripts/correct   → handleCorrect   (Service Role Key 인증)
POST /scripts/refine    → handleRefine    (Service Role Key 인증)
POST /scripts/evaluate  → handleEvaluate  (Anon Key 인증)
```

### Pass 1: 스크립트 생성 (GPT-4.1)

```
프롬프트 조립 (assemblePass1Prompt):
   │
   ├─ System Prompt 로드
   │   → ai_prompt_templates.template_id = 'script_system'
   │   → RCTF 프롬프트 구조 (Role/Context/Task/Format)
   │   → T/E/F 분류 규칙, 등급 준수, 구어체 품질 지침
   │
   ├─ 등급별 규격서 로드
   │   → script_specs.guide_id = '{question_type}_{target_level}'
   │   → 4개 섹션: level_constraints, slot_structure, example_output, eval_criteria
   │
   └─ User Prompt 조립 (6개 섹션)
       ① LEVEL GATE — 등급 제약 (절대 상위 등급 표현 사용 금지)
       ② STRUCTURE — question_type × target_level 슬롯 구조
       ③ EXAMPLE — 등급별 예시 출력
       ④ INPUT — 질문, 사용자 스토리
       ⑤ GENERATE/CORRECT — 생성/교정 지시
       ⑥ REFINE (수정 시만) — 기존 스크립트 + 수정 요청
```

### Pass 1 JSON 출력 스키마

```json
{
  "paragraphs": [
    {
      "type": "introduction | body | conclusion",
      "label": "도입 | 본론 | 마무리",
      "slots": [
        {
          "slot_index": 1,
          "slot_function": "주제 도입 | 세부 묘사 | ...",
          "text": "전체 슬롯 영어 텍스트",
          "translation_ko": "전체 슬롯 한글 번역",
          "sentences": [
            {
              "index": 1,
              "english": "문장 영어",
              "korean": "문장 한글"
            }
          ],
          "keywords": ["핵심", "키워드"]
        }
      ]
    }
  ],
  "full_text": {
    "english": "전체 영어 텍스트",
    "korean": "전체 한글 번역"
  },
  "word_count": 120
}
```

### Pass 2: 학습 분석 (GPT-4.1)

```
입력:
   ├─ system: ai_prompt_templates.template_id = 'script_analysis'
   ├─ user: target_level, question_type, question, full_text
   └─ temperature: 0.3 (낮은 창의성, 정확한 추출)

출력: 7가지 학습 콘텐츠
   ├─ structure_summary   — 뼈대 구조 [{tag, description}]
   ├─ key_sentences       — 핵심 문장 [{english, reason}]
   ├─ key_expressions     — 핵심 표현 [{en, ko, tip}]
   ├─ discourse_markers   — 담화 장치 [{en, ko, function, usage}]
   ├─ reusable_patterns   — 만능 패턴 [{template, description_ko, example}]
   ├─ similar_questions   — 유사 질문 [{question, reuse_hint}]
   └─ expansion_ideas     — 확장 아이디어 [string] (IM3+ 전용)
```

### Pass 2 실패 처리

```
Pass 2 실패 → EMPTY_LISTS 반환 (빈 배열 7개)
   → Pass 1 결과는 정상 저장
   → 학습 분석만 누락 (치명적이지 않음)
```

## 등급별 규격서 (script_specs, 60행)

### 구성

```
guide_id = {question_type}_{target_level}

10 question_types × 6 levels = 60행
   ├─ description, routine          → INT 계열
   ├─ comparison, past_childhood    → ADV 계열
   ├─ past_special, past_recent     → ADV 계열
   ├─ rp_11, rp_12                  → 롤플레이 계열
   └─ adv_14, adv_15                → 어드밴스 계열
```

### 4개 섹션 (JSONB)

| 섹션 | 내용 |
|------|------|
| `level_constraints` | 등급 제약 (허용/금지 표현, 문장 복잡도) |
| `slot_structure` | 슬롯 구조 (단락 수, 슬롯 역할, 문장 수) |
| `example_output` | 예시 JSON 출력 (GPT 참고용) |
| `eval_criteria` | 평가 기준 (자연스러움, 등급 준수, 과제 충족) |

## AI 프롬프트 (ai_prompt_templates)

### 공유 구조

```
script_system   — 스크립트 생성 시스템 프롬프트 (튜터링과 공유)
script_analysis — 학습 분석 시스템 프롬프트
evaluate_shadowing — 쉐도잉 평가 시스템 프롬프트
```

### RCTF 구조 (script_system)

```
Role:    OPIc 전문 스크립트 작성자
Context: 등급별 제약, T/E/F 분류, 구어체 규칙
Task:    주어진 입력으로 스크립트 생성/교정
Format:  JSON Schema (strict mode)

핵심 규칙:
   - T/E/F 분류: Template(주제 무관) / Expression(주제 종속) / Filler(담화 장치)
   - 등급 게이트: 상위 등급 표현 절대 사용 금지
   - 구어체: 축약형, 필러, 자연스러운 전환어 필수
   - 콘텐츠 확장: 입력이 없으면 Level 3(범용 콘텐츠) 자동 생성
```

## 수정 (refine) 로직

### 제한 사항

```
최대 3회 (refine_count <= 3)
크레딧 소모 없음
확정된 스크립트(confirmed) 수정 불가
```

### 동작 흐름

```
refineScript SA:
   │
   ├─ ① refine_count 확인 (≤ 3)
   ├─ ② status !== 'confirmed' 확인
   ├─ ③ refine_count + 1
   ├─ ④ english_text, korean_translation, paragraphs = null (리셋)
   │   → 폴링이 "아직 미완료"로 감지하도록
   └─ ⑤ fire-and-forget EF /scripts/refine
       → 기존 스크립트 + 수정 요청을 ⑥ REFINE 섹션으로 추가
```

### EF refine 프롬프트 구조

```
기존 프롬프트 (① ~ ⑤) + 추가:

⑥ REFINE (수정 요청)
   - 기존 스크립트 전문
   - 사용자 수정 요청
   - "등급 제약과 슬롯 구조는 반드시 유지" 지시
```

## 교정 (correct) 모드

```
createCorrectScript SA:
   │
   ├─ 사용자가 직접 작성한 영어 답변 입력
   │   → user_original_answer에 저장
   │   → source: 'correct'
   │
   └─ EF /scripts/correct 호출
       → 동일한 Two-Pass 구조
       → inputLabel: "Learner answer (English)"

```

## 폴링 메커니즘 (클라이언트)

```typescript
useQuery({
  queryKey: ["script-detail", scriptId],
  queryFn: () => getScriptDetail(scriptId),
  staleTime: 30 * 1000,
  refetchInterval: (query) => {
    // english_text가 비어있으면 3초마다 폴링
    if (data && !data.english_text) return 3000;
    return false;  // 완료되면 폴링 중지
  },
});
```

### 자동 전환 로직

```typescript
useEffect(() => {
  if (step === 3 && scriptDetail?.english_text) {
    setStep(4);  // 생성 완료 → 결과 확인
  }
}, [step, scriptDetail]);
```
