# 06. DB 스키마

> 스크립트+쉐도잉 모듈 6테이블 + 2 RPC + Storage

---

## 테이블 관계도

```
ai_prompt_templates (공유)
   │
   │  template_id로 EF에서 직접 조회
   │
script_specs (공유)
   │
   │  guide_id = '{answer_type}_{target_level}'
   │
scripts ──────────────── master_questions
   │  UNIQUE(user_id, question_id)
   │
   ├── script_packages
   │       │
   │       └── shadowing_sessions
   │               │
   │               └── shadowing_evaluations
   │
   └── (직접) shadowing_sessions
```

## 1. ai_prompt_templates (AI 프롬프트 — 스크립트+튜터링 공유)

### 용도

스크립트 생성/쉐도잉 평가/튜터링에서 공유하는 System Prompt + User Template 저장.
EF에서 `template_id`로 조회하여 GPT API 호출 시 사용.

### 컬럼

| 컬럼 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `id` | SERIAL PK | auto | 자동 증가 |
| `template_id` | TEXT UNIQUE | - | 조회 키: `script_system`, `script_analysis`, `evaluate_shadowing` |
| `prompt_name` | TEXT | - | 사람이 읽을 수 있는 이름 |
| `system_prompt` | TEXT NOT NULL | - | System Prompt (고정 원칙 — API 캐싱 대상) |
| `user_template` | TEXT | - | User Prompt 템플릿 (변수 치환용 골격) |
| `model` | TEXT | `gpt-4.1` | 사용 모델 |
| `temperature` | NUMERIC(2,1) | `0.8` | 생성 온도 |
| `max_tokens` | INTEGER | `4000` | 최대 토큰 |
| `response_format` | TEXT | `json_schema` | 응답 형식 |
| `is_active` | BOOLEAN | `true` | 활성 여부 |
| `created_at` | TIMESTAMPTZ | NOW() | 생성일 |
| `updated_at` | TIMESTAMPTZ | NOW() | 수정일 (트리거) |

### 시드 데이터 (2행)

| template_id | 용도 | temperature | response_format |
|-------------|------|-------------|-----------------|
| `script_system` | 스크립트 생성/교정 System Prompt (RCTF 구조) | 0.8 | json_schema |
| `evaluate_shadowing` | 쉐도잉 실전 평가 | 0.3 | json_object |

### RCTF System Prompt 구조 (`script_system`)

```
ROLE:    ACTFL OPIc script coach (10+ years)
CONTEXT: 4가지 핵심 원칙
         ├─ LEVEL HONESTY — 등급 초과 금지
         ├─ SPOKEN ENGLISH — 구어체, 축약형, 필러
         ├─ STORY PRESERVATION — 학습자 스토리 보존
         └─ SLOT FIDELITY — 슬롯 기능 충족
TASK:    T/E/F 분리 규칙 + 콘텐츠 확장 정책 3단계 + 한국어 번역 규칙
FORMAT:  자가 검증 5단계 체크리스트
```

### User Template 변수 (`script_system`)

```
{target_level}     → 목표 등급
{level_constraints} → script_specs.level_constraints
{answer_type}      → 답변 유형
{slot_structure}   → script_specs.slot_structure
{example_output}   → script_specs.example_output
{question_english} → 영어 질문
{question_korean}  → 한국어 질문
{user_story}       → 사용자 스토리 입력
{total_slots}      → 슬롯 수
```

### RLS

| 정책 | 대상 | 규칙 |
|------|------|------|
| `ai_prompt_templates_read` | SELECT | 전체 허용 (읽기 전용) |

> 수정은 서비스 키(EF, 마이그레이션)로만 가능

## 2. script_specs (등급별 규격서 — 60행)

### 용도

10가지 question_type × 6가지 target_level = **60행**.
스크립트 생성 시 GPT 프롬프트의 3개 섹션(LEVEL GATE, STRUCTURE, EXAMPLE)에 사용.

### 컬럼

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | SERIAL PK | 자동 증가 |
| `guide_id` | TEXT UNIQUE | 조합 키: `{answer_type}_{target_level}` (예: `description_IM2`) |
| `answer_type` | TEXT | 10가지 답변 유형 |
| `target_level` | TEXT | IL, IM1, IM2, IM3, IH, AL |
| `total_slots` | SMALLINT | 등급별 슬롯 개수 |
| `level_constraints` | TEXT | ① LEVEL GATE — 등급 제약 (허용/금지 표현, 문장 복잡도) |
| `slot_structure` | TEXT | ② STRUCTURE — 분량 기준, 담화 패턴, 슬롯 구조 |
| `example_output` | JSONB | ③ EXAMPLE — 완성된 JSON 출력 예시 |
| `eval_criteria` | TEXT | 자가 검증 평가 기준 |
| `created_at` | TIMESTAMPTZ | 생성일 |
| `updated_at` | TIMESTAMPTZ | 수정일 (트리거) |

### 10가지 answer_type

```
INT 계열:  description, routine
ADV 계열:  comparison, past_childhood, past_special, past_recent
롤플레이:  rp_11, rp_12
어드밴스:  adv_14, adv_15
```

### 4개 섹션 역할

```
① level_constraints → User Prompt ① LEVEL GATE
   "IM2는 관계대명사 사용 금지, 가정법 사용 금지..."

② slot_structure → User Prompt ② STRUCTURE
   "Introduction(1슬롯) + Body(2~3슬롯) + Conclusion(1슬롯)"

③ example_output → User Prompt ③ EXAMPLE
   완전한 JSON 예시 (paragraphs, full_text, word_count)

④ eval_criteria → System Prompt 자가 검증
   "자연스러움, 등급 준수, 과제 충족"
```

### 인덱스

| 인덱스 | 컬럼 |
|--------|------|
| `idx_script_specs_answer_type` | answer_type |
| `idx_script_specs_target_level` | target_level |

### RLS

| 정책 | 대상 | 규칙 |
|------|------|------|
| `script_specs_read` | SELECT | 전체 허용 (읽기 전용) |

## 3. scripts (스크립트 마스터)

### 용도

사용자가 생성한 스크립트의 모든 데이터를 저장하는 중심 테이블.
`generate`(AI 생성)과 `correct`(교정) 두 모드를 `source` 컬럼으로 구분.

### 핵심 제약

```sql
UNIQUE(user_id, question_id)  -- 사용자당 질문당 1개 (UPSERT)
```

→ 같은 질문에 대해 다시 생성하면 기존 레코드를 **덮어씀**

### 컬럼

| 컬럼 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `id` | UUID PK | gen_random_uuid() | 스크립트 ID |
| `user_id` | UUID FK → auth.users | - | 작성자 |
| `question_id` | TEXT FK → master_questions | - | 질문 ID |
| `source` | TEXT | `generate` | 생성 모드: `generate` \| `correct` |
| `title` | TEXT | - | 제목 (미사용) |
| `english_text` | TEXT NOT NULL | - | 전체 영어 텍스트 (Pass 1 full_text.english) |
| `korean_translation` | TEXT | - | 전체 한국어 번역 (Pass 1 full_text.korean) |
| `paragraphs` | JSONB | - | 4계층 구조 JSON (Pass 1 + Pass 2 병합) |
| `total_slots` | SMALLINT | - | 슬롯 수 |
| `category` | TEXT | - | topic_category 값 |
| `topic` | TEXT | - | 주제명 |
| `question_korean` | TEXT | - | 한국어 질문 |
| `question_english` | TEXT | - | 영어 질문 |
| `user_story` | TEXT | - | 생성 시 한국어 스토리 입력 |
| `user_original_answer` | TEXT | - | 교정 시 학습자 원본 영어 답변 |
| `target_level` | TEXT | - | 목표 OPIc 등급 (NH~AL) |
| `answer_type` | TEXT | - | 10가지 question_type |
| `ai_model` | TEXT | `gpt-4.1` | 사용 AI 모델 |
| `word_count` | INTEGER | - | 단어 수 |
| `generation_time` | INTEGER | - | 생성 소요 시간 (초) |
| `key_expressions` | JSONB | `[]` | 핵심 표현 string[] (Pass 2 추출) |
| `highlighted_script` | TEXT | - | 하이라이트 스크립트 (미사용) |
| `status` | TEXT | `draft` | 상태: `draft` \| `confirmed` |
| `refine_count` | SMALLINT | `0` | 수정 횟수 (최대 3) |
| `created_at` | TIMESTAMPTZ | NOW() | 생성일 |
| `updated_at` | TIMESTAMPTZ | NOW() | 수정일 (트리거) |

| `json_file_path` | TEXT | - | Storage URL (타임스탬프 JSON) |

## RPC 함수

### consume_script_credit(p_user_id UUID) → BOOLEAN

```
목적: 스크립트 크레딧 1개 차감
호출: createScript SA, handleEvaluate EF

차감 순서:
  1. plan_script_credits (플랜 구매 → 만료됨)
  2. script_credits (횟수권 → 영구)

동작:
  SELECT ... FOR UPDATE (행 잠금)
  → plan_script_credits > 0 → plan 차감 → RETURN true
  → script_credits > 0 → 횟수권 차감 → RETURN true
  → 둘 다 0 → RETURN false

보안: SECURITY DEFINER (호출자 권한 무관)
```
