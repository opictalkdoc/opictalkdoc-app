## 세션 초기화 흐름

```
/tutoring/training?prescription_id={id} 페이지 진입
   │
   ├─ ① createTrainingSession(prescription_id) SA 호출
   │   → 기존 미완료 훈련이 있으면 해당 ID 반환 (중복 방지)
   │   → 없으면 새 레코드 INSERT + prescription status → in_progress
   │   → training_session_id 반환
   │
   ├─ ② prescription 정보 조회 (Supabase 직접)
   │   → question_type, weakness_tags, level_params, timerConfig 추출
   │
   └─ ③ EF session-brief 호출 → Screen 0 데이터 설정
       → setBrief(result.brief)
       → setLoading(false) → Screen 0 표시
```

## Screen 1: 워밍업 (4분)

### EF 호출

```
action: "generate-warmup"
params: { training_session_id, question_type, target_level, text_policy }
인증: ❌ (stateless)

GPT: gpt-4.1-mini, temp 0.6, max 1000 tokens
입력: 랜덤 질문 1개 + question_type + target_level + levelGuide
```

### 질문 선택 로직

```sql
SELECT id, question_english, question_korean, question_type_eng, topic
FROM questions
WHERE question_type_eng = $1 AND category = '일반'
LIMIT 10
→ 10개 중 랜덤 1개 선택
```

### 등급별 텍스트 정책

```typescript
const textPolicy =
  ["IL", "NL", "NM", "NH", "IM1"].includes(level) ? "full" :
  ["IM2", "IM3", "IH"].includes(level) ? "keywords" :
  "off";  // AL
```

| 정책 | 표시 방식 |
|------|----------|
| `full` | 전체 텍스트 + 한국어 번역 항상 표시 |
| `keywords` | 키워드만 볼드, 나머지 반투명 (토글 가능) |
| `off` | 텍스트 완전 숨김, 음성만 |

### 응답 구조

```json
{
  "warmup": {
    "question": { "id": "q_123", "english": "...", "korean": "..." },
    "model_answer": "Well, I usually go to the gym...",
    "sentences": [
      {
        "english": "Well, I usually go to the gym three times a week.",
        "korean": "음, 저는 보통 일주일에 세 번 헬스장에 가요.",
        "keywords": ["gym", "three times", "week"]
      }
    ],
    "rhythm_tips": ["문장 끝을 내리는 억양으로", "..."],
    "key_expressions": ["work out", "three times a week"]
  }
}
```

### 쉐도잉 3라운드 진행

```
Round 0: 모범 답변 읽기
   → 전체 텍스트 표시 + 핵심 표현 + 리듬 팁
   → [따라 말하기 시작] 버튼

Round 1: 첫 번째 쉐도잉
   → 문장별 진행 (currentSentence 추적)
   → 프로그레스 바: n / total
   → 텍스트 정책 적용
   → [다음 문장] 버튼

Round 2: 두 번째 쉐도잉
   → 동일 구조, 2회차 표시
   → 마지막 문장 완료 → [EPP 패턴으로] 버튼

→ Screen 2로 전환
```

## Screen 2: EPP 패턴 고정 (5분)

### 프로토콜: EPP (Essential Pattern Pack)

**목적**: 유형별 필수 답변 구조를 반사적으로 출력하게 만들기

### EF 호출

```
action: "generate-epp"
params: { training_session_id, question_type, target_level, weakness_tags }
인증: ❌ (stateless)

GPT: gpt-4.1-mini, temp 0.6, max 800 tokens
입력: 랜덤 질문 1개 + question_type + target_level + weakness_tags
```

### 응답 구조

```json
{
  "epp": {
    "question": { "id": "q_234", "english": "...", "korean": "..." },
    "epp_cards": [
      {
        "template": "To be honest, [주제] is something I [동사] quite often.",
        "korean_hint": "솔직히 말하면, [주제]는 제가 자주 [동사]하는 거예요.",
        "slots": ["주제", "동사"]
      },
      {
        "template": "The main reason is that [이유].",
        "korean_hint": "가장 큰 이유는 [이유]이기 때문이에요.",
        "slots": ["이유"]
      }
    ],
    "required_connectors": ["First of all", "For example"],
    "forbidden_patterns": ["I think so", "Something like that"],
    "tip": "슬롯을 자신의 경험으로 채워서 3번 반복하세요."
  }
}
```

### 사용자 상호작용

```
EPP 카드 표시 (뼈대 + 슬롯 + 한국어 힌트)
   │
   ├─ 필수 연결어 표시 (파란 뱃지)
   ├─ 금지 패턴 표시 (빨간 뱃지)
   │
   ├─ 텍스트 입력 (textarea)
   │   → EPP 뼈대를 자기 내용으로 채워 작성
   │
   └─ 제출 (3회 반복)
       → saveAttempt(screen: 2, protocol: "epp", attempt: N, passed: true)
       → attemptCount < 3이면 textarea 초기화 → 다시 작성
       → attemptCount >= 3이면 [변형 훈련으로] 버튼 활성화
```

### 유형별 EPP 예시

| question_type | 필수 뼈대 |
|---------------|----------|
| description | 전체 소개 → 개인 연결 → 구체적 예시 → 마무리 느낌 |
| routine | 빈도/시간 → 구체 행동 → 이유 → 좋아하는 점 |
| past_experience_* | 시점 설정 → 에피소드 → 문제/전환 → 감정/교훈 |
| comparison | A 설명 → B 설명 → 차이점 → 내 선택 + 이유 |
| rp_12 | 사과 → 이유 → 옵션 2개 → 상대 선택 유도 |
| adv_15 | 입장 → 이유 1 + 예시 → 이유 2 + 설명 → 결론 |

## Screen 3: 변형 훈련 (7분)

### 두 가지 프로토콜

```
Phase 1: Forced Variation (강제 변주) — 4~5분
Phase 2: Oral Transformation (구두 변환) — 2~3분
```

### Phase 1: Forced Variation

**목적**: 외운 답 깨기 + 변형 질문에서도 동일 품질 유지

#### EF 호출

```
action: "generate-variation"
params: { training_session_id, question_type, target_level, weakness_tags, variation_changes }
인증: ❌ (stateless)

GPT: gpt-4.1-mini, temp 0.7, max 1200 tokens
질문 풀: 동일 question_type에서 20개 로드 → 3개 랜덤 선택
카드 수: Math.min(3, variation_changes + 1)
```

#### 변형 유형 4가지

```
time_change:       시간 변화 (now vs used to, past vs present)
condition_change:  조건 변화 (alone vs with friends, weekday vs weekend)
comparison:        A vs B 비교
reason_reversal:   이유 반전 (좋아하지만 단점 1개)
```

#### 등급별 변경 요소 수

```
IL→IM:  1개 변경 요소 → 카드 2장
IM→IH:  2개 변경 요소 → 카드 3장
IH→AL:  3개 변경 요소 → 카드 3장 (즉흥 랜덤)
```

#### 응답 구조

```json
{
  "variation": {
    "variation_cards": [
      {
        "question": { "id": "q_345", "english": "...", "korean": "..." },
        "variation_type": "time_change",
        "instruction": "과거형으로 바꿔서 말해보세요",
        "forced_mission": {
          "connector": "Back then",
          "detail_prompt": "구체적인 장소를 추가하세요"
        },
        "time_limit_seconds": 40,
        "example_starter": "When I was in college, I used to..."
      }
    ]
  }
}
```

#### UI

```
카드 선택 버튼 (1, 2, 3 — 완료 시 초록)
   │
   현재 카드:
   ├─ 질문 표시
   ├─ 변형 지시 (instruction)
   ├─ 필수 미션: 연결어 + 디테일 요구
   ├─ 시작 예시 문장
   │
   └─ 텍스트 입력 → 제출
       → saveAttempt(screen: 3, protocol: "variation")
       → 카드 완료 표시 → 다음 카드
       → 모든 카드 완료 → Phase 2 전환 가능
```

### Phase 2: Oral Transformation

**목적**: 문법을 "말로 자동 출력"으로 체화

#### EF 호출

```
action: "generate-transformation"
params: { training_session_id, question_type, target_level }
인증: ❌ (stateless)

GPT: gpt-4.1-mini, temp 0.7, max 1200 tokens
카드 수: AL → 10장, IH → 8장, 나머지 → 6장
```

#### 변환 유형 5가지

```
tense_change:  시제 변환 (현재→과거, 과거→현재완료)
add_reason:    이유 추가 (statement → statement + because...)
comparison:    비교 추가 (statement → compared to X, ...)
subjunctive:   가정법 (statement → If I had/could...)
expansion:     확장 (짧은 문장 → 디테일/예시 추가)
```

#### 등급별 변환 유형

```
IL→IM:  현재↔과거, because 1개 붙이기
IM→IH:  비교/이유절/간단 관계절
IH→AL:  완곡표현/조건부 주장/부분동의
```

#### 응답 구조

```json
{
  "transformation": {
    "transformation_cards": [
      {
        "original": "I go to the gym on weekdays.",
        "transform_type": "tense_change",
        "instruction": "과거형으로 바꿔보세요",
        "expected_pattern": "I went to the gym... / I used to go...",
        "time_limit_seconds": 4
      }
    ]
  }
}
```

#### UI

```
프로그레스 바: n / total 변환 완료
   │
   현재 카드:
   ├─ 원본 문장 (original)
   ├─ 변환 지시 (instruction)
   ├─ [힌트 보기] 토글 → expected_pattern 표시
   │
   └─ 텍스트 입력 → 제출
       → saveAttempt(screen: 3, protocol: "transformation")
       → 다음 카드
       → 모든 카드 완료 → Screen 4로 전환
```

### 탭 UI

```
[강제 변주 (2/3)] [구두 변환 (0/6)]
   │                │
   Phase 1          Phase 2
```

## Screen 4: 타임드 실전 (7분)

### 프로토콜: 15-60-15

**목적**: 시간 압박 하 실전 감각 + 구조적 답변 완주

### 등급별 타이머

```
IL→IM:  15초 준비 → 30초 본론 → 15초 마무리
IM→IH:  15초 준비 → 60초 본론 → 15초 마무리
IH→AL:  15초 준비 → 75초 본론 → 15초 마무리
```

### 질문 로딩

```sql
SELECT id, question_english, question_korean, question_type_eng, topic
FROM questions
WHERE question_type_eng = $1 AND category = '일반'
LIMIT 10
→ 랜덤 1개 선택
```

### 타이머 Phase 전환

```
ready (대기)
  │ "시작" 버튼
  ▼
prep (준비 — 15초 카운트다운)
  │ 타이머 종료 또는 "다음 단계로" 버튼
  ▼
speak (본론 — 30/60/75초 카운트다운)
  │ 텍스트 입력 가능 (autoFocus)
  │ 경과 시간 기록 (speakDurationRef)
  │ 타이머 종료 또는 "다음 단계로" 버튼
  ▼
wrap (마무리 — 15초 카운트다운)
  │ 추가 입력 가능
  │ 타이머 종료
  ▼
evaluating (평가 중)
  │ EF evaluate-timed 호출
  ▼
result (결과 표시)
```

### EF 호출 (평가)

```
action: "evaluate-timed"
params: {
  training_session_id,
  question_id,
  question_english,
  question_type,
  user_answer: userText,
  audio_duration_seconds: speakDurationRef.current,
  target_level
}
인증: ❌ (stateless)

GPT: gpt-4.1 (정밀 모델), temp 0.3, max 1200 tokens
```

### 평가 응답 구조

```json
{
  "evaluation": {
    "block_checklist": {
      "opening":    { "met": true,  "comment": "자연스러운 도입" },
      "background": { "met": true,  "comment": "배경 설명 충분" },
      "detail":     { "met": false, "comment": "구체적 예시 부족" },
      "emotion":    { "met": false, "comment": "감정 표현 없음" },
      "closing":    { "met": true,  "comment": "마무리 적절" }
    },
    "structure_score": 7,
    "content_score": 6,
    "time_management": "적절",
    "grammar_issues": [
      {
        "original": "I go there yesterday",
        "corrected": "I went there yesterday",
        "rule": "과거 행동은 과거시제 사용"
      }
    ],
    "strengths": ["연결어 활용이 좋습니다"],
    "improvements": ["구체적 예시를 추가하세요"],
    "overall_comment": "구조는 잘 잡혀있으나 디테일이 부족합니다.",
    "passed": true
  }
}
```

### 결과 UI

```
블록 체크리스트: ✓ opening / ✓ background / ✗ detail / ✗ emotion / ✓ closing
점수 카드: 구조 7/10 | 내용 6/10 | 시간관리 적절
총평: "구조는 잘 잡혀있으나..."
문법 교정: "I go there yesterday" → "I went there yesterday" (규칙: 과거시제)

→ saveAttempt(screen: 4, protocol: "timed", evaluation, passed)
→ Screen 5로 전환
```

## Screen 5: Self-repair (4분)

### 프로토콜: Self-repair

**목적**: 실수해도 자연스럽게 고치고 이어가는 능력

### 스킵 조건

```
Screen 4의 grammar_issues가 0개
   → "교정할 문법 오류가 없습니다. 잘하셨어요!" 표시
   → [다음으로] 버튼 → Screen 6으로
```

### Self-repair 문구 선택

```
6가지 프리셋:
   "I mean..."
   "Let me rephrase that."
   "What I'm trying to say is..."
   "Actually, I should say..."
   "Well, not exactly..."
   "Sorry, what I meant was..."

→ 토글 버튼으로 1개 선택
```

### 사용자 흐름

```
① 문법 오류 하이라이트 표시
   → 원본(빨간 취소선) + 수정(초록) + 규칙 설명

② Self-repair 문구 선택

③ 수정된 답변 작성 (textarea)
   → placeholder: "{선택한 문구} ..."

④ 제출
   → EF evaluate-repair 호출
```

### EF 호출

```
action: "evaluate-repair"
params: {
  training_session_id,
  original_text: grammar_issues.map(g => g.original).join(". "),
  errors: grammar_issues.map(g => ({ text: g.original, type: "grammar" })),
  repair_text: repairText,
  repair_phrase: selectedPhrase
}
인증: ❌ (stateless)

GPT: gpt-4.1-mini, temp 0.3, max 500 tokens
```

### 평가 응답 구조

```json
{
  "evaluation": {
    "repair_quality": "good",
    "corrections_addressed": 2,
    "corrections_fixed": 1,
    "naturalness_score": 7,
    "before_after_comparison": "시제 교정은 잘 했으나 연결어가 어색합니다",
    "tip": "I mean 다음에 잠시 멈추면 더 자연스럽습니다",
    "passed": true
  }
}
```

### 결과 UI

```
repair_quality에 따른 표시:
   "excellent" → ✅ 훌륭합니다!
   "good"      → ✅ 잘했어요!
   기타        → ⚠️ 조금 더 연습이 필요해요

비교 설명: "시제 교정은 잘 했으나..."
팁: "I mean 다음에 잠시 멈추면..."

→ saveAttempt(screen: 5, protocol: "self_repair",
     repair_before, repair_after, repair_type: "grammar", passed)
→ Screen 6으로 전환
```

### 등급별 Self-repair 요구

```
IL→IM:  0회 (의무 아님, 있으면 추가 연습)
IM→IH:  최소 1회 (게이트)
IH→AL:  최소 2회 + 자연스러운 리프레이즈
```
