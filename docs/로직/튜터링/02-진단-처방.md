# 02. 진단 · 처방

> 모의고사 결과 → 진단 시각화 → 처방 엔진 자동 생성

---

## 진단 탭 (DiagnosisTab)

### 데이터 소스

```
getDiagnosis() SA — 4개 병렬 쿼리:
   │
   ├─ ① tutoring_sessions (최근 10개)
   │   → 활성 세션 존재 여부 확인
   │
   ├─ ② mock_test_reports (완료된 리포트 최근 10개)
   │   → final_level, total_score, score_f/a/c/t
   │   → coaching_report (JSONB)
   │   → tutoring_prescription (JSONB)
   │
   ├─ ③ tutoring_skill_history (최근 50개)
   │   → 성장 추이 데이터
   │
   └─ ④ tutoring_prescriptions (활성 세션의 처방 목록)
       → 활성 세션이 있을 때만 조회
```

### 표시 항목 4가지

#### 1. 등급 + 스냅샷

```
확정 등급 뱃지 (final_level)
├─ 총점 (total_score)
├─ 한줄 헤드라인 (coaching_report.snapshot.headline)
└─ 진단 태그 (coaching_report.snapshot.diagnosis_tags)
```

#### 2. FACT 4영역 프로그레스 바

```
Fluency    (score_f) ─────────── ■■■■■■■□□□ 72
Accuracy   (score_a) ─────────── ■■■■■□□□□□ 55
Content    (score_c) ─────────── ■■■■■■□□□□ 65
Task       (score_t) ─────────── ■■■■■■■■□□ 80
```

- Supabase NUMERIC → string 변환 처리: `Number(rawScore)`
- 각 영역별 해석 텍스트 표시

#### 3. 약점 Top 3 카드

```
coaching_report.top3_priorities 배열:
   [
     { rank: 1, area: "시제 일관성", why: "과거경험에서...", where: "past_special" },
     { rank: 2, area: "디테일 확장", why: "구체적 예시...", where: "description" },
     { rank: 3, area: "시간 관리", why: "답변이 30초...", where: "routine" }
   ]
```

#### 4. 유형별 진단 그리드

```
coaching_report.question_type_map 배열:
   → 2~3열 그리드로 표시
   → 상태 뱃지 색상:
     strong → 초록 (양호)
     stable → 파랑 (안정)
     weak   → 주황 (약함)
     기타   → 빨강 (매우약함)
```

## 처방 탭 (PrescriptionTab)

### 두 가지 뷰

```
활성 세션 없음 → 처방 미리보기 + "처방 생성" CTA
활성 세션 있음 → ActivePrescriptionView (진행 현황)
```

### 처방 미리보기 (활성 세션 없음)

```
latestReport.tutoring_prescription에서 추출:
   │
   ├─ priority_weaknesses 카드
   │   → rank, area, drill_tag 표시
   │
   ├─ must_fix_for_next_grade 태그
   │   → 빨간 뱃지로 표시
   │
   └─ "처방 생성하기" 버튼
       → startTutoringSession(mock_test_session_id) 호출
       → useMutation + invalidate ["tutoring-diagnosis"]
```

### ActivePrescriptionView (활성 세션 있음)

```
진행률 바: completed_prescriptions / total_prescriptions

처방 카드 목록 (priority 순 정렬):
   │
   ├─ PrescriptionCard #1 (높음)
   │   ├─ 상태 아이콘: ✅ 완료 / 🔄 진행 중 / ⏳ 대기
   │   ├─ 우선순위 뱃지: 높음(빨강) / 권장(주황) / 추가(초록)
   │   ├─ question_type 한국어명
   │   ├─ weakness_tags 태그 목록
   │   └─ "훈련 시작" 버튼 → /tutoring/training?prescription_id={id}
   │
   ├─ PrescriptionCard #2 (권장)
   └─ ...
```

## 처방 엔진 (buildPrescriptions)

### 위치

```
lib/actions/tutoring.ts 내부 함수 (SA 내에서만 호출)
```

### 입력

```typescript
{
  prescription: {                          // mock_test_reports.tutoring_prescription
    priority_weaknesses: [
      { rank: 1, area: "시제 일관성", drill_tag: "tense_consistency" },
      { rank: 2, area: "디테일 확장", drill_tag: "detail_expansion" }
    ],
    error_drill_tags: ["pause_control", "filler_reduction"],
    weak_types: ["past_special", "comparison"],
    training_order: ["past_special", "routine", "comparison"],
    must_fix_for_next_grade: ["tense_consistency", "detail_expansion"]
  },
  coachingReport: { ... },
  currentLevel: "IM2",
  targetLevel: "IH",
  levelParams: { ... }                    // LEVEL_PARAMS[getLevelRange()]
}
```

### 알고리즘

```
Priority 1 (높음): priority_weaknesses 상위 2개
   → drill_tag → DRILL_TAG_TO_QUESTION_TYPE 매핑
   → usedTypes Set에 추가 (중복 방지)

Priority 2 (권장): error_drill_tags 상위 2개
   → drill_tag → question_type 매핑
   → 이미 usedTypes에 있으면 건너뜀

Priority 3 (추가): weak_types 상위 2개
   → question_type 직접 사용 (또는 drill_tag 매핑)
   → 이미 usedTypes에 있으면 건너뜀

결과: 1~6개 PrescriptionItem 배열
```

### DRILL_TAG_TO_QUESTION_TYPE 매핑

```typescript
{
  description_detail:    "description",
  routine_sequence:      "routine",
  comparison_frame:      "comparison",
  past_narrative:        "past_special",
  tense_consistency:     "past_special",
  roleplay_questions:    "rp_12",
  roleplay_recovery:     "rp_12",
  opinion_support:       "adv_15",
  social_perspective:    "adv_15",
  detail_expansion:      "description",
  pause_control:         "past_special",
  filler_reduction:      "routine",
  sentence_completion:   "routine",
  opening_entry:         "description",
  closing_wrap:          "description",
  transition_words:      "routine",
  vocabulary_variety:    "description",
  polite_expression:     "rp_12",

  // fallback: "description"
}
```

### 폴백 규칙

```
매핑 결과가 0개 → 기본 처방 1개 자동 삽입:
   {
     priority: 1,
     question_type: "description",
     weakness_tags: ["general"],
     source: "top3_priorities",
     source_data: {}
   }
```

## startTutoringSession SA 상세 흐름

```
startTutoringSession({ mock_test_session_id })
   │
   ├─ ① 인증 확인 (requireUser)
   │
   ├─ ② mock_test_reports 조회
   │   → session_id = mock_test_session_id
   │   → user_id = userId
   │   → report_status = 'completed'
   │   → tutoring_prescription 존재 확인
   │
   ├─ ③ 기존 세션 확인 (멱등성)
   │   → tutoring_sessions에서 같은 mock_test_session_id
   │   → 있으면 기존 세션 + 처방 반환 (새로 생성하지 않음)
   │
   ├─ ④ 등급 범위 결정
   │   → getLevelRange(currentLevel, targetLevel) → IL_IM | IM_IH | IH_AL
   │   → LEVEL_PARAMS[range] 조회
   │
   ├─ ⑤ 처방 엔진 실행
   │   → buildPrescriptions(...)
   │   → 1~6개 PrescriptionItem 생성
   │
   ├─ ⑥ tutoring_sessions INSERT
   │   → status: 'active'
   │   → total_prescriptions: items.length
   │
   ├─ ⑦ tutoring_prescriptions BULK INSERT
   │   → 각 item에 level_params 복사
   │   → status: 'pending'
   │
   └─ ⑧ 결과 반환
       → { session, prescriptions }
```

## 등급 범위 결정 (getLevelRange)

```typescript
const LEVEL_ORDER = ["NL","NM","NH","IL","IM1","IM2","IM3","IH","AL"];

function getLevelRange(current: string, target: string): TutoringLevelRange {
  const idx = LEVEL_ORDER.indexOf(current);
  if (idx >= 7) return "IH_AL";    // IH 이상
  if (idx >= 4) return "IM_IH";    // IM1 이상
  return "IL_IM";                   // 나머지
}
```

## 데이터 흐름 요약

```
mock_test_reports
├── coaching_report (JSONB)
│   ├── snapshot.headline → 진단 탭 헤드라인
│   ├── snapshot.diagnosis_tags → 진단 탭 태그
│   ├── top3_priorities → 진단 탭 약점 카드 + 처방 Priority 1
│   └── question_type_map → 진단 탭 유형 그리드
│
└── tutoring_prescription (JSONB)
    ├── priority_weaknesses → 처방 Priority 1
    ├── error_drill_tags → 처방 Priority 2
    ├── weak_types → 처방 Priority 3
    ├── training_order → (참조용)
    └── must_fix_for_next_grade → 처방 탭 빨간 뱃지
```
