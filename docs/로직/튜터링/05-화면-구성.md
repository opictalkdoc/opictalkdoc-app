# 05. 화면 구성

> 3탭 허브 + 7Screen 몰입형 훈련 세션 UI 상세

---

## 페이지 구조

```
/tutoring (dashboard 레이아웃 — Navbar + Footer)
   └─ TutoringContent.tsx (3탭 허브)
       ├─ 진단 탭 (DiagnosisTab)
       ├─ 처방 탭 (PrescriptionTab)
       └─ 훈련 탭 (TrainingTab)

/tutoring/training?prescription_id={id} (immersive 레이아웃 — 풀스크린)
   └─ TrainingSession.tsx (7 Screen 몰입형)
       ├─ Screen 0: 세션 브리프
       ├─ Screen 1: 워밍업
       ├─ Screen 2: EPP 패턴
       ├─ Screen 3: 변형 훈련
       ├─ Screen 4: 타임드 실전
       ├─ Screen 5: Self-repair
       └─ Screen 6: 리캡
```

### PrescriptionCard 구성요소

```
├─ 상태 아이콘: ✅ completed / 🔄 in_progress / ⏳ pending
├─ question_type 한국어명 (QT_KO 매핑)
├─ 우선순위 뱃지
│   PRIORITY_BADGE = { 1: "높음"(빨강), 2: "권장"(주황), 3: "추가"(초록) }
├─ weakness_tags (회색 알약 뱃지)
├─ training_count / 3 진행 표시
└─ "훈련 시작" 링크 (status !== "completed"일 때만)
   → /tutoring/training?prescription_id={id}
```

## 훈련 탭 (TrainingTab)

### 조건부 렌더링

```
활성 세션 없음
   → "처방 탭에서 처방을 먼저 생성해주세요" Empty State

활성 세션 있음 + 미완료 처방 존재
   → 정보 박스 + 다음 처방 정보 + "훈련 시작하기" 링크

모든 처방 완료
   → ✅ "모든 처방을 완료했습니다!" 성공 상태
```

### 정보 박스

```
ℹ️ 집중 훈련 모드
"처방에 따라 EPP 패턴 고정, 타임드 실전, Self-repair를 순서대로 수행합니다.
한 세션 약 20~30분 소요됩니다."
```

## 컴포넌트 의존성 맵

```
TutoringContent.tsx
├─ getDiagnosis() SA
├─ startTutoringSession() SA (mutation)
├─ getPrescriptions() SA
├─ useQuery (["tutoring-diagnosis"])
├─ useQuery (["tutoring-prescriptions", sessionId])
└─ useQuery (["tutoring-training-prescriptions", sessionId])

TrainingSession.tsx
├─ createTrainingSession() SA
├─ saveAttempt() SA (매 Screen)
├─ completeTrainingSession() SA
├─ callTutoringEF() (8 handlers)
│   ├─ session-brief → brief 상태
│   ├─ generate-warmup → warmupData 상태
│   ├─ generate-epp → eppData 상태
│   ├─ generate-variation → variationData 상태
│   ├─ generate-transformation → transformationData 상태
│   ├─ evaluate-timed → timedEval 상태
│   ├─ evaluate-repair → repairResult 상태
│   └─ complete-session → recap 상태
└─ Supabase 직접 쿼리 (prescription 정보)
```

## 비동기 데이터 로딩 패턴

```
각 Screen은 진입 시 EF 호출하여 데이터 로드 (사전 로드 아님):

Screen 0 → 초기화 시 session-brief 호출
Screen 1 → nextScreen() 실행 시 generate-warmup 호출
Screen 2 → nextScreen() 실행 시 generate-epp 호출
Screen 3 → nextScreen() 실행 시 generate-variation 호출
         → Phase 2 전환 시 generate-transformation 호출
Screen 4 → nextScreen() 실행 시 질문 로딩 (Supabase 직접)
         → 답변 완료 시 evaluate-timed 호출
Screen 5 → timedEval 결과에서 grammar_issues 직접 사용
         → 답변 완료 시 evaluate-repair 호출
Screen 6 → 진입 즉시 complete-session 호출 + SA 호출
```

## 주요 구현 노트

1. **텍스트 입력 MVP**: 모든 Screen에서 음성 녹음 대신 텍스트 입력 사용 (향후 녹음 통합)
2. **모바일 우선**: immersive 레이아웃(`h-dvh`) + `h-0 flex-grow` 스크롤 패턴
3. **쿼리 무효화**: 훈련 종료 시 `["tutoring-diagnosis"]` 무효화 → 부모 탭 갱신
4. **에러 복구**: Screen 6에서 EF 실패해도 SA(세션 완료 처리)는 별도 try/catch
5. **등급별 커스터마이즈**: 텍스트 정책(워밍업), Variation 수(변형), 타이머(실전) 모두 level_params 기반
