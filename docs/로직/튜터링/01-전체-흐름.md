# 01. 전체 흐름

> 모의고사 진단 → 처방 자동 생성 → 코치드 훈련 → 성장 추적의 폐루프

---

## 핵심 철학

### 3모듈 역할 분담 (의료 비유)

```
모의고사 개별평가 (Stage B) = 의사의 진찰
모의고사 종합평가 (Stage C) = 의사의 진단서 + 처방전
튜터링                      = 물리치료사 (치료 + 재활 + 추적)
```

### 원칙: "피드백은 1회, 수행은 5회"

- 한 문항을 평가해도 "아는 것"에 머무름
- 한 능력을 **형태를 바꿔 5번 수행**해야 "할 수 있는 것"이 됨

---

## 사용자 시나리오

### Phase 1: 진단 (진단 탭)

```
① 모의고사 선택
   최근 완료된 모의고사 목록에서 진단 대상 선택 (드롭다운)

② 현재 상태 확인
   ├─ 확정 등급 + 목표 등급
   ├─ FACT 4영역 점수 (유창성/정확성/내용/과제)
   ├─ 핵심 약점 Top 3 (행동 지표로 변환)
   │   예: "과거시제 일관성 60% 미만"
   │   예: "예시 디테일(when/where/who) 2개 미만"
   └─ 유형별 진단 (양호/안정/약함/매우약함)
```

### Phase 2: 처방 (처방 탭)

```
③ 처방 자동 생성
   종합평가 tutoring_prescription → 처방 엔진 → 과제 1~6개 자동 생성
   ├─ Priority 1: 약점 Top 2 (drill_tag → question_type 매핑)
   ├─ Priority 2: 반복 오류 패턴 Top 2
   └─ Priority 3: 취약 유형 Top 2

④ 처방 과제 목록 확인
   각 과제: 유형 + 약점 태그 + 우선순위(높음/권장/추가)
   → "훈련 시작" 버튼 클릭
```

### Phase 3: 코치드 훈련 (훈련 탭 → 훈련 세션)

```
⑤ Screen 0: 세션 브리프 (1분)
   → 오늘의 목표, 약점, 금지 습관, 성공 기준 3개

⑥ Screen 1: 워밍업 (4분)
   → 모범 답변 쉐도잉 2회 (등급별 텍스트 정책)

⑦ Screen 2: EPP 패턴 고정 (5분)
   → 뼈대 카드 2~3개로 3회 반복 발화

⑧ Screen 3: 변형 훈련 (7분)
   → Forced Variation: 조건 바꿔 2~3회 발화
   → Oral Transformation: 즉석 문법 변환 6~10장

⑨ Screen 4: 타임드 실전 (7분)
   → 15-60-15 프레임 (준비-본론-마무리)
   → GPT 블록 체크리스트 + 문법 교정

⑩ Screen 5: Self-repair (4분)
   → Screen 4 문법 오류 하이라이트 → 수정 재녹음

⑪ Screen 6: 리캡 (2분)
   → 3줄 요약 + 다음 세션 추천
```

### Phase 4: 폐루프

```
⑫ 처방 자동 완료
   같은 처방에서 훈련 3회 → 처방 status: completed
   → 다음 처방으로 자동 이동

⑬ 전체 처방 완료
   모든 처방 completed → 새 모의고사 → 새 진단 → 새 처방
   → 폐루프 반복
```

## 전체 데이터 흐름도

```
[모의고사 종합평가]
   │
   │  coaching_report + tutoring_prescription
   │
   ▼
┌───────────────────────────────────────────┐
│         진단 탭                            │
│  FACT 점수 + 약점 Top 3 + 유형별 진단       │
└──────────────────┬────────────────────────┘
                   │ 처방 생성 버튼
                   ▼
┌───────────────────────────────────────────┐
│         처방 탭                            │
│  startTutoringSession() SA                │
│  ├─ mock_test_reports 조회                │
│  ├─ buildPrescriptions() 처방 엔진        │
│  │   ├─ priority_weaknesses → drill_tag    │
│  │   ├─ error_drill_tags                   │
│  │   └─ weak_types                         │
│  ├─ tutoring_sessions INSERT               │
│  └─ tutoring_prescriptions BULK INSERT     │
└──────────────────┬────────────────────────┘
                   │ 훈련 시작 링크
                   ▼
┌───────────────────────────────────────────┐
│         훈련 세션 (7 Screen)               │
│                                           │
│  Screen 0: 세션 브리프 (EF session-brief)  │
│       ↓                                   │
│  Screen 1: 워밍업 (EF generate-warmup)     │
│       ↓                                   │
│  Screen 2: EPP (EF generate-epp)           │
│       ↓                                   │
│  Screen 3: 변형 (EF generate-variation     │
│           + generate-transformation)       │
│       ↓                                   │
│  Screen 4: 실전 (EF evaluate-timed)        │
│       ↓                                   │
│  Screen 5: Self-repair (EF evaluate-repair)│
│       ↓                                   │
│  Screen 6: 리캡 (EF complete-session)      │
│       + completeTrainingSession() SA       │
└──────────────────┬────────────────────────┘
                   │
                   ▼
┌───────────────────────────────────────────┐
│  처방 진행 업데이트                         │
│  training_count++ → 3회 도달 시 completed  │
│  → 다음 처방 또는 전체 완료                 │
└───────────────────────────────────────────┘
```

## 상태 전이 다이어그램

### 튜터링 세션 (tutoring_sessions.status)

```
active (세션 생성 — startTutoringSession)
  │ 모든 처방 완료 시
  ▼
completed
```

### 처방 과제 (tutoring_prescriptions.status)

```
pending (세션 생성 시 BULK INSERT)
  │ 첫 훈련 세션 생성 시
  ▼
in_progress (createTrainingSession)
  │ training_count >= 3 도달
  ▼
completed (completeTrainingSession → 자동 전환)
```

### 훈련 세션 (tutoring_training_sessions)

```
started_at 설정 (createTrainingSession)
  │ Screen 0~6 순차 진행
  │ screens_completed 증가
  │ Screen 6 완료
  ▼
completed_at 설정 (completeTrainingSession)
```

## 등급별 파라미터 자동 조절

```
getLevelRange(currentLevel, targetLevel) → IL_IM | IM_IH | IH_AL
```

| 파라미터 | IL→IM | IM→IH | IH→AL |
|---------|-------|-------|-------|
| **텍스트 정책** | 전체 표시 | 키워드만 | OFF |
| **타이머** | 15-30-15 | 15-60-15 | 15-75-15 |
| **EPP 카드** | 3개 | 3개 | 2개 |
| **Variation 변경 요소** | 1개 | 2개 | 3개 |
| **Self-repair 최소** | 0회 | 1회 | 2회 |
| **목표 답변 길이** | 30~45초 | 60~75초 | 75~90초 |
| **필수 블록** | 3개 | 5개 | 6개 |
| **GPT 피드백** | light | standard | deep |

## Server Actions 목록 (7개)

| # | 함수 | 역할 |
|---|------|------|
| 1 | `getDiagnosis` | 진단 데이터 조회 (세션+리포트+스킬+처방) |
| 2 | `startTutoringSession` | 세션 생성 + 처방 엔진 자동 생성 |
| 3 | `getPrescriptions` | 처방 과제 목록 조회 |
| 4 | `createTrainingSession` | 훈련 세션 생성 + 처방 상태 in_progress |
| 5 | `saveAttempt` | 훈련 시도 기록 (Screen별) |
| 6 | `completeTrainingSession` | 훈련 완료 + 처방 자동 완료 판정 |
| 7 | `getTrainingHistory` | 훈련 이력 조회 |

## Edge Function 핸들러 목록 (8개)

| # | action | Screen | 역할 |
|---|--------|--------|------|
| 1 | `session-brief` | 0 | 목표 + 성공 기준 생성 |
| 2 | `generate-warmup` | 1 | 모범 답변 + 쉐도잉 데이터 |
| 3 | `generate-epp` | 2 | EPP 뼈대 카드 생성 |
| 4 | `generate-variation` | 3 | 강제 변주 카드 생성 |
| 5 | `generate-transformation` | 3 | 구두 변환 문제 생성 |
| 6 | `evaluate-timed` | 4 | 타임드 실전 평가 |
| 7 | `evaluate-repair` | 5 | Self-repair 평가 |
| 8 | `complete-session` | 6 | 리캡 생성 + 세션 완료 |

## 모듈 연결 관계

```
모의고사 (15문항 평가)
  │
  ├──→ [진단 탭] coaching_report 기반 시각화
  │      ├── FACT 영역 점수 (score_f/a/c/t)
  │      ├── top3_priorities (약점 Top 3)
  │      └── question_type_map (유형별 상태)
  │
  ├──→ [처방 탭] tutoring_prescription 기반 자동 생성
  │      ├── priority_weaknesses → drill_tag 매핑
  │      ├── error_drill_tags → question_type 매핑
  │      └── weak_types → question_type 매핑
  │
  └──→ [훈련 탭] 실제 수행
         ├── questions DB (471개) — 변형 질문 랜덤 선택
         ├── Screen 2 EPP — 스크립트 슬롯 구조 활용
         └── Screen 6 → 처방 진행률 업데이트
               │
               └──→ 전체 완료 → 새 모의고사 → 진단 갱신 (폐루프)
```

## 소리담 대비 핵심 변경

| 항목 | 소리담 (v2) | 오픽톡닥 (v3) |
|------|-----------|-------------|
| 훈련 구조 | 4레벨 (드래그→쉐도잉→힌트→실전) | **6 Screen + 5 프로토콜** |
| 훈련 목적 | 교정 답변 암기 | **패턴 체화 + 변형 내성 + 자기수정** |
| 새 질문 연습 | 없음 | **Forced Variation + 471 DB** |
| 시간 훈련 | 없음 | **15-60-15 타이머** |
| 자기수정 | 없음 | **Self-repair 의무 수행** |
| 과제 생성 | pass_rate 기반 | **종합평가 처방 자동 변환** |
| 등급별 차이 | 동일 경로 | **파라미터 자동 조절** |
| 테이블 | 12+ | 7개 |
| Edge Function | 10개 + 유틸 20개+ | 1개 (8 handlers) |
