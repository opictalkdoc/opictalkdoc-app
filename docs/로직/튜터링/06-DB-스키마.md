# 06. DB 스키마

> 7개 테이블 + Storage 버킷 + RLS 정책 + 인덱스 + JSONB 구조

---

## 테이블 관계도

```
auth.users (id)
   │
   ├──→ tutoring_sessions (user_id)
   │       │
   │       └──→ tutoring_prescriptions (session_id, user_id)
   │               │
   │               ├──→ tutoring_training_sessions (prescription_id, user_id)
   │               │       │
   │               │       └──→ tutoring_attempts (training_session_id, user_id)
   │               │
   │               └──→ tutoring_review_schedule (prescription_id, user_id)
   │
   └──→ tutoring_skill_history (user_id)

mock_test_sessions (id)
   │
   └──→ tutoring_sessions (mock_test_session_id)  ← TEXT FK (soft reference)
```

### CASCADE 체인

```
auth.users DELETE CASCADE
├── tutoring_sessions → CASCADE
│   └── tutoring_prescriptions → CASCADE
│       └── tutoring_review_schedule → CASCADE
│       (training_sessions: SET NULL — 훈련 기록 보존)
├── tutoring_review_schedule → CASCADE (user_id)
└── tutoring_skill_history → CASCADE (user_id)

tutoring_training_sessions DELETE CASCADE
└── tutoring_attempts → CASCADE
```

**설계 의도**:
- `tutoring_prescriptions → tutoring_training_sessions`: **SET NULL** — 처방 삭제 시 훈련 기록은 보존 (prescription_id만 NULL)
- 나머지: CASCADE — 상위 삭제 시 하위도 함께 삭제

## 1. tutoring_sessions (튜터링 세션)

> 모의고사 결과 기반으로 생성되는 최상위 컨테이너

```sql
CREATE TABLE tutoring_sessions (
  id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id                 UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  mock_test_session_id    TEXT NOT NULL,        -- soft FK (mock_test_sessions.id)
  target_level            TEXT,                 -- 목표 등급 (IH, AL 등)
  current_level           TEXT,                 -- 현재 등급 (IM2, IM3 등)
  status                  TEXT DEFAULT 'active'
    CHECK (status IN ('active', 'paused', 'completed')),
  total_prescriptions     SMALLINT,             -- 처방 총 개수 (1~6)
  completed_prescriptions SMALLINT DEFAULT 0,   -- 완료된 처방 수 (atomic RPC 증가)
  started_at              TIMESTAMPTZ DEFAULT NOW(),
  last_activity_at        TIMESTAMPTZ,          -- 마지막 활동 시각
  created_at              TIMESTAMPTZ DEFAULT NOW()
);
```

### 상태 전이

```
active (startTutoringSession SA에서 생성)
  │ 모든 처방 완료 시
  ▼
completed
```

- `paused` 상태는 스키마에 정의되어 있으나 현재 미사용
- `completed_prescriptions >= total_prescriptions` 시 자동 전환 로직은 SA에서 처리

### 멱등성

```
startTutoringSession() SA:
  ① 같은 mock_test_session_id로 기존 세션 조회
  ② 있으면 기존 세션 + 처방 반환 (새로 생성하지 않음)
  ③ 없으면 INSERT
```

## 2. tutoring_prescriptions (처방 과제)

> 모의고사 종합평가 → 처방 엔진 → 1~6개 자동 생성

```sql
CREATE TABLE tutoring_prescriptions (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id        UUID NOT NULL REFERENCES tutoring_sessions(id) ON DELETE CASCADE,
  user_id           UUID NOT NULL REFERENCES auth.users(id),
  priority          SMALLINT NOT NULL,         -- 1(높음), 2(권장), 3(추가)
  question_type     TEXT NOT NULL,             -- description, routine, comparison, ...
  topic_id          TEXT,                      -- 주제 ID (선택)
  weakness_tags     JSONB,                     -- 약점 태그 배열
  source            TEXT NOT NULL
    CHECK (source IN ('top3_priorities', 'recurring_patterns', 'question_type_map', 'individual_eval')),
  source_data       JSONB,                     -- 처방 근거 원본 데이터
  level_params      JSONB,                     -- LEVEL_PARAMS 복사본
  status            TEXT DEFAULT 'pending'
    CHECK (status IN ('pending', 'in_progress', 'completed')),
  training_count    SMALLINT DEFAULT 0,        -- 훈련 횟수 (atomic RPC 증가)
  best_score        JSONB,                     -- 최고 점수 기록
  created_at        TIMESTAMPTZ DEFAULT NOW()
);
```

### JSONB 필드 구조

#### weakness_tags

```json
["tense_consistency", "detail_expansion"]
```

- 문자열 배열
- DRILL_TAG_TO_QUESTION_TYPE의 drill_tag 값들

#### source_data

```json
{
  "rank": 1,
  "area": "시제 일관성",
  "drill_tag": "tense_consistency"
}
```

- 처방 생성 근거 원본 (priority_weaknesses, error_drill_tags, weak_types 중 해당 항목)

#### level_params

```json
{
  "text_policy": "keywords",
  "timer": { "prep": 15, "main": 60, "wrap": 15 },
  "epp_count": 3,
  "variation_changes": 2,
  "self_repair_min": 1,
  "target_duration": { "min": 60, "max": 75 },
  "required_blocks": 5,
  "gpt_feedback_depth": "standard"
}
```

- `LEVEL_PARAMS[getLevelRange()]` 전체 복사
- 처방 생성 시점의 파라미터 스냅샷 (나중에 상수 변경되어도 영향 없음)

#### best_score

```json
{
  "structure_score": 85,
  "content_score": 78,
  "pronunciation_score": 72
}
```

### 상태 전이

```
pending (startTutoringSession에서 BULK INSERT)
  │ 첫 훈련 세션 생성 시 (createTrainingSession)
  ▼
in_progress
  │ training_count >= 3 도달 시 (completeTrainingSession)
  ▼
completed
```

### priority → source 매핑

| priority | source | 데이터 출처 |
|----------|--------|------------|
| 1 (높음) | `top3_priorities` | `tutoring_prescription.priority_weaknesses` 상위 2개 |
| 2 (권장) | `recurring_patterns` | `tutoring_prescription.error_drill_tags` 상위 2개 |
| 3 (추가) | `question_type_map` | `tutoring_prescription.weak_types` 상위 2개 |

## 3. tutoring_training_sessions (훈련 세션)

> 처방 1개당 최대 3회 훈련. Screen 0~6 진행 기록

```sql
CREATE TABLE tutoring_training_sessions (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id             UUID NOT NULL REFERENCES auth.users(id),
  prescription_id     UUID REFERENCES tutoring_prescriptions(id) ON DELETE SET NULL,
  session_type        TEXT DEFAULT 'guided'
    CHECK (session_type IN ('guided', 'free', 'simulation')),
  question_type       TEXT NOT NULL,            -- prescription에서 복사
  topic_id            TEXT,
  target_level        TEXT,                     -- 세션 생성 시 부모에서 조회
  level_params        JSONB,                    -- prescription.level_params 복사
  session_goal        TEXT,                     -- EF session-brief 응답
  success_criteria    JSONB,                    -- EF session-brief 응답
  duration_seconds    INTEGER,                  -- 총 훈련 시간 (완료 시 계산)
  screens_completed   SMALLINT DEFAULT 0,       -- 완료한 Screen 수 (0~6)
  overall_score       JSONB,                    -- 완료 시 종합 점수
  kpi_results         JSONB,                    -- KPI 달성 결과
  next_recommendation JSONB,                    -- EF complete-session 응답
  started_at          TIMESTAMPTZ DEFAULT NOW(),
  completed_at        TIMESTAMPTZ               -- 완료 시각 (NULL = 미완료)
);
```

### JSONB 필드 구조

#### success_criteria

```json
[
  { "id": 1, "text": "과거시제를 3회 이상 정확히 사용", "met": false },
  { "id": 2, "text": "구체적 예시를 2개 이상 포함", "met": false },
  { "id": 3, "text": "60초 이상 발화", "met": false }
]
```

- EF `session-brief` 핸들러가 생성
- Screen 4 평가 후 `met` 업데이트 (현재는 미구현)

#### overall_score

```json
{
  "structure": 80,
  "content": 75,
  "pronunciation": 70,
  "time_management": 85
}
```

#### kpi_results

```json
{
  "blocks_completed": 4,
  "required_blocks": 5,
  "self_repairs": 2,
  "self_repair_min": 1,
  "passed": true
}
```

#### next_recommendation

```json
{
  "type": "same_type",
  "reason": "시제 일관성이 아직 불안정합니다. 같은 유형으로 1회 더 연습하면 좋겠습니다."
}
```

- EF `complete-session` 핸들러가 생성

### 중복 방지

```
createTrainingSession() SA:
  ① completed_at IS NULL인 기존 훈련 조회
  ② 있으면 기존 ID 반환 (새로 생성하지 않음)
```

### session_type 설명

| 값 | 설명 | 현재 사용 |
|----|------|----------|
| `guided` | 처방 기반 가이드 훈련 (표준) | ✅ 기본값 |
| `free` | 자유 훈련 (처방 없이) | ⏳ 미구현 |
| `simulation` | 시뮬레이션 모드 | ⏳ 미구현 |

## 4. tutoring_attempts (훈련 시도)

> Screen별 개별 시도 기록. 가장 많은 데이터가 쌓이는 테이블

```sql
CREATE TABLE tutoring_attempts (
  id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id                 UUID NOT NULL REFERENCES auth.users(id),
  training_session_id     UUID NOT NULL REFERENCES tutoring_training_sessions(id)
    ON DELETE CASCADE,
  screen_number           SMALLINT NOT NULL,    -- 0~6
  protocol                TEXT
    CHECK (protocol IS NULL OR protocol IN (
      'epp', 'variation', 'transformation',
      'timed', 'self_repair', 'warmup', 'simulation'
    )),
  question_id             TEXT,                 -- questions.id 참조
  attempt_number          SMALLINT,             -- 같은 Screen 내 시도 번호
  user_answer             TEXT,                 -- STT 변환 텍스트
  user_audio_url          TEXT,                 -- Storage URL
  audio_duration_seconds  INTEGER,
  -- 측정 지표
  metrics                 JSONB,
  pronunciation           JSONB,
  -- 평가 결과
  evaluation              JSONB,
  passed                  BOOLEAN,              -- 통과 여부
  -- Self-repair 전용 필드
  repair_before           TEXT,                 -- 수정 전 원문
  repair_after            TEXT,                 -- 수정 후 텍스트
  repair_type             TEXT
    CHECK (repair_type IS NULL OR repair_type IN (
      'grammar', 'l1_interference', 'upgrade'
    )),
  created_at              TIMESTAMPTZ DEFAULT NOW()
);
```

### Screen → Protocol 매핑

| Screen | protocol | 설명 |
|--------|----------|------|
| 0 | NULL | 세션 브리프 (시도 기록 없음) |
| 1 | `warmup` | 워밍업 쉐도잉 |
| 2 | `epp` | EPP 패턴 고정 |
| 3 | `variation` / `transformation` | 변형 훈련 |
| 4 | `timed` | 타임드 실전 |
| 5 | `self_repair` | Self-repair |
| 6 | NULL | 리캡 (시도 기록 없음) |

### JSONB 필드 구조

#### metrics

```json
{
  "wpm": 120,
  "filler_count": 3,
  "filler_pct": 2.5,
  "silence_seconds": 5,
  "connector_count": 4,
  "block_completion": {
    "opening": true,
    "background": true,
    "detail": true,
    "emotion": false,
    "closing": true
  }
}
```

- Screen 4 (timed) 전용 — EF `evaluate-timed` 응답에서 추출

#### pronunciation

```json
{
  "accuracy_score": 78,
  "fluency_score": 72,
  "prosody_score": 68,
  "pronunciation_score": 65,
  "completeness_score": 90
}
```

- Azure Speech SDK 발음 평가 결과 (현재 튜터링에서는 미사용, 향후 확장용)

#### evaluation

```json
{
  "block_checklist": {
    "opening": { "present": true, "quality": "good" },
    "background": { "present": true, "quality": "fair" },
    "detail": { "present": true, "quality": "good" },
    "emotion": { "present": false, "quality": "none" },
    "closing": { "present": true, "quality": "fair" }
  },
  "structure_score": 80,
  "content_score": 75,
  "time_management": "적절",
  "grammar_issues": [
    { "original": "I go there yesterday", "corrected": "I went there yesterday", "type": "tense" }
  ],
  "strengths": ["자연스러운 시작", "구체적 예시 포함"],
  "improvements": ["과거시제 일관성", "마무리 문장 추가"],
  "overall_comment": "전반적으로 좋은 구조...",
  "passed": true
}
```

- Screen 4 (timed): EF `evaluate-timed` 전체 응답 저장
- Screen 5 (self_repair): EF `evaluate-repair` 전체 응답 저장

#### repair_type 설명

| 값 | 설명 |
|----|------|
| `grammar` | 문법 오류 수정 (시제, 주어-동사 일치 등) |
| `l1_interference` | 모국어 간섭 수정 (직역체, 어순 등) |
| `upgrade` | 표현 업그레이드 (단순→고급 표현) |

## 5. tutoring_review_schedule (SRS 복습 스케줄)

> 간격 반복 복습 스케줄. 처방 완료 후 자동 생성 (향후 구현)

```sql
CREATE TABLE tutoring_review_schedule (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  prescription_id UUID NOT NULL REFERENCES tutoring_prescriptions(id) ON DELETE CASCADE,
  question_type   TEXT NOT NULL,
  next_review_at  TIMESTAMPTZ NOT NULL,        -- 다음 복습 예정일
  review_count    SMALLINT DEFAULT 0,          -- 누적 복습 횟수
  interval_days   SMALLINT DEFAULT 1,          -- 다음 간격 (1→3→7→14→30)
  status          TEXT DEFAULT 'scheduled'
    CHECK (status IN ('scheduled', 'completed', 'skipped')),
  created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### SRS 간격 규칙 (설계)

```
복습 0회: 1일 후
복습 1회: 3일 후
복습 2회: 7일 후
복습 3회: 14일 후
복습 4회: 30일 후
```

### 현재 구현 상태

- **스키마**: ✅ 생성 완료
- **로직**: ⏳ 미구현 (향후 확장)
- **인덱스**: 부분 인덱스 `WHERE status = 'scheduled'` (스케줄 조회 최적화)

## 6. tutoring_skill_history (성장 추적)

> 훈련 완료 시마다 스킬 스냅샷 기록. 진단 탭 성장 추이 데이터 소스

```sql
CREATE TABLE tutoring_skill_history (
  id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id                 UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  source                  TEXT NOT NULL
    CHECK (source IN ('mock_test', 'training', 'simulation')),
  source_id               TEXT,                 -- session/training ID
  question_type           TEXT,
  avg_structure_score     NUMERIC(3,1),         -- 구조 점수 평균
  avg_wpm                 NUMERIC(5,1),         -- WPM 평균
  avg_filler_pct          NUMERIC(4,1),         -- 필러 비율(%) 평균
  avg_connector_count     NUMERIC(3,1),         -- 연결어 사용 횟수 평균
  avg_silence_seconds     NUMERIC(4,1),         -- 침묵 시간(초) 평균
  self_repair_count       SMALLINT,             -- Self-repair 횟수
  variation_success_rate  NUMERIC(4,2),         -- 변형 훈련 성공률
  block_completion_rate   NUMERIC(4,2),         -- 블록 완성률
  recorded_at             TIMESTAMPTZ DEFAULT NOW()
);
```

### source 설명

| 값 | 설명 | source_id |
|----|------|-----------|
| `mock_test` | 모의고사 결과 반영 | mock_test_session.id |
| `training` | 훈련 세션 완료 반영 | training_session.id |
| `simulation` | 시뮬레이션 결과 반영 | training_session.id |

### NUMERIC 타입 선택 이유

- Supabase NUMERIC → JavaScript string 변환 이슈 → 클라이언트에서 `Number()` 변환 필요
- 정밀한 소수점 제어: `NUMERIC(3,1)` = 최대 99.9, `NUMERIC(4,2)` = 최대 99.99

### 진단 탭 활용

```sql
-- getDiagnosis() SA에서 사용
SELECT * FROM tutoring_skill_history
WHERE user_id = $1
ORDER BY recorded_at DESC
LIMIT 50;
```

## 인덱스 전체 목록 (10개)

| # | 테이블 | 인덱스명 | 컬럼 | 타입 |
|---|--------|----------|------|------|
| 1 | tutoring_sessions | idx_tutoring_sessions_user | user_id | 일반 |
| 2 | tutoring_sessions | idx_tutoring_sessions_mock | mock_test_session_id | 일반 |
| 3 | tutoring_prescriptions | idx_tutoring_prescriptions_session | session_id | 일반 |
| 4 | tutoring_prescriptions | idx_tutoring_prescriptions_user | user_id | 일반 |
| 5 | tutoring_training_sessions | idx_tutoring_training_sessions_user | user_id | 일반 |
| 6 | tutoring_training_sessions | idx_tutoring_training_sessions_prescription | prescription_id | 일반 |
| 7 | tutoring_attempts | idx_tutoring_attempts_training | training_session_id | 일반 |
| 8 | tutoring_attempts | idx_tutoring_attempts_user | user_id | 일반 |
| 9 | tutoring_review_schedule | idx_tutoring_review_user | user_id | 일반 |
| 10 | tutoring_review_schedule | idx_tutoring_review_next | next_review_at | **부분** (WHERE status = 'scheduled') |

### 인덱스 설계 원칙

- **user_id**: 모든 테이블에 존재 — RLS `auth.uid() = user_id` 조건 최적화
- **FK 컬럼**: JOIN 빈번한 외래키에 인덱스 (session_id, prescription_id, training_session_id)
- **부분 인덱스**: `tutoring_review_schedule`의 `next_review_at` — `scheduled` 상태만 인덱싱 (완료/건너뛴 행 제외)

### DELETE 정책 없음 — 설계 의도

- 훈련 기록은 **삭제 불가** (사용자 실수 방지 + 성장 추적 데이터 보존)
- 관리자 삭제 필요 시 → Supabase Dashboard 또는 SERVICE_ROLE_KEY 직접 사용

### skill_history UPDATE 없음 — 설계 의도

- 스냅샷 성격의 데이터 → **INSERT-ONLY**
- 한번 기록된 스킬 히스토리는 변경하지 않음 (이력 보존)

## 모의고사 모듈과의 데이터 연결

```
mock_test_sessions
├── mock_test_reports
│   ├── coaching_report (JSONB) → 진단 탭 시각화
│   └── tutoring_prescription (JSONB) → 처방 엔진 입력
│
└── tutoring_sessions (mock_test_session_id = TEXT soft FK)
    └── tutoring_prescriptions → tutoring_training_sessions → tutoring_attempts
```

### soft FK 사용 이유

- `mock_test_sessions.id`는 `mt_` prefix TEXT
- UUID FK 불가 → TEXT 컬럼으로 soft reference
- JOIN 시 명시적 조건: `WHERE mock_test_session_id = $1`

## queryKey 매핑 (TanStack Query)

| queryKey | 관련 테이블 | staleTime | 비고 |

| `["tutoring-diagnosis"]` | sessions, reports, skill_history, prescriptions | 5분 | 4개 병렬 쿼리 |

### 캐시 무효화 시점

```
completeTrainingSession() 성공 → invalidate ["tutoring-diagnosis"]
                                  invalidate ["tutoring-prescriptions", sessionId]
```
