# 06. DB 스키마

> 3개 테이블 + RPC 1개 + RLS 정책 + 인덱스 + 트리거

---

## 테이블 관계도

```
auth.users (id)
   │
   └──→ submissions (user_id)
           │
           ├──→ submission_questions (submission_id)
           │       │
           │       └──→ questions (question_id)  ← FK JOIN
           │
           └──→ submission_combos (submission_id)

user_credits (user_id)
   └── RPC increment_script_credits (크레딧 보상)
```

### CASCADE 체인

```
submissions DELETE
├── submission_questions → CASCADE (FK)
└── submission_combos → CASCADE (FK)
```

- 사용자 삭제 시: `auth.users DELETE CASCADE → submissions → questions, combos`
- 후기 삭제 시: 하위 질문/콤보 자동 삭제

## 1. submissions (후기 마스터)

> 시험후기 핵심 테이블. Step 1~3 데이터 + 상태 관리 + 크레딧 기록.

```sql
CREATE TABLE submissions (
  id                    BIGSERIAL PRIMARY KEY,
  user_id               UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,

  -- Step 1: 시험 정보
  exam_date             DATE NOT NULL,
  exam_difficulty       TEXT NOT NULL,          -- "6-5" 형식 (16가지)
  pre_exam_level        TEXT NOT NULL,          -- none/NL/NM/.../AL
  achieved_level        TEXT,                   -- NULL = 아직 모름

  -- Step 1: 시험 배경
  exam_purpose          TEXT NOT NULL,          -- employment/promotion/...
  study_methods         TEXT[] NOT NULL,        -- 배열: ['self_study', 'academy']
  prep_duration         TEXT NOT NULL,          -- under_1w/1_2w/...
  attempt_count         TEXT NOT NULL,          -- first/2nd/3rd/...

  -- Step 1: 체감 후기
  perceived_difficulty  TEXT NOT NULL,          -- easy/normal/hard
  time_sufficiency      TEXT,                   -- (미사용, 하위호환)
  actual_duration       TEXT NOT NULL,          -- under_20/20_25/...

  -- Step 1: 서베이
  used_recommended_survey BOOLEAN DEFAULT true,
  survey_occupation     TEXT,                   -- 추천 서베이 사용 시 NULL
  survey_student        TEXT,
  survey_course         TEXT,
  survey_housing        TEXT,
  survey_leisure        TEXT[],                 -- 배열
  survey_hobbies        TEXT[],
  survey_sports         TEXT[],
  survey_travel         TEXT[],

  -- Step 3: 자유 후기
  one_line_review       TEXT,                   -- 1~100자
  tips                  TEXT,                   -- 0~300자

  -- 상태 관리
  status                TEXT DEFAULT 'draft'
    CHECK (status IN ('draft', 'complete')),
  step_completed        SMALLINT DEFAULT 1      -- 1, 2, 3
    CHECK (step_completed BETWEEN 1 AND 3),

  -- 크레딧 기록
  credit_granted        BOOLEAN DEFAULT false,  -- 크레딧 지급 여부

  -- 타임스탬프
  submitted_at          TIMESTAMPTZ,            -- 완료 시각 (complete 전환 시 NOW())
  created_at            TIMESTAMPTZ DEFAULT NOW(),
  updated_at            TIMESTAMPTZ DEFAULT NOW()
);
```

### 주요 컬럼 상세

#### exam_difficulty (2단계 Select)

```
형식: "{시작난이도}-{재조정난이도}"
범위: 1~6
예: "6-5" = 시작 6, 재조정 5

16가지 조합:
6-6, 6-5, 5-6, 5-5, 5-4, 4-5, 4-4, 4-3,
3-4, 3-3, 3-2, 2-3, 2-2, 2-1, 1-2, 1-1
```

#### study_methods (TEXT 배열)

```sql
-- PostgreSQL TEXT[] 타입
study_methods TEXT[] NOT NULL
-- 예: ARRAY['self_study', 'academy']
```

- Supabase JS에서 배열로 직접 저장/조회
- 5가지 값 중 1개 이상: `self_study`, `online_lecture`, `academy`, `study_group`, `app`

#### credit_granted

```
false (기본값): 크레딧 미지급
true: 스크립트 크레딧 2개 지급 완료

지급 시점: completeSubmission() → RPC 성공 후 UPDATE
```

### 상태별 허용 연산

| 상태 | SELECT | INSERT | UPDATE | DELETE |
|------|--------|--------|--------|--------|
| draft | 본인만 | ✅ | Step 진행 | ✅ |
| complete | 전체 공개 | — | 등급만 | ❌ |

## 2. submission_questions (질문 기록)

> 14개 출제 질문 + Q1 자기소개 = 15행

```sql
CREATE TABLE submission_questions (
  id                    BIGSERIAL PRIMARY KEY,
  submission_id         BIGINT NOT NULL REFERENCES submissions(id) ON DELETE CASCADE,
  question_number       SMALLINT NOT NULL       -- 1~15
    CHECK (question_number BETWEEN 1 AND 15),
  combo_type            TEXT                    -- general_1~3, roleplay, advance (Q1=NULL)
    CHECK (combo_type IS NULL OR combo_type IN (
      'general_1', 'general_2', 'general_3', 'roleplay', 'advance'
    )),
  topic                 TEXT NOT NULL,           -- "집", "여행", "자기소개" 등
  question_id           TEXT REFERENCES questions(id),  -- master question FK (nullable)
  custom_question_text  TEXT,                   -- 직접 입력 텍스트 (nullable)
  is_not_remembered     BOOLEAN DEFAULT false,  -- 기억 안남 여부
  created_at            TIMESTAMPTZ DEFAULT NOW()
);
```

### question_number 범위

| 번호 | 콤보 | 비고 |
|------|------|------|
| 1 | NULL | 자기소개 (자동 추가, combo 미소속) |
| 2~4 | general_1 | 일반 콤보 1 |
| 5~7 | general_2 | 일반 콤보 2 |
| 8~10 | general_3 | 일반 콤보 3 |
| 11~13 | roleplay | 롤플레이 |
| 14~15 | advance | 어드밴스 |

### 질문 입력 3가지 모드

| 모드 | question_id | custom_question_text | is_not_remembered |
|------|-------------|---------------------|-------------------|
| master 선택 | "Q_DESC_001" | NULL | false |
| 직접 입력 | NULL | "What about..." | false |
| 기억 안남 | NULL | NULL | true |

### questions 테이블 JOIN

```sql
-- getSubmissionWithQuestions()에서 사용
SELECT sq.*, q.*
FROM submission_questions sq
LEFT JOIN questions q ON sq.question_id = q.id
WHERE sq.submission_id = $1
ORDER BY sq.question_number;
```

- `question_id` NULL이면 JOIN 결과 없음 (직접 입력 / 기억 안남)

## 3. submission_combos (통합 콤보)

> completeSubmission()에서 자동 추출. 빈도 분석 데이터 소스.

```sql
CREATE TABLE submission_combos (
  id                BIGSERIAL PRIMARY KEY,
  submission_id     BIGINT NOT NULL REFERENCES submissions(id) ON DELETE CASCADE,
  combo_type        TEXT NOT NULL
    CHECK (combo_type IN (
      'general_1', 'general_2', 'general_3', 'roleplay', 'advance'
    )),
  topic             TEXT NOT NULL,             -- 유니크 주제 쉼표 연결 ("집, 여행")
  question_ids      TEXT[] NOT NULL,           -- 선택된 질문 ID 배열
  created_at        TIMESTAMPTZ DEFAULT NOW()
);
```

### 콤보 생성 조건

모든 질문이 다음을 충족해야 콤보 생성:

```
1. question_id 존재 (master question)
2. is_not_remembered = false
3. custom_question_text = null
4. 제외 주제 아님 (공원, 카페, 하이킹 — general+advance만)
```

### 데이터 예시

```json
{
  "id": 1,
  "submission_id": 42,
  "combo_type": "general_1",
  "topic": "집",
  "question_ids": ["Q_DESC_001", "Q_ROUT_023", "Q_COMP_005"]
}
```

- **topic**: 콤보 내 유니크 주제를 쉼표로 연결 (보통 1개)
- **question_ids**: TEXT 배열 (PostgreSQL `TEXT[]`)
- 후기당 **최대 5개** 콤보 (유효하지 않은 콤보는 스킵)

## RLS 정책

### submissions (4개)

```sql
-- 본인 후기: 모든 상태 조회
CREATE POLICY "submissions_select_own"
  ON submissions FOR SELECT
  USING (auth.uid() = user_id);

-- 공개 후기: complete만 전체 조회 (비인증 포함)
CREATE POLICY "submissions_select_public"
  ON submissions FOR SELECT
  USING (status = 'complete');

-- INSERT: 본인만
CREATE POLICY "submissions_insert_own"
  ON submissions FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- UPDATE: 본인만
CREATE POLICY "submissions_update_own"
  ON submissions FOR UPDATE
  USING (auth.uid() = user_id);

-- DELETE: 본인의 draft만
CREATE POLICY "submissions_delete_own_draft"
  ON submissions FOR DELETE
  USING (auth.uid() = user_id AND status = 'draft');
```

**핵심**: complete 후기는 삭제 불가 (RLS 레벨에서 차단)

### submission_questions (3개)

```sql
-- SELECT: submission 소유자 또는 complete 후기
CREATE POLICY "sq_select_own_or_public"
  ON submission_questions FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM submissions s
      WHERE s.id = submission_id
        AND (s.user_id = auth.uid() OR s.status = 'complete')
    )
  );

-- INSERT: submission 소유자
CREATE POLICY "sq_insert_own"
  ON submission_questions FOR INSERT
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM submissions s
      WHERE s.id = submission_id AND s.user_id = auth.uid()
    )
  );

-- DELETE: submission 소유자
CREATE POLICY "sq_delete_own"
  ON submission_questions FOR DELETE
  USING (
    EXISTS (
      SELECT 1 FROM submissions s
      WHERE s.id = submission_id AND s.user_id = auth.uid()
    )
  );
```

### submission_combos (2개)

```sql
-- 인증 사용자: 전체 SELECT (빈도 분석용)
CREATE POLICY "sc_select_authenticated"
  ON submission_combos FOR SELECT
  USING (auth.role() = 'authenticated');

-- 비인증 사용자: advance만 SELECT
CREATE POLICY "sc_select_advance_anon"
  ON submission_combos FOR SELECT
  USING (combo_type = 'advance');
```

**비인증 제한 이유**: 일반/롤플레이 빈도 데이터는 유료 사용자 혜택 (향후 차별화 가능)

## 트리거

### updated_at 자동 갱신

```sql
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER submissions_updated_at
  BEFORE UPDATE ON submissions
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

- submissions 테이블에만 적용
- UPDATE 시 `updated_at` 자동 현재 시각

## RPC 함수

### increment_script_credits

```sql
CREATE OR REPLACE FUNCTION increment_script_credits(
  p_user_id UUID,
  p_amount INTEGER
) RETURNS VOID AS $$
BEGIN
  UPDATE user_credits
  SET script_credits = script_credits + p_amount
  WHERE user_id = p_user_id;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

| 속성 | 값 |
|------|-----|
| 호출자 | `completeSubmission()` SA |
| 권한 | SECURITY DEFINER (RLS 우회) |
| 파라미터 | user_id, amount (보통 2) |
| 대상 | user_credits.script_credits |

**SECURITY DEFINER 사용 이유**:
- `user_credits`에 사용자 직접 UPDATE 허용 시 크레딧 조작 가능
- RPC로만 크레딧 변경 → SA에서만 호출 → 비즈니스 로직으로 제어

## 열거형 값 정리

### achieved_level / pre_exam_level

```
NL  — Novice Low
NM  — Novice Mid
NH  — Novice High
IL  — Intermediate Low
IM1 — Intermediate Mid 1
IM2 — Intermediate Mid 2
IM3 — Intermediate Mid 3
IH  — Intermediate High
AL  — Advanced Low

pre_exam_level 추가: 'none' (처음 응시)
```

### combo_type

```
general_1  — 일반 콤보 1 (Q2~4)
general_2  — 일반 콤보 2 (Q5~7)
general_3  — 일반 콤보 3 (Q8~10)
roleplay   — 롤플레이 (Q11~13)
advance    — 어드밴스 (Q14~15)
```

### status

```
draft    — 작성 중 (수정/삭제 가능)
complete — 완료 (등급만 수정 가능, 삭제 불가)
```

## queryKey → 테이블 매핑

| queryKey | 테이블 | 연산 |
|----------|--------|------|
| `["review-frequency"]` | submission_combos, submissions | 집계 |
| `["my-submissions"]` | submissions | SELECT |
| `["public-reviews", filter]` | submissions | SELECT (complete) |
| `["question-frequency", topic]` | submission_combos, questions | JOIN 집계 |
| `["submission-detail", id]` | submission_questions, questions | JOIN |
| `["user-credits", userId]` | user_credits | SELECT (외부 테이블) |

### 캐시 무효화

```
completeSubmission() 성공 시:
  → invalidate ["my-submissions"]
  → invalidate ["user-credits", userId]
  → invalidate ["review-frequency"]

deleteSubmission() 성공 시:
  → invalidate ["my-submissions"]

updateGrade() 성공 시:
  → invalidate ["my-submissions"]
  → invalidate ["public-reviews"]  (등급 필터 영향)
```

## user_credits 연동 (외부 테이블)

```sql
-- user_credits 테이블 (002_payment_tables.sql에서 생성)
user_credits (
  user_id              UUID PRIMARY KEY,
  plan_script_credits  INTEGER DEFAULT 0,  -- 플랜 구매 크레딧 (만료됨)
  script_credits       INTEGER DEFAULT 0,  -- 횟수권 크레딧 (영구)
  ...
)
```

### 크레딧 소진 순서

```
1. plan_script_credits (플랜 — 만료되므로 먼저 사용)
2. script_credits (횟수권 — 영구)
```

- 후기 보상은 **script_credits** (영구 크레딧)에 적립
- 소진은 스크립트 생성 시 `consume_script_credit` RPC에서 처리
