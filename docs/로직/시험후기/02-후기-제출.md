## 크레딧 보상 시스템 (25일 룰)

### 비즈니스 규칙

```
OPIc 응시 주기 ≈ 25일 → 크레딧 지급 주기도 25일

최초 2회: 무조건 script_credits +2
3회차~: 마지막 지급일로부터 25일 경과 시에만 지급
```

### 구현 로직

```typescript
// 1. 크레딧이 지급된 이전 후기 조회
const creditHistory = await supabase
  .from("submissions")
  .select("submitted_at")
  .eq("user_id", userId)
  .eq("status", "complete")
  .eq("credit_granted", true)
  .order("submitted_at", { ascending: false });

const creditCount = creditHistory?.length ?? 0;

// 2. 지급 판정
if (creditCount < 2) {
  // 최초 2회: 무조건 지급
  creditGranted = true;
} else {
  // 3회차+: 25일 경과 확인
  const lastGrantedAt = creditHistory![0].submitted_at;
  const daysSince = Math.floor(
    (Date.now() - new Date(lastGrantedAt).getTime()) / (1000 * 60 * 60 * 24)
  );
  creditGranted = daysSince >= 25;
  if (!creditGranted) {
    nextCreditDate = new Date(lastGrantedAt + 25일);
  }
}

// 3. RPC 호출 (SECURITY DEFINER — RLS 우회)
if (creditGranted) {
  await supabase.rpc("increment_script_credits", {
    p_user_id: userId,
    p_amount: 2,
  });
  // 성공 시에만 credit_granted = true 기록
}
```

### 안전장치

- **원자적 상태 전환**: `.eq("status", "draft")` → 이미 complete면 UPDATE 0건 (중복 방지)
- **크레딧 실패 분리**: RPC 실패해도 후기는 complete로 유지 (credit_granted=false)
- **삭제 불가**: complete 후기 삭제 금지 (크레딧 악용 방지)
- **SECURITY DEFINER**: 크레딧 RPC는 관리자 권한으로 실행 (user_credits 직접 수정 불가)

## 콤보 자동 추출

> 파일: `lib/utils/combo-extractor.ts` (79줄)

### 추출 규칙

```typescript
function extractCombos(questions: QuestionItem[]): ExtractedCombo[]
```

**5개 콤보 범위 (COMBO_RANGES)**:

| 콤보 | 범위 | 제외 주제 적용 |
|------|------|---------------|
| general_1 | Q2~Q4 | ✅ |
| general_2 | Q5~Q7 | ✅ |
| general_3 | Q8~Q10 | ✅ |
| roleplay | Q11~Q13 | ❌ |

### 유효성 검증 (isValidForCombo)

콤보로 인정되려면 **모든 질문이 다음 4가지를 충족**:

```
1. question_id 존재 (master question 선택됨)
2. is_not_remembered = false (기억함)
3. custom_question_text = null (직접 입력 아님)
4. EXCLUDED_TOPICS에 미포함 (excludeTopics=true인 경우만)
```

### 제외 주제

```typescript
EXCLUDED_TOPICS = ["공원", "카페", "하이킹"]
```

- 일반 콤보(1,2,3) + 어드밴스에서 제외
- 롤플레이는 제외 주제 미적용

```typescript
{
```

- 최대 5개 콤보 (유효하지 않은 콤보는 스킵)

## Draft 관리

### 1개 유지 정책

```
createDraft() 호출 시:
  ① 기존 draft 조회 (status='draft', user_id)
  ② 있으면 DELETE
  ③ 새 INSERT

결과: 사용자당 draft 최대 1개
```

### 이어쓰기 흐름

```
1. 후기 제출 탭 → Draft 카드의 "이어쓰기" 클릭
2. /reviews/submit?resume={submissionId} 이동
3. ReviewWizard:
   ① getDraft(resumeId) → Step 1 폼 데이터 복원
   ② getDraftQuestions(resumeId) → Step 2 질문 복원
   ③ step_completed에 따라 해당 Step으로 점프
```

### 삭제 규칙

```
Draft: ✅ 삭제 가능 (deleteSubmission)
Complete: ❌ 삭제 불가
  └─ 이유: 크레딧 악용 방지 + 빈도 분석 데이터 보존
```
