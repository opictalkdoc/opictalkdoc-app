# 04. 백엔드 — Server Actions

> 시험후기 모듈은 Edge Functions 없이 **Server Actions만으로 구성**.
> 12개 SA + 3개 읽기 전용 함수 + 1개 RPC + 콤보 추출 유틸리티.

---

## 아키텍처 개요

```
클라이언트 (React)
    │
    │ Server Actions (lib/actions/reviews.ts)
    ▼
Supabase Client (서버 측)
    │
    ├─ submissions 테이블 (CRUD)
    ├─ submission_questions 테이블 (CRD)
    ├─ submission_combos 테이블 (CR)
    └─ user_credits (RPC 호출만)
```

- **Edge Functions 없음**: 외부 AI API 호출이 필요 없으므로 SA만으로 충분
- **인증**: `requireUser()` → Supabase Auth 세션 검증 (쿠키 기반)
- **RLS**: 모든 쿼리에 `user_id` 조건 → Supabase RLS가 이중 검증

### 공통 헬퍼 함수

#### requireUser

```typescript
async function requireUser()
  → { supabase, userId }
```

인증 확인 + Supabase 클라이언트 반환. 미인증 시 `throw Error("로그인이 필요합니다")`.

#### buildStep1Fields

```typescript
function buildStep1Fields(data: Step1Input)
  → Record<string, unknown>  // DB INSERT/UPDATE용 필드 객체
```

Step 1 폼 데이터를 DB 컬럼에 매핑하는 헬퍼. `createDraft`와 `updateDraft`에서 공유.

- 서베이 배열 필드(leisure/hobbies/sports/travel)를 `.join(",")` 으로 **쉼표 구분 문자열**로 변환
- `used_recommended_survey=true` 시 커스텀 서베이 필드를 NULL 처리
- `achieved_level='unknown'` 시 NULL로 변환

### completeSubmission

```typescript
async function completeSubmission(data: {
  submission_id: number,
  one_line_review: string,
  tips?: string
})
  → { creditGranted: boolean, nextCreditDate?: string }
```

**로직** (가장 복잡한 SA — 3개 헬퍼로 분리):

```
1. Zod 검증 (step3Schema)
2. requireUser()

3. 소유권 + draft 상태 확인 (중복 완료 방지)
   → status === "complete" → "이미 완료된 후기입니다"

4. 원자적 상태 전환:
   submissions UPDATE SET
     status = 'complete',
     one_line_review = $1,
     tips = $2,
     submitted_at = NOW(),
     step_completed = 3
   WHERE id = $3
     AND user_id = $4
     AND status = 'draft'  ← 핵심: draft만 complete 허용
   .select("id").single()

5. 콤보 추출 + 기출풀 자격 확인 (try-catch — 실패해도 후기는 유지):
   submission_questions 조회 (submission_id)
   → extractCombos(questions) → submission_combos INSERT (0~5개)
   → checkExamApproval() 헬퍼 호출 (기출풀 자격 판정)

6. 크레딧 보상:
   → processCredits() 헬퍼 호출 (25일 룰)

7. revalidatePath("/reviews")
8. 반환: { creditGranted, nextCreditDate }
```

#### checkExamApproval 헬퍼

```typescript
async function checkExamApproval(supabase, submissionId, questions: SubmissionQuestionRow[])
```

기출풀 자격을 자동 판정하는 헬퍼:

```
1. self_intro 제외한 질문 필터
2. question_id 존재 + is_not_remembered=false인 유효 질문 추출
3. 유효 질문 < 14개 → exam_approved = 'not_applicable'
4. 유효 질문의 survey_type 조회 (questions 테이블)
5. 5세트(general_1~3, roleplay, advance)별 대표 survey_type 판정
6. 선택형 3개 + 공통형 2개 충족 → 'pending' 유지 (관리자 승인 대기)
7. 미충족 → exam_approved = 'not_applicable'
```

`SubmissionQuestionRow` 타입: `{ combo_type, question_id, is_not_remembered }`

#### processCredits 헬퍼

```typescript
async function processCredits(supabase, userId, submissionId)
  → { creditGranted: boolean, nextCreditDate: string | null }
```

25일 룰 크레딧 보상 로직:

```
1. credit_granted=true인 이전 후기 조회
2. creditCount < 2 → 무조건 지급
3. creditCount >= 2 → 마지막 지급일로부터 25일 경과 확인
4. 지급 시: RPC increment_script_credits (SECURITY DEFINER)
5. 성공 시: submissions UPDATE credit_granted = true
```

**에러 처리**:

```
콤보 추출/기출풀 확인 실패 → console.error + 후기는 유지
크레딧 RPC 실패 → console.error + credit_granted = false
→ 재시도 가능 (다음 후기 작성 시 기록 갱신)
```

## RPC 함수

### increment_script_credits

```sql
CREATE OR REPLACE FUNCTION increment_script_credits(p_user_id UUID, p_amount INTEGER)
RETURNS VOID AS $$
BEGIN
  UPDATE user_credits
  SET script_credits = script_credits + p_amount
  WHERE user_id = p_user_id;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

- **SECURITY DEFINER**: 함수 소유자(postgres) 권한으로 실행 → RLS 우회
- 이유: user_credits 직접 UPDATE는 RLS로 차단됨
- 호출: `completeSubmission()` → 크레딧 보상 시

## 콤보 추출 유틸리티

> `lib/utils/combo-extractor.ts`

### extractCombos

```typescript
function extractCombos(questions: QuestionItem[]): ExtractedCombo[]
```

**COMBO_RANGES 정의**:

```typescript
const COMBO_RANGES = [
  { type: "general_1",  start: 2,  end: 4,  count: 3, excludeTopics: true },
  { type: "general_2",  start: 5,  end: 7,  count: 3, excludeTopics: true },
  { type: "general_3",  start: 8,  end: 10, count: 3, excludeTopics: true },
  { type: "roleplay",   start: 11, end: 13, count: 3, excludeTopics: false },
  { type: "advance",    start: 14, end: 15, count: 2, excludeTopics: true },
];
```

**EXCLUDED_TOPICS**:

```typescript
const EXCLUDED_TOPICS = ["공원", "카페", "하이킹"];
```

**isValidForCombo 검증**:

```typescript
function isValidForCombo(q: QuestionItem, excludeTopics: boolean): boolean {
  if (!q.question_id) return false;           // master question 필수
  if (q.is_not_remembered) return false;       // 기억해야 함
  if (q.custom_question_text) return false;    // 직접 입력 불가
  if (excludeTopics && EXCLUDED_TOPICS.includes(q.topic)) return false;
  return true;
}
```

**추출 과정**:

```
각 COMBO_RANGE에 대해:
  1. 해당 question_number 범위 질문 필터링
  2. 질문 수 = count 확인
  3. 모든 질문이 isValidForCombo() 통과
  4. topic 추출 → 유니크 → 쉼표 연결
  5. question_ids 추출
  6. ExtractedCombo 생성

→ 최대 5개 반환 (유효하지 않은 콤보 스킵)
```

**안전 설계**:

```
- try-catch 전체 감싸기 → 추출 실패 시 빈 배열 반환
- 콤보 0개도 정상 (후기는 항상 저장)
- 콤보는 빈도 분석 "부가 데이터" — 필수 아님
```

## Zod 검증 스키마

> `lib/validations/reviews.ts`

### step1Schema

```typescript
const step1Schema = z.object({
  // 시험 정보
  exam_date: z.string().min(1),
  exam_difficulty: z.string().min(1),
  pre_exam_level: z.enum(PRE_EXAM_LEVELS),
  achieved_level: z.string(),  // 'unknown' 허용

  // 시험 배경
  exam_purpose: z.enum(EXAM_PURPOSES),
  study_methods: z.array(z.enum(STUDY_METHODS)).min(1),
  prep_duration: z.enum(PREP_DURATIONS),
  attempt_count: z.enum(ATTEMPT_COUNTS),

  // 체감 후기
  perceived_difficulty: z.enum(PERCEIVED_DIFFICULTIES),
  actual_duration: z.enum(ACTUAL_DURATIONS),

  // 서베이
  used_recommended_survey: z.boolean(),

  // 커스텀 서베이 (조건부)
  survey_occupation: z.string().nullable(),
  survey_student: z.string().nullable(),
  survey_course: z.string().nullable(),
  survey_housing: z.string().nullable(),
  survey_leisure: z.array(z.string()),   // buildStep1Fields()에서 .join(",")으로 TEXT 저장
  survey_hobbies: z.array(z.string()),
  survey_sports: z.array(z.string()),
  survey_travel: z.array(z.string()),
})
.refine((data) => {
  if (data.used_recommended_survey) return true;
  // 커스텀: 라디오 4개 필수
  if (!data.survey_occupation || !data.survey_student ||
      !data.survey_course || !data.survey_housing) return false;
  // 체크박스 합산 12개 이상
  const checkboxTotal =
    (data.survey_leisure?.length ?? 0) +
    (data.survey_hobbies?.length ?? 0) +
    (data.survey_sports?.length ?? 0) +
    (data.survey_travel?.length ?? 0);
  return checkboxTotal >= 12;
});
```

### /reviews 페이지

```typescript
// app/(dashboard)/reviews/page.tsx
export default async function ReviewsPage() {
  const [statsAndFrequency, mySubmissions, publicReviews] = await Promise.all([
    getStatsAndFrequency(),
    getMySubmissions(),
    getPublicReviews({ page: 1, limit: 10 }),
  ]);

  // 완료 후기 ID 추출 → 일괄 상세 조회
  const completedIds = mySubmissions
    .filter(s => s.status === "complete")
    .map(s => s.id);
  const submissionDetails = completedIds.length > 0
    ? await getSubmissionsWithQuestionsBatch(completedIds)
    : {};

  return (
    <ReviewsContent
      initialStats={statsAndFrequency.stats}
      initialFrequency={statsAndFrequency.frequency}
      initialSubmissions={mySubmissions}
      initialPublicReviews={publicReviews}
      initialSubmissionDetails={submissionDetails}
    />
  );
}
```

## 에러 처리 패턴

| 상황 | 처리 |
|------|------|
| 인증 실패 | `throw new Error("로그인이 필요합니다")` |
| draft 없음 (이어쓰기) | `throw new Error("작성 중인 후기를 찾을 수 없습니다")` |
| complete 삭제 시도 | `throw new Error("완료된 후기는 삭제할 수 없습니다")` |
| 중복 complete | UPDATE 0건 (이미 complete) → 에러 없이 반환 |
| 콤보 추출 실패 | try-catch → console.error → 후기는 유지 |
| 크레딧 RPC 실패 | try-catch → console.error → credit_granted=false |
| Supabase 쿼리 실패 | `if (error) throw error` → 클라이언트에 에러 전파 |
