# 01. 전체 흐름

> 시험후기 모듈은 OPIc 응시자의 시험 경험을 수집하여 빈도 분석 데이터를 생성하는 모듈이다.
> 3개 탭(빈도 분석 · 후기 제출 · 시험 후기)으로 구성되며, 3단계 위저드로 후기를 작성한다.

---

## 사용자 시나리오

### Phase 1: 빈도 분석 조회 (비로그인 가능)

```
빈도 분석 탭 진입
 → 통계 요약 카드 3개 (총 후기 | 분석 주제 | 참여자)
 → 카테고리 서브탭 (일반 | 롤플레이 | 어드밴스) 선택
 → 주제별 빈도 리스트 표시
 → 주제 클릭 → 질문별 빈도 상세
```

### Phase 2: 후기 제출 (로그인 필수)

```
후기 제출 탭 진입
 → [Step 1] 시험 정보 + 설문 입력
   └─ 시험 날짜, 난이도, 등급, 응시 배경, 체감 후기, 서베이
 → [Step 2] 출제 질문 14개 선택
   └─ 5개 콤보별로 주제 → 질문 순차 선택
 → [Step 3] 한줄 후기 + 팁 작성 → 완료
   └─ 콤보 자동 추출 + 크레딧 보상 (25일 룰)
```

### Phase 3: 공개 후기 조회 (비로그인 가능)

```
시험 후기 탭 진입
 → 등급별 필터 (선택)
 → 공개 후기 리스트 (등급 + 날짜 + 한줄 후기)
 → 페이지네이션 (10개씩)
```

### Phase 4: 후기 관리 (로그인 필수)

```
후기 제출 탭 → 내 후기 목록
 → Draft: 이어쓰기 / 삭제
 → Complete: 상세보기 / 등급 수정
```

## 데이터 흐름도

```
[Step 1: 시험 정보 입력]
    │
    │  createDraft() SA — 기존 draft 삭제 → INSERT
    ▼
submissions (status='draft', step_completed=1)
    │
    │
[Step 2: 출제 질문 14개]
    │
    │  saveQuestions() SA — 15행 INSERT (Q1 자동 추가)
    ▼
submission_questions (question_number 1~15)
submissions (step_completed=2)
    │
    │
[Step 3: 한줄 후기 + 완료]
    │
    │  completeSubmission() SA
    │  ├─ UPDATE status='complete'
    │  ├─ extractCombos() → submission_combos INSERT (0~5개)
    │  ├─ checkExamApproval() → 기출풀 자격 판정 (선택3:공통2 → pending 유지, 미충족 → not_applicable)
    │  └─ processCredits() → 크레딧 보상 (25일 룰) → RPC increment_script_credits
    ▼
submissions (status='complete', exam_approved='pending'|'not_applicable', credit_granted=true/false)
submission_combos (combo_type × topic × question_ids[])
user_credits (script_credits +2)


[빈도 분석 탭]
    │
    │  getStatsAndFrequency() SA — 1회 통합 조회
    │  getQuestionFrequency() SA — 주제별 상세
    ▼
submission_combos 집계 → 주제별 빈도
submission_questions JOIN questions → 질문별 빈도


[공개 후기 탭]
    │
    │  getPublicReviews() SA — 페이지네이션
    ▼
submissions (status='complete', source≠'admin') → 등급별 필터 → 공개 리스트
```

## 상태 전이

### submissions.status

```
draft (createDraft로 생성)
  │
  ├─ [이어쓰기] ← updateDraft / saveQuestions
  │
  ├─ [삭제] → 물리 삭제 (deleteSubmission, draft만 가능)
  │
  │ completeSubmission()
  ▼
complete (영구 — 삭제 불가)
  │
  ├─ [등급 수정만 가능] ← updateGrade() (Zod 검증)
  │
  └─ exam_approved: pending → approved (관리자) / not_applicable (자동)
```

### submissions.step_completed

```
1 (Step 1 저장 완료)
  │ saveQuestions()
  ▼
2 (Step 2 저장 완료)
  │ completeSubmission()
  ▼
3 (완료)
```

### Draft 생존 규칙

- **1개만 허용**: `createDraft()` 시 기존 draft 삭제 후 새로 생성
- **이어쓰기**: URL 파라미터 `?resume={id}`로 draft 복원
- **삭제**: draft만 가능 (complete 후기는 삭제 불가 — 크레딧 악용 방지 + 빈도 데이터 보존)

## 타이밍 & 성능

| 단계 | 소요 | 비고 |
|------|------|------|
| Step 1 저장 | ~200ms | draft INSERT 1건 |
| Step 2 저장 | ~300ms | 15개 INSERT + UPDATE 1건 |
| Step 3 완료 | ~500ms | UPDATE + combo INSERT + RPC 크레딧 |
| 빈도 분석 초기 로드 | ~400ms | 서버 사전로드 (Promise.all) |
| 주제별 질문 빈도 | ~200ms | prefetch 완료 시 0ms (캐시 히트) |
| 공개 후기 조회 | ~300ms | 페이지네이션 (10건) |

## 소리담과의 차이점

| 항목 | 소리담 | 오픽톡닥 |
|------|--------|----------|
| 후기 구조 | 단일 폼 제출 | 3단계 위저드 (draft → complete) |
| 이어쓰기 | 없음 | draft 복원 |
| 서베이 | 매번 수동 입력 | 추천 서베이 원클릭 |
| 콤보 추출 | 수동 | 자동 (extractCombos) |
| 크레딧 보상 | 없음 | 25일 룰 스크립트 크레딧 |
| 빈도 분석 | 별도 페이지 | 같은 페이지 탭 |
| 공개 범위 | 전체 공개 | complete만 공개 (RLS) |
| 질문 선택 | 자유 텍스트 | master questions DB + 커스텀 혼합 |
| 삭제 정책 | 자유 삭제 | complete 삭제 불가 (데이터 보존) |

## 모듈 간 연결

```
시험후기 ──[콤보 데이터]──→ 빈도 분석 (자체)
         ──[크레딧 +2]──→ 스크립트 (user_credits.script_credits)
         ──[등급 이력]──→ 대시보드 (통계 카드)
```

- **빈도 분석**: `submission_combos` 집계 → 주제별 출제 빈도
- **스크립트 크레딧**: 후기 완료 시 `script_credits +2` (25일 룰)
- **대시보드**: 총 후기 수, 최근 등급 표시

## queryKey 매핑

| queryKey | staleTime | 용도 |
|----------|-----------|------|
| `["review-frequency"]` | 5분 | 빈도 분석 데이터 |
| `["my-submissions"]` | 5분 | 내 후기 목록 |
| `["public-reviews", levelFilter]` | 5분 | 공개 후기 (필터별) |
| `["question-frequency", topic]` | 5분 | 주제별 질문 빈도 (prefetch) |
| `["submission-detail", submissionId]` | Infinity | 완료 후기 상세 (변경 없음) |
| `["user-credits", userId]` | 5분 | 크레딧 잔량 (후기 완료 시 invalidate) |
