# 03. 빈도 분석 + 공개 후기

> 빈도 분석 탭: 콤보 데이터 → 주제별/질문별 출제 빈도 시각화
> 시험 후기 탭: complete 후기 공개 리스트 (등급별 필터)

## 빈도 분석 탭

```
  └─ 통계: 총 후기 수, 참여자 수
```

### 카테고리 서브탭

| 카테고리 | 포함 combo_type | 설명 |
|----------|----------------|------|
| 일반 | general_1, general_2, general_3 | Q2~Q10 일반 콤보 3개 합산 |
| 롤플레이 | roleplay | Q11~Q13 |
| 어드밴스 | advance | Q14~Q15 |

### SA: getStatsAndFrequency()

```typescript
getStatsAndFrequency()
  → Promise.all([
      // 1. 통계
      supabase.from("submissions")
        .select("id", { count: "exact" })
        .eq("status", "complete"),

      // 2. 참여자 수
      supabase.from("submissions")
        .select("user_id")
        .eq("status", "complete"),

      // 3. 빈도 데이터
      supabase.from("submission_combos")
        .select("combo_type, topic")
    ])

  → 주제별 빈도 집계 (topic × combo_type)
  → 카테고리별 그루핑
```

### SA: getQuestionFrequency()

```typescript
getQuestionFrequency({ topic, comboTypes })
  → submission_combos
    .select("question_ids")
    .in("combo_type", comboTypes)
    .ilike("topic", `%${topic}%`)

  → question_ids 배열 펼치기
  → 빈도 카운트
  → questions 테이블 JOIN (question_short, question_english 등)
  → 빈도 내림차순 정렬
```

### 캐싱 전략

```
["review-frequency"]            → staleTime: 5분 (서버 사전로드)
["question-frequency", topic]   → staleTime: 5분 (주제별 prefetch)
```

- **서버 사전로드**: 페이지 진입 시 `getStatsAndFrequency()` 실행 → `initialData` 전달
- **주제별 Prefetch**: 초기 로드 시 모든 주제 × 카테고리 조합 prefetch (클릭 시 즉시 표시)

### "더 보기" 패턴

```typescript
const [displayCount, setDisplayCount] = useState(10);

// 더 보기 클릭 → displayCount 증분
const visibleItems = frequencyData.slice(0, displayCount);
```

- 전체 데이터를 한 번에 로드 → 클라이언트에서 slice
- 실제 페이지네이션 아님 (데이터량 적음)

### SA: getPublicReviews()

```typescript
getPublicReviews({ page, limit, levelFilter? })
  → submissions
    .select("*, count")
    .eq("status", "complete")
    .eq("achieved_level", levelFilter)  // optional
    .order("submitted_at", { ascending: false })
    .range(offset, offset + limit - 1)

  → { reviews: Submission[], total: number }
```

### 캐싱 전략

```
["public-reviews", levelFilter]  → staleTime: 5분
```

- 필터 변경 시 새 queryKey → 캐시 미스 → 서버 조회
- 같은 필터 재선택 시 캐시 히트

### RLS 규칙

```sql
-- 비인증 사용자도 complete 후기 조회 가능
CREATE POLICY "submissions_select_public"
  ON submissions FOR SELECT
  USING (status = 'complete');

-- 본인 후기는 모든 상태 조회 가능
CREATE POLICY "submissions_select_own"
  ON submissions FOR SELECT
  USING (auth.uid() = user_id);
```


## 데이터 집계 흐름

```
[후기 제출 완료]
    │
    │ completeSubmission()
    │ └─ extractCombos() → submission_combos INSERT
    │
    ▼
submission_combos (topic × combo_type × question_ids)
    │
    │ getStatsAndFrequency() — 빈도 탭 초기 로드
    │ getQuestionFrequency() — 주제별 상세
    │
    ▼
빈도 분석 탭
    ├─ 통계 카드: 총 후기 | 분석 주제 | 참여자
    ├─ 카테고리별: 일반 | 롤플레이 | 어드밴스
    ├─ 주제별 빈도 리스트 (빈도 내림차순)
    └─ 주제 → 질문별 빈도 상세
```

### 빈도 산출 공식

```
주제 빈도 = 해당 주제가 포함된 combo 수
질문 빈도 = 해당 question_id가 question_ids[]에 포함된 combo 수
```

- 한 후기에서 같은 주제가 다른 콤보에 나올 수 있음 (각각 카운트)
- "기억 안남", "직접 입력" 질문은 콤보에 포함되지 않음 (question_id 없음)
