# 06. DB 스키마

> 테이블이 뭐가 있고, 데이터가 어떻게 흘러가는지

---

## 테이블 전체 목록

| # | 테이블 | 역할 | 마이그레이션 |
|---|--------|------|-------------|
| 1 | `mock_test_sessions` | 세션 관리 (모드, 상태, 만료) | 011 |
| 2 | `mock_test_answers` | 답변 기록 + 평가 큐 (eval_status 추적) | 011 + 012 |
| 3 | `mock_test_evaluations` | 개별 문항 평가 (체크박스 + 코칭) | 011 + 012 |
| 4 | `mock_test_reports` | 종합 리포트 (등급 + FACT + GPT 리포트 + 성장) | 011 + 012 + 016 |
| 5 | `mock_test_eval_settings` | 관리자 설정 (단일 행) | 011 |
| 6 | `evaluation_prompts` | 평가 프롬프트 템플릿 (9행) | 011 |

관련 테이블:
- `submissions`: 기출 세트 원본 (FK 참조)
- `user_credits`: 크레딧 차감/환불 (RPC)
- Storage 버킷: `mock-test-recordings`

## 테이블 관계도

```
submissions (기출 세트)
    │
    │ FK: submission_id
    ▼
mock_test_sessions ─────────────────────────────────────┐
    │                                                    │
    │ FK: session_id (TEXT, UNIQUE)                      │ FK: session_id
    ▼                                                    ▼
mock_test_answers ◀───── 평가 데이터 ─────▶ mock_test_evaluations
    │                                           │
    │ eval_status 추적                           │ question_number, session_id
    │                                           │
    └───────────── 모든 평가 완료 시 ──────────▶ mock_test_reports
                                                    │
                                                    │ coaching_report (GPT 9섹션)
                                                    │ growth_comparison (규칙 기반)
                                                    │ growth_analysis (GPT 생성)
                                                    │ tutoring_prescription (자동 변환)
                                                    ▼
                                              튜터링 모듈 연결

mock_test_eval_settings (단일 행, 관리자)
evaluation_prompts (9행, 프롬프트 템플릿)
```

## 1. mock_test_sessions

세션의 생애주기를 관리하는 마스터 테이블.

### 컬럼 상세

| 컬럼 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `id` | UUID | `gen_random_uuid()` | PK |
| `session_id` | TEXT | — | UNIQUE. 'mt_' + 8자리 hex (예: `mt_8099015f`) |
| `user_id` | UUID | — | FK → auth.users. NOT NULL |
| `submission_id` | INTEGER | — | FK → submissions. 기출 세트 원본 |
| `mode` | TEXT | — | `training` \| `test`. NOT NULL |
| `status` | TEXT | `active` | `active` \| `completed` \| `expired` |
| `question_ids` | TEXT[] | — | Q2~Q15 question_id 배열 (14개). NOT NULL |
| `current_question` | SMALLINT | 1 | 현재 진행 문항 (1~15) |
| `total_questions` | SMALLINT | 15 | 총 문항 수 |
| `holistic_status` | TEXT | `pending` | `pending` \| `processing` \| `completed` \| `failed` |
| `report_retry_count` | SMALLINT | 0 | Stage C 재시도 횟수 (최대 3) |
| `report_error` | TEXT | NULL | 리포트 생성 오류 메시지 |
| `started_at` | TIMESTAMPTZ | `NOW()` | 세션 시작 시각 |
| `completed_at` | TIMESTAMPTZ | NULL | 세션 완료 시각 |
| `expires_at` | TIMESTAMPTZ | — | 만료 시각. training: +72h, test: +90min |

### 상태 전이

```
active → completed  (Q15 제출 후 completeSession)
active → expired    (만료 시간 경과 시 expireSession)
```

### 인덱스

| 인덱스 | 컬럼 | 용도 |
|--------|------|------|
| `idx_mock_sessions_user_status` | `(user_id, status)` | 활성 세션 조회, 이력 조회 |
| `idx_mock_sessions_expires_at` | `(expires_at)` | 만료 세션 일괄 처리 |
| `idx_mock_sessions_submission` | `(submission_id)` | 기출 사용 이력 확인 |

### RLS 정책

| 정책 | 작업 | 조건 |
|------|------|------|
| `mock_sessions_select_own` | SELECT | `auth.uid() = user_id` |
| `mock_sessions_insert_own` | INSERT | `auth.uid() = user_id` |
| `mock_sessions_update_own` | UPDATE | `auth.uid() = user_id` |

## 2. mock_test_answers

답변 기록 + 평가 큐 역할을 겸한다. `eval_status`로 평가 파이프라인 진행 상태를 추적.

### 컬럼 상세

| 컬럼 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `id` | UUID | `gen_random_uuid()` | PK |
| `session_id` | TEXT | — | FK → mock_test_sessions.session_id. CASCADE |
| `question_number` | SMALLINT | — | 1~15 |
| `question_id` | TEXT | NULL | Q1은 NULL (시스템 질문) |
| `audio_url` | TEXT | NULL | Storage 음성 파일 URL |
| `audio_duration` | NUMERIC(8,2) | NULL | 음성 길이 (초) |
| `transcript` | TEXT | NULL | Whisper STT 결과 |
| `word_count` | INTEGER | NULL | 단어 수 |
| `filler_word_count` | INTEGER | 0 | 필러 단어 수 |
| `long_pause_count` | INTEGER | 0 | 긴 침묵 횟수 |
| `pronunciation_assessment` | JSONB | NULL | Azure Speech SDK 결과 |
| `eval_status` | TEXT | `pending` | 7단계 (아래 참조) |
| `eval_retry_count` | SMALLINT | 0 | 재시도 횟수 |
| `eval_error` | TEXT | NULL | 오류 메시지 |
| `skipped` | BOOLEAN | false | 건너뛴 문항 |
| `wpm` | NUMERIC(6,2) | NULL | Words Per Minute (v3 추가) |
| `meta_only` | BOOLEAN | false | 메타 데이터만 있는 스킵 (v3 추가) |
| `unfinished_end` | BOOLEAN | false | 미완결 종료 (v3 추가) |
| `filler_ratio` | NUMERIC(5,4) | NULL | 필러 비율 (v3 추가) |
| `created_at` | TIMESTAMPTZ | `NOW()` | 생성 시각 |

### eval_status 7단계

```
pending → processing → stt_completed → evaluating → judge_completed → completed
   │          │               │                                          │
   └── skipped ◀──────────────┘                                         │
   └── failed ◀──────────────────────────────────────────────────────────┘
```

| 상태 | 설정 시점 | 의미 |
|------|----------|------|
| `pending` | submitAnswer | 답변 제출됨, 평가 대기 |
| `processing` | Stage A 시작 | STT + 발음 평가 중 |
| `stt_completed` | Stage A 완료 | STT 완료, 체크박스 평가 대기 |
| `evaluating` | Stage B-1 시작 | GPT 체크박스 판정 중 |
| `judge_completed` | Stage B-1 완료 | 판정 완료, 코칭 생성 대기 |
| `completed` | Stage B-2 완료 | 전체 평가 완료 |
| `failed` | 에러 발생 시 | 평가 실패 (재시도 소진) |
| `skipped` | Q1 또는 건너뛰기 | 평가 대상 아님 |

### UNIQUE 제약

```sql
UNIQUE(session_id, question_number)
```

### 인덱스

| 인덱스 | 컬럼 | 조건 | 용도 |
|--------|------|------|------|
| `idx_mock_answers_session` | `(session_id)` | — | 세션별 답변 조회 |
| `idx_mock_answers_eval_status` | `(eval_status)` | `!= completed AND != skipped` | 미완료 답변 확인 |

### RLS 정책

| 정책 | 작업 | 조건 |
|------|------|------|
| `mock_answers_select_own` | SELECT | 서브쿼리: 본인 세션의 답변 |
| `mock_answers_insert_own` | INSERT | 서브쿼리: 본인 세션에만 |
| `mock_answers_update_own` | UPDATE | 서브쿼리: 본인 세션만 |

> EF(Edge Function)는 `service_role` 키로 RLS를 우회하여 직접 UPDATE.

## 데이터 흐름 요약

```
[사용자 답변 제출]
   │
   ├─ submitAnswer() → mock_test_answers INSERT (eval_status: pending)
   │                  → Storage 업로드 (WAV)
   │
   ├─ Stage A (process) → mock_test_answers UPDATE
   │   transcript, wpm, pronunciation_assessment, filler_count, long_pause_count
   │   eval_status: pending → processing → stt_completed (또는 skipped)
   │
   ├─ Stage B-1 (eval-judge) → mock_test_evaluations INSERT
   │   checkboxes, pass_rate, task_fulfillment, feedback_branch, priority_prescription
   │   mock_test_answers.eval_status: stt_completed → evaluating → judge_completed
   │
   ├─ Stage B-2 (eval-coach) → mock_test_evaluations UPDATE
   │   coaching_feedback, model, tokens_used
   │   mock_test_answers.eval_status: judge_completed → completed
   │
   └─ Stage C (report) → mock_test_reports INSERT (또는 UPDATE)
       규칙엔진 결과 (final_level, FACT 점수, 체크박스 집계)
       coaching_report (GPT 9섹션)
       growth_comparison + growth_analysis (2회차+)
       tutoring_prescription (자동 변환)
       mock_test_sessions.holistic_status: pending → completed
```
