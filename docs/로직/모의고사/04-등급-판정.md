# 04. 등급 판정

> 규칙엔진 7-Step이 뭐고, 등급은 어떻게 결정되는지, FACT 점수는 어떻게 나오는지

---

## V7 규칙엔진 개요

OPIc 시험의 공식 채점 기준을 코드로 구현한 7단계 판정 알고리즘.
체크박스 74개의 pass/fail 결과를 입력으로, 최종 등급(NH~AL)을 출력한다.

### 7등급 체계

```
NH → IL → IM1 → IM2 → IM3 → IH → AL
 1     2     3      4      5     6    7
```

### 입력 데이터

규칙엔진은 Stage C(mock-test-report)에서 호출되며, 모든 문항의 평가 결과를 받는다:

```typescript
interface EvaluationInput {
  question_number: number;        // 2~15 (Q1 제외)
  question_type: string;          // DB의 question_type_eng
  checkbox_type: "INT" | "ADV" | "AL";
  checkboxes: Record<string, { pass: boolean; evidence?: string }>;
  skipped: boolean;
  pronunciation_assessment?: {
    accuracy_score?: number;
    prosody_score?: number;
    fluency_score?: number;
  } | null;
}
```

## 체크박스 74개

### 분류 체계

| 등급 | 개수 | 체크박스 ID 범위 |
|------|------|----------------|
| **INT** (Intermediate) | 20 | INT-1 (3), INT-2 (3), INT-3 (2), INT-4 (12) |
| **ADV** (Advanced) | 42 | ADV-1 (11), ADV-2 (11), ADV-3 (3), ADV-4 (4), ADV-5 (13) |
| **AL** (Advanced Low) | 12 | AL-14 (6), AL-15 (6) |

### question_type → 체크박스 세트 매핑

| question_type | 체크박스 세트 | 개수 | checkbox_type |
|---------------|-------------|------|--------------|
| description, routine | INT (INT-3 제외) | 18 | INT |
| rp_11 | INT (INT-3 포함) | 20 | INT |
| comparison, past_childhood, past_special, past_recent | ADV (ADV-4 제외) | 38 | ADV |
| rp_12 | ADV (ADV-4 포함) | 42 | ADV |
| adv_14 | AL-14 | 6 | AL |
| adv_15 | AL-15 | 6 | AL |

### 누적 로직 (Cumulative)

**INT-2 (담화 수준)**: 상위 pass → 하위 자동 pass

```
INT-2-3 pass → INT-2-1, INT-2-2도 자동 pass
INT-2-2 pass → INT-2-1도 자동 pass
```

**ADV-2-SP (발화 길이)**: 상위 pass → 하위 자동 pass

```
ADV-2-SP5 pass → SP1~SP4 자동 pass
ADV-2-SP4 pass → SP1~SP3 자동 pass
ADV-2-SP3 pass → SP1~SP2 자동 pass
ADV-2-SP2 pass → SP1 자동 pass
```

### 체크박스 집계 방식

각 체크박스는 여러 문항에서 평가될 수 있다 (같은 체크박스가 여러 문항에 매핑).

```
각 체크박스별:
  pass_count = 해당 체크박스가 pass된 문항 수
  total = 해당 체크박스가 평가된 문항 수
  pass_rate = pass_count / total
  final_pass = pass_rate ≥ 0.80 (checkbox_pass_threshold)

누적 로직 적용 후:
  전체 통과율 = final_pass인 체크박스 수 / 전체 체크박스 수
```

## 7-Step 상세

### Step 1: 체크박스 집계

```
14문항의 체크박스 → INT/ADV/AL 3그룹으로 분류
  → 각 체크박스별 pass_count/total 집계
  → pass_rate ≥ 0.80 → final_pass
  → 누적 로직 적용
  → 그룹별 전체 통과율 계산
```

### Step 2: Floor 판정 (INT 통과율 기반)

INT 체크박스 전체 통과율로 "최소 등급"을 결정한다.

```
INT 통과율 < 0.45 → NH   (BELOW_MINIMUM)
INT 통과율 < 0.65 → IL   (UNSTABLE)
INT 통과율 < 0.75 → IM1  (LOWER_RANGE)
INT 통과율 < 0.95 → IM2  (STABLE)
INT 통과율 ≥ 0.95 → IM3  (UPPER_RANGE)
```

### Step 3: Ceiling 판정 (ADV 통과율 기반)

ADV 체크박스 전체 통과율로 "상위 역량"을 판정한다.

```
ADV 통과율 < 0.50 → BROKE_DOWN (Random)     ← 무너짐
ADV 통과율 < 0.70 → BROKE_DOWN (Emerging)   ← 무너짐
ADV 통과율 < 0.90 → RESPOND_NOT_SUSTAIN     ← 시도는 하나 유지 못함
ADV 통과율 ≥ 0.90 → SUSTAIN_ADVANCED        ← 고급 역량 유지
```

### Step 4: Sympathetic Listener (이해 용이성)

발음 평균과 ADV-5(이해 용이성 13개) 통과율을 복합 점수로 계산한다.

```
복합 점수 = 발음평균 × 0.60 + ADV-5통과율(0~100) × 0.40

복합 점수 < 50 → "Required"              ← 듣는 사람 배려 필수
복합 점수 < 70 → "Required_at_times"     ← 가끔 필요
복합 점수 < 85 → "Not_required_but_close" ← 거의 필요 없음
복합 점수 ≥ 85 → "Not_required"           ← 필요 없음
```

발음 평균 = (accuracy + prosody + fluency) / 3

### Step 5: Q12 게이트키퍼

Q12(rp_12, 상황 대응 롤플레이)의 ADV-4(돌발 상황 대처) 통과율을 확인한다.
IH/AL 등급의 게이트키퍼 역할.

```
Q12 스킵 → "SKIPPED" (IH/AL 불가)
Q12 미평가 → "SKIPPED"
ADV-4 통과율 ≥ 0.50 → "PASSED"
ADV-4 통과율 < 0.50 → "FAILED"
```

### Step 6: AL 판정

AL 등급 달성 가능 여부를 판정한다.

```
Ceiling = BROKE_DOWN → "NOT_ELIGIBLE"
Q12 게이트키퍼 SKIPPED/FAILED → "NOT_ELIGIBLE"
Q14 또는 Q15 스킵 → "AL_FAILED_SKIP"
AL 게이트키퍼 체크박스 미통과 → "AL_FAILED_GATEKEEPER"
   → 필수 체크박스: AL-14-PS, AL-14-CD, AL-15-AS, AL-15-CD
AL 통과율 ≥ 0.70 → "AL_CONFIRMED"
AL 통과율 < 0.70 → "AL_FAILED_SCORE"
```

### Step 7: 최종 등급 결합

Floor, Ceiling, Sympathetic, Q12, AL 판정을 종합하여 최종 등급을 결정한다.

```
Floor가 NH 또는 IL → 그대로 반환 (다른 판정 무시)

Q12 게이트키퍼 SKIPPED/FAILED:
  → 기본: Floor 등급 유지
  → ADV < 40%: Floor에서 1단계 하향
  → 최대 IM3 캡 (IH/AL 불가)

Ceiling = SUSTAIN_ADVANCED (ADV ≥ 90%):
  → AL 확정 + Sympathetic Not_required + INT ≥ 90% → AL
  → INT ≥ 90% + Sympathetic ≥ Required_at_times → IH
  → 나머지 → Floor 등급

Ceiling = RESPOND_NOT_SUSTAIN (ADV 70~90%):
  → AL 확정 + Sympathetic Not_required + INT ≥ 90% → AL
  → INT ≥ 90% + Sympathetic ≥ Required_at_times → IH
  → 나머지 → Floor 등급

Ceiling = BROKE_DOWN (ADV < 70%):
  → ADV < 40%: Floor에서 1단계 하향
  → 나머지: Floor 등급 유지
```

### 등급 판정 결과 예시

```
INT 통과율: 0.85 → Floor: IM2
ADV 통과율: 0.78 → Ceiling: RESPOND_NOT_SUSTAIN
Sympathetic: Not_required_but_close
Q12: PASSED
AL: NOT_ELIGIBLE (Ceiling 부족)
→ INT ≥ 0.90 아님 → IH 불가
→ 최종 등급: IM2

INT 통과율: 0.92 → Floor: IM2
ADV 통과율: 0.91 → Ceiling: SUSTAIN_ADVANCED
Sympathetic: Not_required
Q12: PASSED
AL: AL_CONFIRMED (AL 통과율 0.75)
→ AL 확정 + Not_required + INT 0.92 ≥ 0.90 → AL
→ 최종 등급: AL
```

## FACT 점수 계산

FACT = Functions, Accuracy, Context, Text type
각 0~10점, 총점 = (F+A+C+T) × 2.5 = 최대 100점

### F — Functions & Tasks (16개 체크박스)

```
매핑:
  INT-1-1~3 (어휘/청해/문장)
  INT-3-1~2 (질문 구성) — rp_11 전용
  ADV-1-D1~D3 (시제별 묘사)
  ADV-1-N1~N3 (시제별 서사)
  ADV-4-CR1~CR3, ADV-4-CD1 (돌발 대처) — rp_12 전용

계산: (pass된 체크박스 수 / 16) × 10 (반올림)
```

### A — Accuracy (31개 체크박스 + 발음 보정)

```
매핑:
  INT-4 전체 (유창성/발음/문법/어순 12개)
  ADV-1-PR1~PR3 (서사 문제점)
  ADV-2-WO1~WO3 (어순 문제)
  ADV-5 전체 (이해 용이성 13개)

계산: (pass된 체크박스 수 / 31) × 10 (반올림)
  + 발음 보정:
    avgAccuracy ≥ 80 → +1
    avgAccuracy < 50 → -1
```

### C — Context & Content (5개 체크박스 + 유효 문항 보정)

```
매핑:
  ADV-3-V1~V3 (어휘 범위)
  ADV-1-L1~L2 (묘사 품질)

계산: (pass된 체크박스 수 / 5) × 10 (반올림)
  + 유효 문항 수 보정:
    < 10개 → -2
    < 14개 → -1
```

### T — Text Type (특수 계산)

INT-2(담화 수준)과 ADV-2-SP(발화 길이)의 계단식 점수를 사용한다.

```
INT-2 계단식:
  INT-2-3 pass → 3점
  INT-2-2 pass → 2점
  INT-2-1 pass → 1점

ADV-2-SP 계단식:
  SP5 pass → 9점
  SP4 pass → 7점
  SP3 pass → 6점
  SP2 pass → 4점
  SP1 pass → 2점

기본 점수 = max(INT-2 점수, SP 점수, 1)

CD 보정 (ADV-2-CD1~CD3 연결어):
  CD 통과율 ≥ 60% → +1
  CD 통과율 < 30% → -1

최종: clamp(1, 10)
```

### 총점 계산

```
총점 = (F + A + C + T) × 2.5
최대 = (10 + 10 + 10 + 10) × 2.5 = 100점
상한 클램프: min(100, 계산값)
소수 첫째 자리 반올림
```
