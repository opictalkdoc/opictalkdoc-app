# 05. 결과 화면

> 결과 페이지에 뭐가 보이고, 데이터는 어디서 오는지

---

## 결과 페이지 접근 경로

```
방법 1: 평가 완료 후 자동 이동
   eval-waiting.tsx → holistic_status = 'completed' 감지 → /mock-exam/result/{sessionId}

방법 2: 나의 이력 탭에서 클릭
   history 탭 → 세션 카드 클릭 → /mock-exam/result/{sessionId}
```

### 서버 사전 조회 (page.tsx)

```
/mock-exam/result/[sessionId]/page.tsx:
   Promise.all([
     getSession({ session_id: sessionId }),  // 세션+답변+평가+리포트+질문
     getHistory(),                            // 이전 결과 비교용
   ])
   → 현재 세션 제외 가장 최근 완료 세션 = previousResult
   → ResultPageContent에 initialData + previousResult 전달
```

## 4탭 구조

```
┌────────────────────────────────────────────┐
│  [종합]  [진단]  [문항별]  [성장]           │
├────────────────────────────────────────────┤
│                                            │
│  탭별 콘텐츠 (스크롤 가능)                  │
│                                            │
└────────────────────────────────────────────┘
```

### 탭 상태 관리

- URL 쿼리 파라미터 `?tab=overview|diagnosis|questions|growth`
- `history.replaceState`로 동기화 (새로고침 시 유지)
- TanStack Query: `staleTime: Infinity` (결과는 변경되지 않음)
- 리포트 미완료 시 10초 간격 폴링 (`refetchInterval`)

## 종합 탭 (tab-overview.tsx)

### 데이터 소스

| UI 섹션 | 데이터 소스 | 필드 |
|---------|-----------|------|
| 결과 스냅샷 | `report` + `coaching.snapshot` | `final_level`, `total_score`, `headline`, `diagnosis_tags`, `grade_interpretation` |
| 이전 비교 | `previousResult` | `final_level` → `getLevelDiff()` |
| 왜 이 등급인지 | `coaching.grade_explanation` | `fact_interpretation`, `difficulty_interpretation`, `grade_blockers` |
| 목표 등급 로드맵 | `coaching.roadmap` | `current_to_next`, `next_to_target`, `personal_blockers`, `long_term_goals` |
| 훈련 추천 CTA | `coaching.training_recommendation` | `course_title`, `estimated_daily_minutes`, `session_count`, `focus_areas` |

### UI 구조

```
S1: 결과 스냅샷
   ├─ 등급 배지 (원형, primary 색상)
   │   └─ 등급 상승 시 녹색 TrendingUp 아이콘
   ├─ 등급 해석 텍스트
   ├─ 현재 → 목표 등급 표시
   ├─ FACT 총점 (x.x / 100)
   ├─ 한줄 총평 (headline)
   ├─ 진단 태그 (3~5개 라운드 배지)
   └─ 이전 대비 변화 (±N단계)

S2: 왜 이 등급인지
   ├─ FACT 4축 프로그레스 바 (각 0~10)
   │   └─ 각 축별 해석 문장 (fact_interpretation)
   ├─ 기초/심화 달성도 (int_pass_rate, adv_pass_rate → %)
   ├─ 난이도 해석 (difficulty_interpretation)
   └─ 등급 결정 핵심 사유 (grade_blockers, 번호 리스트)

S4: 목표 등급 로드맵
   ├─ 등급 진행 경로 (현재 → 다음 → 목표)
   ├─ 지금 넘어야 할 장벽 (personal_blockers)
   └─ 장기 목표 (long_term_goals)

CTA: 맞춤 훈련 시작하기
   ├─ 추천 코스명, 일일 훈련 시간, 세션 수
   ├─ 집중 영역 태그 (TYPE_MAP_KO 변환)
   └─ /tutoring?tab=prescription 링크
```

### FACT 점수 표시

```
FACT_LABELS = {
  F: "과제 수행력 (F)",
  A: "정확성 (A)",
  C: "맥락·내용 (C)",
  T: "텍스트 유형 (T)",
}

각 축: 프로그레스 바 + 점수(0~10) + 해석 문장
총점: (F+A+C+T) × 2.5 = 최대 100점
```

## 진단 탭 (tab-diagnosis.tsx)

### 데이터 소스

| UI 섹션 | 데이터 소스 | 타입 |
|---------|-----------|------|
| Top 3 우선 처방 | `coaching.top3_priorities` | `Top3Priority[]` |
| 유형별 진단 맵 | `coaching.question_type_map` | `QuestionTypeMapItem[]` |
| 반복 오류 패턴 | `coaching.recurring_patterns` | `RecurringPattern[]` |
| 말하기 습관/발음 | `coaching.delivery_interpretation` + `report.avg_*_score` | |
| 강점 | `coaching.strengths` | `{label, detail}[]` |

### S3: 먼저 고칠 Top 3

```typescript
interface Top3Priority {
  rank: number;          // 1~3
  area: string;          // task_performance | content_structure | delivery
  label: string;         // "시제 혼용"
  why: string;           // 왜 중요한지
  before: string;        // 교정 전 예시
  after: string;         // 교정 후 예시
  fix: string;           // 교정 원칙
  where: string[];       // 발견 문항 ["Q3", "Q7"]
}
```

영역 한글 매핑:
```
task_performance → 과제수행 (파랑)
content_structure → 내용구조 (보라)
delivery → 전달 (주황)
```

### S5: 유형별 진단 맵

```typescript
interface QuestionTypeMapItem {
  type: string;          // GPT 유형맵 키 (description_detail 등)
  status: string;        // strong | stable | weak | very_weak
  comment: string;       // 한 줄 진단
  priority: boolean;     // 우선 훈련 대상 여부
}
```

상태 색상 매핑:
```
strong → 강함 (초록)
stable → 안정 (파랑)
weak → 약함 (노랑)
very_weak → 매우 약함 (빨강)
```

### S6: 반복 오류 패턴

```typescript
interface RecurringPattern {
  label: string;         // "주어 생략"
  category: string;      // structure | task_performance | delivery_habit | expression | grammar | vocabulary
  severity: string;      // high | medium | low
  frequency: number;     // 반복 횟수
  where: string[];       // 발견 문항
  before: string;        // 오류 예시
  after: string;         // 교정 예시
  why_recurring: string; // 반복 원인
  fix_principle: string; // 교정 원칙
}
```

- 아코디언 UI: 클릭하면 상세 (before→after, 원인, 교정 원칙) 펼침
- 심각도별 색상 바: high(빨강), medium(노랑), low(회색)

### S7: 말하기 습관 & 발음

```
발음 수치:
  정확도 (avg_accuracy_score)
  운율 (avg_prosody_score)
  유창성 (avg_fluency_score)

해석 항목 (delivery_interpretation):
  답변 길이 (duration_comment)
  필러 (filler_comment)
  침묵 (pause_comment)
  발음 (pronunciation_comment)
  전반 전달 (overall_delivery)

강점 (strengths):
  {label, detail}[] — 잘한 점 목록
```

### 데이터 통합 (QuestionData)

```
14문항(Q2~Q15) 각각:
  answer(mock_test_answers) + evaluation(mock_test_evaluations) + question(questions)
  → 통합 데이터 구성:
     questionNumber, isSkipped, passRate, fulfillmentStatus, pronunciationAvg
```

### 막대 색상 기준

```
passRate ≥ 0.8 → 초록
passRate < 0.5 → 빨강
나머지        → 프라이머리
건너뜀        → 회색
```

### 과제충족 아이콘

```
fulfilled → ✓ (초록)
partial   → △ (노랑)
failed    → ✗ (빨강)
skipped   → — (회색)
```

### 5단계 코칭 상세 (DetailPanel)

문항 클릭 시 표시되는 상세 내용. 데이터 소스: `evaluation.coaching_feedback` (CoachingFeedback JSONB)

```
Step 1: 인사이트 + 과제충족
   ├─ one_line_insight (한 줄 핵심 코멘트)
   └─ task_fulfillment
       ├─ status: fulfilled | partial | failed
       ├─ checklist.required: [{item, pass}] (🔴필수)
       └─ checklist.advanced: [{item, pass}] (🟡심화)

Step 2: 최우선 처방
   └─ priority_prescription: [{action, why, example}]

Step 3: 구조 + 교정
   ├─ structure_evaluation: {키: 분석 문장}
   └─ key_corrections: string[] (번호 리스트)

Step 4: 답변 개선
   └─ answer_improvement
       ├─ corrected_version (문법만 수정)
       ├─ better_version (목표 등급 수준)
       └─ what_changed (개선 포인트)

Step 5: 전달 + 습관
   ├─ skill_summary: {영역: 점수(1~5)}
   ├─ pronunciation_assessment: {accuracy, prosody, fluency}
   └─ deep_analysis
       ├─ strengths
       ├─ weaknesses
       ├─ target_gap
       └─ practice_suggestion
```

```
TranscriptWithAudio 컴포넌트:
```

## 성장 탭 (tab-growth.tsx + growth-report.tsx)

### 조건

- **1회차**: "아직 성장 리포트를 볼 수 없습니다" 안내 메시지
- **2회차 이상**: `report.growth_comparison`이 있으면 GrowthReport 렌더링

### 데이터 소스

| 섹션 | 데이터 소스 | 생성 방식 |
|------|-----------|----------|
| 한 줄 요약 | `report.growth_summary` | GPT gpt-4.1-mini |
| 결과 변화 표 | `report.growth_comparison` | 규칙 기반 (EF에서 계산) |
| 등급 변화 원인 | `report.growth_analysis.level_change_reason` | GPT |
| FACT별 해석 | `report.growth_analysis.fact_comments` | GPT |
| 유형별 비교 | `report.growth_comparison.type_comparison` | 규칙 기반 |
| 병목 분석 | `report.growth_analysis.bottleneck` | GPT |
| 추천 행동 | `report.growth_analysis.recommended_actions` | GPT |
| 성장 패턴 | `report.growth_analysis.growth_pattern` + `pattern_message` | 규칙 기반 (3회차+) |

### GrowthComparison (규칙 기반)

```typescript
interface GrowthComparison {
  previous_session_id: string;
  session_count: number;
  previous_level: string;
  level_diff: number;
  previous_total_score: number;
  score_diff: number;
  previous_score_f: number;
  score_f_diff: number;
  previous_score_a: number;
  score_a_diff: number;
  previous_score_c: number;
  score_c_diff: number;
  previous_score_t: number;
  score_t_diff: number;
  type_comparison: {
    type: string;                // question_type
    current_pass_rate: number;
    previous_pass_rate: number | null;
    change: number | null;
  }[];
}
```

### GrowthAnalysis (GPT 생성)

```typescript
interface GrowthAnalysis {
  level_change_reason: string;
  fact_comments: Record<"F"|"A"|"C"|"T", string>;
  bottleneck: {
    primary: string;             // "F" | "A" | "C" | "T"
    reason: string;
  };
  recommended_actions: {
    priority: number;
    action: string;
    training_type?: string;      // epp | forced_variation | timed_practice 등
  }[];
  growth_pattern?: string;       // 3회차+ 전용 (4패턴)
  pattern_message?: string;
}
```

### 성장 패턴 4종 (3회차 이상)

```
f_fast_c_slow    → F(과제수행) 빠르게 성장, C(맥락) 정체
t_unstable       → T(텍스트유형) 점수 불안정 (등락 반복)
a_plateau        → A(정확성) 정체 구간 (변화 ≤0.5)
score_up_level_same → 총점 상승했지만 등급 변화 없음
```

### 7섹션 UI

```
① 한 줄 요약
   → growth_summary (GPT 한 문장)

② 결과 변화 표
   → 등급, 총점, FACT 4축 — 이전 vs 현재 + 변화량 (+N단계, +x.x)
   → DiffBadge: 상승(초록 TrendingUp), 하락(빨강 TrendingDown), 유지(Minus)

③ 등급 변화 원인
   → level_change_reason (GPT 해석)

④ FACT별 해석
   → 4축 각각: 라벨 + diff + 해석 문장
   → FACT_COLORS: F(파랑), A(초록), C(주황), T(보라)

⑤ 유형별 달성률 비교
   → question_type별 프로그레스 바 (이전=점선, 현재=실선)
   → 기본 5개 표시, "+N개 더보기" 토글
   → 고정 표시 순서: description, routine, comparison, past 3종, rp_11, rp_12, adv_14, adv_15

⑥ 현재 병목
   → bottleneck.primary (FACT 축 배지) + reason

⑦ 추천 행동 + 튜터링 CTA
   → 3개 행동 (이번 주 반드시 / 다음 모의고사 전 / 시험장에서 주의)
   → training_type 있으면 튜터링 CTA 버튼 (특정 훈련 바로가기)

[성장 패턴 안내] — 3회차+ 전용
   → pattern + pattern_message 표시
```

## result-summary.tsx (레거시 9섹션 뷰)

> 결과 탭 리디자인 이전의 통합 뷰. 현재는 4탭 구조로 분리되어 각 탭에서 사용.

```
섹션 1: 결과 스냅샷 (= 종합 S1)
섹션 2: 왜 이 등급인지 (= 종합 S2)
섹션 3: 먼저 고칠 Top 3 (= 진단 S3)
```

## 등급 추이 차트 (나의 이력 탭)

> `grade-progress-chart.tsx` — 나의 이력 탭 상단에 표시되는 Recharts 기반 3단 복합 차트

```
┌─────────────────────────────────────────────────┐
│  등급 라인 (계단형 StepAfter)                     │
│  NH ─ IL ─ IM1 ─ IM2 ─ IM3 ─ IH ─ AL           │
│       ●───●                                     │
│           └───●                                  │
├─────────────────────────────────────────────────┤
│  준비도 바 (GRADE_RANGES 기반 선형 보간, 0~100%) │
│  ████████████████████ 78%                       │
├─────────────────────────────────────────────────┤
│  FACT 미니 라인 (F, A, C, T 4줄)                │
│  ─── F ─── A ─── C ─── T                       │
├─────────────────────────────────────────────────┤
│  병목 감지: "현재 병목: 정확성(A)"               │
└─────────────────────────────────────────────────┘
```

- X축: 세션 번호 (1회, 2회, ...)
- 등급 라인: `OPIC_LEVEL_ORDER` 기반 (NH=1 ~ AL=7)
- 준비도: 등급 + 총점으로 선형 보간 계산 (0~100%)
- FACT 미니: 4축 각각 0~10 범위 라인
- 병목: 가장 점수가 낮은 FACT 축을 감지하여 표시
