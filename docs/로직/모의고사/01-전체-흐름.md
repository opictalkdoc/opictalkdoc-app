# 01. 전체 흐름

> 모의고사 처음부터 끝까지의 전체 동작 흐름

---

## 사용자 시나리오

### Phase 1: 시험 준비

```
① 기출 선택
   사용자가 모의고사 탭에 진입하면 승인된 기출 3개가 카드로 표시된다.
   각 카드에는 시험일, 주제 구성, 난이도 힌트가 표시된다.
   → getExamPool() Server Action
   → submissions + submission_questions + questions 조인

② 모드 선택
   기출을 선택하면 훈련/실전 모드를 선택한다.
   ┌────────────────────────────────────────────┐
   │  훈련 모드               실전 모드          │
   │  • 시간제한 없음         • 40분 제한         │
   │  • 질문 텍스트 보임      • 질문 텍스트 숨김   │
   │  • 개별 평가 즉시 확인   • 전체 완료 후 확인  │
   │  • 72시간 이어하기       • 90분 만료         │
   └────────────────────────────────────────────┘

③ 디바이스 테스트
   스피커 확인 (AVA 음성 재생) → 마이크 테스트 (10초 녹음+재생)
   볼륨바 실시간 표시, 두 테스트 모두 통과해야 시험 시작 가능

④ 세션 생성
   크레딧 즉시 차감 → 세션 레코드 생성 → 만료 시간 설정
   실패 시 크레딧 자동 환불
```

### Phase 2: 시험 진행

```
⑤ Q1 자기소개 (평가 제외)
   시스템 질문 재생 → 녹음 → 제출
   eval_status = 'skipped' (규칙엔진 제외)

⑥ Q2~Q15 답변 녹음 (14문항)
   각 문항마다:
   [질문 듣기] → [다시 듣기(5초 내 1회)] → [답변 녹음] → [제출]

   제출 시 동작:
   1. 음성 파일 → Supabase Storage 업로드 (WAV)
   2. mock_test_answers 레코드 생성
   3. fire-and-forget으로 평가 EF 호출 (백그라운드에서 자동 실행)
   4. current_question +1 → 다음 문항으로 이동

⑦ 15문항 완료
   세션 status = 'completed'
   holistic_status = 'pending'
   → 모든 답변 평가 완료되면 Stage C (종합 리포트) 자동 트리거
```

### Phase 3: 백엔드 평가 (자동, 약 2~3분)

```
⑧ Stage A: STT + 발음 (문항당 ~5초)
   음성 → Whisper STT → Azure 발음 평가
   스킵 판정: 15초 미만 / 15자 미만 / Whisper 환청

⑨ Stage B-1: 체크박스 판정 (문항당 ~12초)
   GPT-4.1-mini가 74개 체크박스 pass/fail 판정
   + 과제충족(🔴필수/🟡심화) 판정
   + feedback_branch 결정 (fulfilled/partial/failed)

⑩ Stage B-2: 코칭 피드백 (문항당 ~8초)
   fulfilled/partial: GPT-4.1이 코칭 6-Layer 생성
   failed: 룰 기반 구제 메시지 (GPT 호출 없음)

⑪ Stage C: 종합 리포트 (~45초)
   규칙엔진 7-Step → 최종 등급 결정
   FACT 점수 계산 → GPT 종합 리포트 생성
   성장 리포트 생성 (2회차부터)
```

### Phase 4: 결과 확인

```
⑫ 평가 대기 화면
   5초 간격 폴링으로 진행률 모니터링
   Q2~Q15 각 문항별 진행 상태 표시
   holistic_status = 'completed' → 결과 페이지 활성화

⑬ 결과 4탭
   ┌─────────────────────────────────────────┐
   │  [종합]  [진단]  [문항별]  [성장]        │
   │                                         │
   │  종합: 등급 + FACT + 한줄평 + 로드맵     │
   │  진단: Top 3 약점 + 유형별 + 반복 오류   │
   │  문항별: 14문항 아코디언 (코칭 상세)      │
   │  성장: 이전 세션 비교 + 패턴 감지         │
   └─────────────────────────────────────────┘

⑭ 튜터링 연결
   결과에서 "맞춤 훈련 시작" CTA 클릭
   → 약점 기반 자동 처방 (tutoring_prescription)
   → 튜터링 모듈로 연결
```

## 전체 데이터 흐름도

```
[사용자]
   │
   ▼
┌──────────────────────────────────────────────┐
│  기출 선택 + 모드 선택 + 디바이스 테스트      │
│  → createSession() : 크레딧 차감 + 세션 생성  │
└──────────────────────┬───────────────────────┘
                       │
   ┌───────────────────▼───────────────────┐
   │         Q1~Q15 녹음 + 제출             │
   │  각 문항: 녹음 → Storage 업로드        │
   │  → submitAnswer() : answers 저장       │
   │  → fire-and-forget EF 호출             │
   └───────────────────┬───────────────────┘
                       │
        ┌──────────────▼──────────────┐
        │      Stage A: process       │
        │  Whisper STT + Azure 발음   │
        │  + 3단계 스킵 판정          │
        └──────────────┬──────────────┘
                       │ fire-and-forget
        ┌──────────────▼──────────────┐
        │    Stage B-1: eval-judge    │
        │  GPT-4.1-mini              │
        │  체크박스 74개 + 과제충족    │
        │  + feedback_branch 결정     │
        └──────────────┬──────────────┘
                       │ fire-and-forget
        ┌──────────────▼──────────────┐
        │    Stage B-2: eval-coach    │
        │  fulfilled → GPT-4.1 코칭   │
        │  partial → GPT-4.1 축소     │
        │  failed → 룰 기반 구제      │
        └──────────────┬──────────────┘
                       │ 모든 문항 완료 시
        ┌──────────────▼──────────────┐
        │      Stage C: report        │
        │  규칙엔진 7-Step → 등급 판정 │
        │  FACT 점수 계산              │
        │  GPT 종합 리포트 (9섹션)     │
        │  성장 리포트 (2회차+)        │
        │  튜터링 처방 자동 변환       │
        └──────────────┬──────────────┘
                       │
   ┌───────────────────▼───────────────────┐
   │         결과 페이지 (4탭)              │
   │  폴링으로 완료 감지 → 결과 표시        │
   │  → 튜터링 CTA 연결                    │
   └───────────────────────────────────────┘
```

## 상태 전이 다이어그램

### 세션 상태 (mock_test_sessions.status)

```
    ┌─────────┐
    │ active  │ ← 세션 생성
    └────┬────┘
         │
    ┌────┴────────────────────┐
    │                         │
    ▼                         ▼
┌───────────┐          ┌───────────┐
│ completed │          │  expired  │
│ (사용자   │          │ (72h/90m  │
│  완료)    │          │  경과)    │
└───────────┘          └───────────┘
```

### 답변 평가 상태 (mock_test_answers.eval_status)

```
pending → processing → stt_completed → evaluating → judge_completed → completed
   │                        │                                             │
   └─────── skipped ◀───────┘                                            │
   └─────── failed ◀─────────────────────────────────────────────────────┘
```

### 종합 평가 상태 (mock_test_sessions.holistic_status)

```
pending → processing → completed
                    └→ failed (3회 재시도 실패)
```

## 타이밍 요약

| 단계 | 소요 시간 | 비고 |
|------|----------|------|
| 시험 준비 | 1~3분 | 기출 선택 + 모드 + 디바이스 테스트 |
| 시험 진행 | 20~40분 | 15문항 녹음 (실전: 40분 제한) |
| Stage A | ~5초/문항 | STT + 발음 (병렬 처리) |
| Stage B-1 | ~12초/문항 | 체크박스 판정 |
| Stage B-2 | ~8초/문항 | 코칭 생성 (무응답은 즉시) |
| Stage C | ~45초 | 규칙엔진 + GPT 종합 리포트 |
| **전체 평가** | **~2~3분** | **14문항 병렬 → Stage C 1회** |
