## 기출 선택 (getExamPool)

### 동작 흐름

```
사용자가 "응시" 탭 진입
   │
   ├─ getExamPool() Server Action 호출
   │   │
   │   ├─ 이미 사용한 기출(submission_id) 조회
   │   │   → mock_test_sessions에서 status ≠ 'expired'인 submission_id 수집
   │   │
   │   ├─ 승인된 기출 조회 (최대 20개)
   │   │   → submissions 테이블
   │   │   → 조건: status='complete', exam_approved='approved', 본인 후기 제외
   │   │   → 이미 사용한 기출 제외 (NOT IN)
   │   │
   │   ├─ 3개 미만이면 → 이력 무시하고 전체에서 재조회
   │   │
   │   └─ 랜덤 3개 추출 → 카드 데이터 구성
   │
   └─ 3장의 기출 카드가 화면에 표시됨
```

### 카드에 표시되는 정보

| 항목 | 데이터 소스 |
|------|------------|
| 시험일 | `submissions.exam_date` |
| 달성 등급 | `submissions.achieved_level` |
| 주제 구성 | `submission_combos` → combo_type별 대표 topic |
| 문항 유형 분포 | `submission_questions` → `questions.question_type_eng` 카운트 |
| 난이도 힌트 (1~3) | ADV 계열 비율: ≥60% → 3, ≥40% → 2, 나머지 → 1 |

### 병렬 조회 최적화

```
기출 3개 선택 후:
Promise.all([
  submission_combos 조회 (3개 기출의 콤보),
  submission_questions 조회 (3개 기출의 질문),
])
→ questions 테이블에서 question_type_eng 조회 (배치)
```

## 세션 생성 (createSession)

### 동작 흐름

```
"시험 시작" 클릭
   │
   ├─ ① 활성 세션 확인 (동시 1개 제한)
   │   → mock_test_sessions에서 user_id + status='active' 조회
   │   → 있으면 에러 반환
   │
   ├─ ② 크레딧 차감 (RPC: consume_mock_exam_credit)
   │   → plan_mock_exam_credits 먼저 차감 (만료되는 것 우선)
   │   → 부족하면 mock_exam_credits에서 차감
   │   → 모두 부족하면 에러 반환
   │
   ├─ ③ 기출 문제 ID 조회
   │   → submission_questions에서 question_number 순서로 15개 추출
   │   → 실패 시 크레딧 자동 환불 (refund_mock_exam_credit)
   │
   ├─ ④ 만료 시간 계산
   │   → 훈련: now + 72시간
   │   → 실전: now + 90분
   │
   └─ ⑤ 세션 INSERT
       → session_id: 'mt_' + 8자리 랜덤 hex (generateSessionId)
       → status: 'active'
       → current_question: 1
       → total_questions: 15
```

### 세션 ID 형식

```
mt_8099015f  (접두사 'mt_' + crypto.randomUUID().slice(0,8))
```

### 실패 시 자동 환불

- 기출 문제 조회 실패 → `refund_mock_exam_credit` RPC 호출
- 세션 INSERT 실패 → `refund_mock_exam_credit` RPC 호출
- 크레딧 차감 후 어떤 단계든 실패하면 반드시 환불

## 시험 진행 (mock-exam-session.tsx, ~520줄)

### 화면 구성

```
┌─────────────────────────────────────────┐
│  ◀ 뒤로   Q3 / 15    [타이머] ⏱️        │  ← ImmersiveHeader
├─────────────────────────────────────────┤
│                                         │
│  [질문 텍스트] (훈련 모드만 표시)         │
│                                         │
│  [질문 오디오 재생 버튼]                  │
│                                         │
│  ┌─────────────────────────────┐        │
│  │      녹음 영역              │        │
│  │  [녹음 시작] / [녹음 중...] │        │
│  │  볼륨 레벨 바               │        │
│  └─────────────────────────────┘        │
│                                         │
│  [건너뛰기]            [제출]            │
│                                         │
├─────────────────────────────────────────┤
│  ① ② ③ ④ ⑤ ⑥ ⑦ ⑧ ⑨ ⑩ ⑪ ⑫ ⑬ ⑭ ⑮   │  ← QuestionGrid
└─────────────────────────────────────────┘
```

### 문항 진행 플로우

```
문항 시작
   │
   ├─ 질문 오디오 재생 (use-question-player.ts)
   │   → Storage에서 질문 WAV 파일 로드
   │   → 재생 완료 또는 사용자 중지
   │   → 다시 듣기: 5초 이내 1회만 허용
   │
   ├─ 답변 녹음 (use-recorder.ts)
   │   │
   │   ├─ MediaRecorder API (WebM 포맷)
   │   ├─ Web Audio API AnalyserNode → 볼륨 실시간 표시
   │   ├─ 녹음 중 무음 감지 (WPM 기반)
   │   │
   │   └─ 녹음 완료 시:
   │       ├─ WebM → WAV 변환 (AudioContext decodeAudioData)
   │       └─ Blob → File 객체 생성
   │
   ├─ 제출 (submitAnswer)
   │   │
   │   ├─ ① Supabase Storage 업로드 (WAV)
   │   │   → 경로: mock-test-recordings/{session_id}/q{number}.wav
   │   │
   │   ├─ ② mock_test_answers UPSERT
   │   │   → Q1: eval_status='skipped', skipped=true
   │   │   → Q2~Q15: eval_status='pending', skipped=false
   │   │
   │   ├─ ③ current_question + 1 업데이트
   │   │
   │   └─ ④ Q2~Q15: fire-and-forget → mock-test-process EF
   │       → fetch(efUrl, { method: 'POST', ... })
   │       → .catch() 로만 에러 처리 (await 하지 않음)
   │
   └─ 다음 문항으로 이동 (또는 15번 완료 시 completeSession)
```

### Q1 자기소개 특수 처리

```
Q1은 평가 제외 (F-6 결정)
  → submitAnswer 시 eval_status = 'skipped'
  → fire-and-forget EF 호출하지 않음 (isQ1 체크)
  → 규칙엔진에서도 skipped로 처리
```

### 건너뛰기 (skipQuestion)

```
사용자가 "건너뛰기" 클릭
  → mock_test_answers UPSERT: eval_status='skipped', skipped=true
  → current_question + 1 업데이트
  → fire-and-forget EF 호출하지 않음
  → 규칙엔진에서 skipped_questions에 추가
```

## 녹음 훅 (use-recorder.ts) 상세

### WebM → WAV 변환 파이프라인

```
MediaRecorder (WebM/opus)
   → Blob 수집 (ondataavailable)
   → 녹음 종료 시 (onstop):
       → WebM Blob 생성
       → AudioContext.decodeAudioData(arrayBuffer)
       → Float32Array → Int16Array (PCM 16bit)
       → WAV 헤더 붙이기 (44바이트)
       → WAV Blob 생성
```

### cancelledRef 패턴 (레이스컨디션 방지)

```
문제: MediaRecorder.stop() → onstop 콜백이 비동기
     reset() 중 stop() 호출 시 비동기 onstop이 상태를 덮어씀

해결:
  const cancelledRef = useRef(false);

  reset() {
    cancelledRef.current = true;  // 현재 onstop 무효화
    recorder.stop();
  }

  onstop() {
    if (cancelledRef.current) return;  // 무시
    // 정상 처리
  }

  start() {
    cancelledRef.current = false;  // 새 녹음 시작 시 리셋
  }
```

   │

   │   │

   │   │   │   → current_question부터 재개
   │   │   │
   │   │   └─ "새로 시작" → 기존 세션 만료 처리
   │   │       → expireSession() → 새 기출 선택 화면
   │   │
   │   └─ 복원 시 답변 상태 확인
   │       → 이미 제출한 문항은 건너뛰기

   └─ 활성 세션 없음:
       → 기출 선택 → 모드 선택 → 디바이스 테스트 → 새 세션

## 세션 완료 (completeSession)

### 동작 흐름

```
Q15 답변 제출 후:
   │
   ├─ ① 세션 상태 업데이트
   │   → status = 'completed'
   │   → completed_at = now
   │   → holistic_status = 'pending'
   │
   ├─ ② 모든 답변 평가 완료 여부 확인
   │   → eval_status NOT IN ('completed', 'skipped', 'failed') 조회
   │
   ├─ 미완료 답변 0개:
   │   │
   │   └─ Stage C 즉시 트리거 (fire-and-forget)
   │       → mock-test-report EF 호출
   │       → "Case A": completeSession이 마지막 eval-coach보다 늦은 경우
   │
   └─ 미완료 답변 있음:
       └─ 대기 (eval-coach가 마지막 답변 완료 시 Stage C 트리거)
           → "Case B": eval-coach가 completeSession보다 늦은 경우
```

### Stage C 트리거 이중 안전장치

```
Case A: completeSession이 늦게 호출
   → completeSession에서 pending 답변 확인 → 0개면 즉시 트리거

Case B: eval-coach가 늦게 완료
   → eval-coach가 completed 답변 갱신 후
   → 세션 status='completed' 확인
   → pending 답변 0개 확인 → 즉시 트리거

→ 어떤 순서로 완료되든 Stage C가 정확히 1번 트리거됨
```

## 만료 처리 (expireSession)

```
만료 조건:
  - 훈련 모드: started_at + 72시간 경과
  - 실전 모드: started_at + 90분 경과

만료 감지 시점:
  ① getActiveSession 호출 시 (앱 진입)
  ② submitAnswer 호출 시 (답변 제출)
  ③ 타이머 컴포넌트에서 (실전 모드 카운트다운)
  ④ SessionWrapper 마운트 시

만료 처리:
  → mock_test_sessions.status = 'expired'
  → 크레딧 환불 없음 (시험은 시작됨)
  → 부분 답변의 평가 결과는 유지 (결과 열람 불가)
```

## Server Actions 전체 목록

| # | 함수 | 역할 |
|---|------|------|
| 1 | `getExamPool` | 기출 3개 랜덤 추출 + 미리보기 카드 |
| 2 | `createSession` | 크레딧 차감 + 세션 생성 |

| 4 | `skipQuestion` | 문항 건너뛰기 |
| 5 | `getSession` | 세션 + 답변 + 평가 + 리포트 + 질문 전체 조회 |
| 6 | `completeSession` | 세션 완료 + Stage C 조건부 트리거 |
| 7 | `expireSession` | 세션 만료 처리 |
| 8 | `getHistory` | 이력 조회 (최신 50개) |
| 9 | `getActiveSession` | 활성 세션 확인 + 만료 자동 처리 |
| 10 | `getEvaluation` | 개별 평가 결과 조회 |
| 11 | `checkMockExamCredit` | 크레딧 잔여량 확인 |
