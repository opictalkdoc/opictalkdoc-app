# 왜 서베이가 가장 중요한가 — 오픽톡닥의 설계 철학

> OPIc 시험에서 Background Survey가 왜 "가장 중요한 변수"인지, 그리고 오픽톡닥은 왜 이 사실을 중심으로 설계되었는지를 데이터로 증명한다.

---

## 1. OPIc에 대한 흔한 오해

### 현실: 68%가 IM2 이하

OPIc은 **1,700개 이상의 기업과 기관**이 채용·인사·교육 평가에 활용하는 사실상의 취업 필수 시험이다. 삼성, 현대, SK, LG 등 대기업은 OPIc 등급을 필수적으로 요구한다.

그런데 실제 등급 분포를 보면:

| 등급 | 비율 | 누적 |
|------|------|------|
| AL | 6% | 6% |
| IH | 22% | 28% |
| IM3 | 4% | 32% |
| **IM2** | **29%** | **61%** |
| IM1 | 13% | 74% |
| IL 이하 | 26% | 100% |

**응시자의 68%가 IM2 이하**에 머문다. 대기업 인문계 커트라인인 IH 이상은 **28%만** 달성한다.

왜 이렇게 많은 사람이 원하는 등급을 받지 못하는가?

### "대충 아는 것"이 진짜 문제다

요즘은 유튜브, 블로그, 인강 덕분에 대부분의 응시자가 OPIc에 대해 **대충은** 안다.

```
"서베이가 중요하다더라"          → 안다. 그런데 얼마나?
"말할 수 있는 주제를 골라라"     → 안다. 그런데 어떤 기준으로?
"난이도는 5-5가 좋다더라"       → 안다. 그런데 왜?
"스크립트 외우면 안 된다더라"    → 안다. 그런데 대안이 뭔데?
```

**"대충 안다"는 "정확히 모른다"와 같다.** 그리고 정확히 모르면 전략이 부정확해진다.

| 대충 아는 것 | 정확히 아는 것 (= 오픽톡닥) |
|-------------|------------------------|
| "서베이가 중요하다" | 서베이가 시험 문제의 **60%를 결정**한다 (84건 데이터 증명) |
| "말할 수 있는 주제를 골라라" | 선택해도 **시험에 안 나오는 항목**이 있다 (필러 전략) |
| "매번 적당히 골라" | 서베이를 **고정**해야 학습 범위가 한정된다 |
| "스크립트 외우면 안 된다" | **내 경험 기반** 스크립트를 AI로 생성해야 한다 |
| "주제가 많아서 힘들다" | 전체 ~610개 질문이 **118개로 압축** (1/5), 집중 학습은 **10개 핵심 주제** (230건 데이터 증명) |

"대충 아는 사람"과 "정확히 아는 사람"의 학습은 완전히 다르다:

```
대충 아는 사람:
  서베이를 매번 바꿈 → 매번 다른 주제 → 수십 개 주제 공부 → 하나도 깊이 못함

정확히 아는 사람:
  서베이를 고정 → 항상 같은 범위 → 핵심 10개만 집중 → 깊이 있는 답변
```

그렇다면 "정확히 안다"는 것은 무엇인가? 이 문서에서 데이터로 증명한다.

### 추가로, 스크립트 암기는 치명적이다

대충 아는 수준에서 벗어나지 못한 학습자가 결국 의존하는 것이 **남의 스크립트 암기**다.

- ACTFL 공식 입장: **암기를 통해 받을 수 있는 최고 등급은 NM (Novice Mid)**
- 채점자들은 여러 수험자에게서 반복되는 답변 패턴을 **감점 요소**로 판단
- 실제 경험을 말해도 빈출 스크립트와 겹치면 암기로 오인받아 등급 하락
- 심한 경우 **시험 무효 처리** 대상 (OPIc 규정: "대화를 미리 연습한 주제로 이끌어가는 경우")

**사람들이 OPIc을 어려워하는 근본 원인은 영어 실력이 아니다. 시험 구조 자체를 모르는 것이다.**

## 2. OPIc 시험을 결정하는 두 가지 변수

OPIc 문제를 결정하는 변수는 딱 두 가지다:

```
변수 1: Background Survey (배경설문)    → "무엇이" 나오는가 (출제 주제)
변수 2: Self-Assessment (난이도 자가진단) → "어떻게" 나오는가 (문제 유형)
```

**핵심: 두 변수 모두 수험자 본인이 직접 선택한다.**
즉, OPIc은 "뭐가 나올지 모르는 시험"이 아니라 **"내가 직접 시험 범위와 유형을 결정하는 시험"**이다.

그런데 이 두 변수 중 **서베이가 압도적으로 더 중요하다.** 그 이유를 데이터로 증명한다.

---

## 3. 변수 1: Background Survey — 시험 범위를 결정하는 메뉴판

### 3-1. 기본 구조

Background Survey는 시험 전에 수험자가 자신의 관심사/경험을 선택하는 **메뉴판**이다.

```
카테고리 1~4: 각 1개 선택
카테고리 5~8: 총 12개 이상 선택
  → 선택한 주제에서 3개 콤보 출제 (선택형)
  → 선택하지 않은 주제에서 2개 콤보 출제 (공통형)
  → 합계 5콤보 × 3문항 = 15문항
```

### 3-2. 8개 카테고리 (3개 그룹)

**그룹 A: 직업 관련** (카테고리 1~3) — 각각 하나 선택

| 카테고리 | 선택 방식 | 비고 |
|---------|----------|------|
| 1. 직업 | 하나 선택 (4개 중) | "일 경험 없음" 가능 |
| 2. 학생 여부 | 하나 선택 (예/아니오) | |
| 3. 수강 | 하나 선택 (4개 중) | "수강 후 5년 이상 지남" 가능 |

**그룹 B: 거주지** (카테고리 4) — 하나 선택

| 카테고리 | 선택 방식 |
|---------|----------|
| 4. 거주지 | 하나 선택 (5개 중) |

**그룹 C: 개인 활동 및 관심사** (카테고리 5~8) — 4개 하위 영역에서 **총 12개 이상** 선택

| 카테고리 | 선택 방식 | 비고 |
|---------|----------|------|
| 5. 여가 활동 | **두 개 이상** (27개 중) | |
| 6. 취미/관심사 | **한 개 이상** (14개 중) | |
| 7. 운동 | **한 개 이상** (24개 중) | "운동을 전혀 하지 않음" 가능 |
| 8. 휴가/출장 | **한 개 이상** (5개 중) | |

### 3-3. 중요한 사실: 답변은 서베이와 달라도 감점 없음

> 출제는 서베이 기준이지만, **채점은 답변 자체만 평가한다.**
> 서베이에서 "홀로 거주"를 선택했어도, 답변에서 "가족과 함께 산다"고 말해도 감점이 없다.
> 이는 서베이를 "전략적 도구"로 활용할 수 있다는 뜻이다.

## 4. 10가지 문제유형과 콤보 시스템

### 4-1. OPIc의 모든 문제는 10가지 유형

| 그룹 | 유형 | 내용 | 예시 |
|------|------|------|------|
| **기본** | 1번 묘사 | 장소나 종류를 현재시제로 묘사 | "좋아하는 공원을 묘사해주세요" |
| **기본** | 2번 루틴 | 활동, 루틴, 단계를 묘사 | "공원에서 보통 뭘 하나요" |
| **기본** | 3번 경험 | 최초 혹은 최근 경험 | "최근에 공원 간 적 있나요" |
| **기본** | 4번 인상 | 인상적인 경험 | "공원에서 인상깊은 일이 있나요" |
| **기본** | 5번 질문 | 상대방에게 간단한 질문 | "상대방에게 공원에 대해 질문하기" |
| **롤플레이** | 6번 정보요청 | 상황 속 정보 요청 | "영화 티켓 구매에 필요한 정보요청" |
| **롤플레이** | 7번 문제해결 | 문제 상황 대안 제시 | "영화관에서 생긴 티켓 문제 해결" |
| **롤플레이** | 8번 경험연결 | 유사한 문제 경험 | "예약이 잘못되었던 경험" |
| **고급** | 9번 비교/대조 | 과거와 현재 비교 | "10년 전과 지금의 여행 문화 비교" |
| **고급** | 10번 사회이슈 | 사회적 관심사 설명 | "환경 문제에 대해 예를 들어 설명" |

### 4-2. 15문제는 무작위가 아니다 — 5개 콤보 시스템

15문제는 **5개 콤보**로 구성되며, 한 콤보 안의 문제는 **같은 주제**에서 연속 3문제가 나온다.

```
난이도 5~6 기준 (추천):

Q1       ─ 자기소개 (채점 안함 / Skip 가능)
Q2~Q4    ─ 일반콤보 1 (설명/묘사 → 행동습관/시간적비교 → 과거경험)
Q5~Q7    ─ 일반콤보 2 (설명/묘사 → 행동습관/시간적비교 → 과거경험)
Q8~Q10   ─ 일반콤보 3 (설명/묘사 → 행동습관/시간적비교 → 과거경험)
Q11~Q13  ─ 롤플레이   (정보요청 → 대안제시 → 과거경험)
Q14~Q15  ─ 어드밴스   (비교 → 사회적이슈)  ← Q14=IH판별, Q15=AL판별

※ 선택형 3개, 공통형 2개가 위 5개 콤보에 배분된다. 위치는 고정되지 않는다.
```

**핵심 포인트:**
- 한 콤보 = 같은 주제 3문제 연속 (예: "영화보기"에서 설명/묘사 → 행동습관 → 과거경험)
- **5개 콤보 중 선택형 3개, 공통형 2개로 구성된다.** 위치는 고정되지 않는다.
- 롤플레이 = 반드시 상황극 (정보요청 → 대안제시 → 과거경험)
- 어드밴스 = 난이도 5~6에서만 고급 문제 (Q14=IH판별: 비교, Q15=AL판별: 사회이슈)

## 5. 데이터가 증명한다: 선택형 3 vs 공통형 2

### 두 가지 유형의 출제 주제

| 유형 | 설명 | 예시 |
|------|------|------|
| **선택형** | 서베이에서 **선택한 항목**에서 출제 | 집, 음악, 영화, 쇼핑, 국내여행 |
| **공통형** | 서베이와 **무관하게** 누구에게나 출제 | 은행, 미용실, 재활용, 지형, 날씨 |

**핵심 질문: 5개 콤보 중 선택형과 공통형의 비율은?**

### 실제 데이터 분석

오픽톡닥 데이터랩에 제출된 **실제 OPIc 시험 후기 84건**을 분석했다.
(필터: 완전한 5콤보 구조 + 고급 난이도 5-5/5-6/6-5/6-6)

| 지표 | 수치 |
|------|------|
| 총 시험 | 84건 (420개 콤보) |
| **선택형 콤보** | **246개 (58.6%)** |
| **공통형 콤보** | **174개 (41.4%)** |
| **평균 비율** | **선택형 2.93 : 공통형 2.07** |

```
5개 콤보 중:
  ■ ■ ■ □ □   선택형 3개 ← 내가 서베이에서 선택한 항목
  □ □ □ ■ ■   공통형 2개 ← 서베이 무관, 누구에게나 출제
```

### 분포까지 확인

| 5콤보 중 선택형 | 시험 수 | 비율 | 누적 |
|----------------|---------|------|------|
| 5/5 | 2건 | 2.4% | 2.4% |
| 4/5 | 12건 | 14.3% | 16.7% |
| **3/5** | **50건** | **59.5%** | **76.2%** |
| 2/5 | 18건 | 21.4% | 97.6% |
| 1/5 | 2건 | 2.4% | 100% |
| 0/5 | 0건 | 0% | - |

**76.2%의 시험에서 선택형이 3개 이상이다.**
**선택형이 0개인 시험은 단 한 건도 없다.**

## 6. 이 사실이 왜 그렇게 중요한가

### 시험의 60%를 내가 직접 결정할 수 있다

```
서베이에서 내가 선택한 항목 → 시험 5콤보 중 3개의 주제를 결정

즉, 서베이 = 시험의 약 60%를 결정하는 컨트롤러
```

이것은 OPIc만의 독특한 구조다.

| 시험 | 출제 범위 예측 가능? |
|------|-------------------|
| 토익 | ❌ 예측 불가 |
| 토플 | ❌ 예측 불가 |
| 아이엘츠 | ❌ 예측 불가 |
| **OPIc** | **✅ 60%를 내가 직접 결정** |

### 서베이를 고정하면 학습 범위가 한정된다

```
서베이를 매번 다르게 선택     → 매번 다른 주제가 출제 → 예측 불가능 → ~610개 질문 대비
서베이를 항상 같게 고정       → 항상 같은 범위에서 출제 → 예측 가능 → 118개만 대비
같은 범위 안에서도 빈도 순    → 핵심 주제부터 깊이 학습 → 10개 핵심 주제 집중
집중 학습                    → 준비된 답변으로 시험 대응 → 시험의 약 78% 커버
```

**이것이 오픽톡닥의 핵심 설계 원리다.**

> 서베이를 "고정"하는 것만으로, OPIc은 "예측 불가능한 시험"에서 "예측 가능한 시험"으로 바뀐다.

### 오픽톡닥의 2단계 전략

오픽톡닥의 학습 전략은 선택형이든 공통형이든 동일한 패턴을 따른다:

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  1차: 대비 범위 축소 — "몇 개를 공부해야 하는가?"               │
│    선택형: 서베이 고정 → ~610개 → 118개 (80% 감소)            │
│    공통형: 줄일 수 없음 → 26개+ 주제 고정                      │
│                                                             │
│  2차: 학습 우선순위 — "어떤 것부터 공부해야 하는가?"             │
│    선택형: 118개 중 빈도 순 → 집(51.7%) 먼저, TV(2.2%) 나중에  │
│    공통형: 26개 중 빈도 순 → 지형(9.3%) 먼저, 치과(0.5%) 나중에 │
│                                                             │
│  → 범위를 줄이고, 그 안에서 빈출 순으로 우선 학습               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 7. 오픽톡닥 추천 서베이 설계 — 카테고리별 전략

### 설계 원칙

```
원칙 1: 시험 주제를 만들지 않는 항목으로 빈칸을 채운다 (필러 전략)
원칙 2: 출제 가능 주제 수를 최소화하는 항목을 선택한다 (범위 최소화)
```

### 카테고리별 추천과 이유

#### 카테고리 1: 직업 → **"없음"** (전략 A: 출제 차단)

| 선택지 | 추천 | 이유 |
|--------|------|------|

```
난이도 3~4 선택 시:
  Q11~Q13: 롤플레이 콤보 (유형 6, 7, 4/8)
  Q14~Q15: IM 문제 — 유형 1 (묘사) + 유형 5 (질문) ← 롤플레이가 2번째!

난이도 5~6 선택 시:
  Q11~Q13: 롤플레이 콤보 (유형 6, 7, 4/8) ← 롤플레이 1회만
  Q14~Q15: AL 문제 — 유형 9 (비교) + 유형 10 (사회이슈)
```

**핵심 차이**: 3~4단계는 롤플레이(질문)가 **2번**, 5~6단계는 **1번**.
한국 수험자는 대부분 질문 유형이 약하므로, 2번 마주치는 것보다 1번이 유리하다.

### 5-5 선택자 vs 전체 평균

| 등급 | 전체 평균 | 5-5 선택자 | 배율 |
|------|----------|-----------|------|
| **AL** | 6% | **20%** | **3.3배** ↑ |
| **IH** | 22% | **41%** | **1.9배** ↑ |
| **IM3** | 4% | **26%** | **6.5배** ↑ |
| IM2 | 29% | 6% | 80% ↓ |
| IL | 15% | 2% | 87% ↓ |

```
5-5 선택자의 상위 3등급(AL+IH+IM3) 달성률: 87%
전체 평균 상위 3등급 달성률:                  32%

→ 같은 실력이라도 5-5를 선택하는 것만으로
  상위 등급 확률이 2배~6.5배 증가한다!
```

### 왜 6-6이 아닌 5-5인가?

| 항목 | 5-5 | 6-6 |
|------|-----|-----|
| 비교 문제 (유형 9) | **1개** | **2개** |
| 답변 패턴 반복 위험 | 낮음 | ⚠️ 높음 — 같은 "A는 ~, B는 ~" 패턴 반복 |
| 채점자 인식 | 다양한 답변 | "같은 패턴으로만 답변하네" |
| Q14~Q15 스킵해도 IH? | ✅ 가능 | ❌ 비교 2개 모두 잘해야 |

> **실제 사례**: IM3 두 번, IH 세 번 받던 수험자가 6-6에서 5-5로 바꾼 후 **2주 만에 AL 획득**.

> **결론: 어떤 등급을 목표로 하든 5-5가 최적이다.**

## 11. 등급별 학습 전략

### 등급별 핵심 비교

| 등급 | 삼성 급수 | 추천 난이도 | 집중 유형 | 답변 길이 | 핵심 |
|------|---------|-----------|---------|----------|------|
| **IL** | 4급 | 3단계 | 3, 4, 6, 7 | 30~40초 | 단문, 짧게 |
| **IM1/IM2** | 3급 | 4단계 | 3, 4, 6, 7 | 60~70초 | 자기 생각 추가 |
| **IM3/IH** | 2급 | **5-5** | 3, 4, **7, 8** | 80~100초 | **복문, 관계사** |
| **AL** | 1급 | **5-5** | **7, 8, 9, 10** | 100~120초 | **디테일, 문법정확** |

### 패턴

```
등급이 올라갈수록:
  → 난이도 높게 선택
  → 롤플레이 + 고급유형 비중 증가
  → 답변 길이 증가 (30초 → 120초)
  → 복잡한 문장 구조 요구

공통점: 모든 등급에서 "쉐도잉으로 영어다운 톤과 억양 만들기"가 기본
```

## 12. 전체 그림: OPIc의 3대 전략 축

```
전략 축 1: Background Survey → "무엇이 나오는가" 결정
  ✅ 서베이 고정 → 출제 범위를 한정하여 예측 가능하게 만든다
  ✅ 필러 전략으로 항목 수 채우면서 출제 주제 수 최소화

전략 축 2: Self-Assessment → "어떻게 나오는가" 결정
  ✅ 5-5 선택 → 롤플레이 1회, 상위등급 확률 2~6.5배 증가
  ❌ 3-4 선택 → 롤플레이 2회, 한국인 약점 노출
  ⚠️ 6-6 선택 → 비교문제 2회, 패턴 반복 위험

전략 축 3: 답변 전략 → "어떻게 말하는가" 결정
  ✅ 내 경험 기반 답변 → 자연스러움, 디테일
  ❌ 남의 스크립트 암기 → 부자연스러움, 감점
  ✅ 등급별 목표 발화량 달성 (30초 → 120초)
  ✅ 등급별 집중 유형에 맞춘 훈련
```

### 기존 학습법 vs 오픽톡닥 학습법

| | 기존 학습법 | 오픽톡닥 학습법 |
|---|------------|-------------|
| 서베이 | 매번 다르게 선택 | **추천 서베이로 고정** |
| 난이도 | 감으로 선택 | **5-5 (데이터 검증)** |
| **1차 전략 (범위)** | ~610개 질문 대비 | **118개로 압축 (1/5)** |
| **2차 전략 (우선순위)** | 모든 주제 균등 학습 | **빈도 순 핵심부터 집중** |
| 데이터 근거 | 없음 (감으로 학습) | **230건+ 데이터 + master_questions 510개** |
| 선택형 대비 | 예측 불가 | **핵심 5개 → 콤보 절반 이상 대비** |
| 공통형 대비 | 예측 불가 | **빈출 6개로 45% 커버** |
| 시험 커버리지 | 알 수 없음 | **약 78% (5문제 중 4문제 대비)** |
| 스크립트 | 남의 것 암기 | **AI가 내 경험 기반으로 생성** |
| 롤플레이 | 랜덤 | **MP3구매, 공연예매, 여행계획 집중** |

### 최종 학습 구조: 2단계 전략

```
━━━ 1차 전략: 서베이로 대비 범위 축소 ━━━━━━━━━━━━━━━━━━━━━━

  선택형: 서베이 고정 → ~610개 → 118개 (80% 감소)
  공통형: 줄일 수 없음 → 26개+ 주제 그대로

━━━ 2차 전략: 빈도 분석으로 학습 우선순위 ━━━━━━━━━━━━━━━━━━

  선택형 118개 중:
    🔴 핵심 5개 먼저 (집, 음악, 국내여행, 집휴가, MP3구매)
    🟠 우선 5개 다음 (영화, 해외여행, 여행계획, 쇼핑, 공연예매)
    ⚪ 후순위 4개 여유 시 (TV, 차량렌트, 영화관람계획, 옷구매)

  공통형 26개 중:
    🔴 상위 6개 집중 (지형, 미용실, 재활용, 은행, 날씨, 산업) → 45% 커버

━━━ 결합 효과 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  선택형 60% (100% 대비) + 공통형 40% × 45% = 약 78%
  → OPIc 15문제 중 약 12문제를 "대비한 상태"로 시험 응시
```

### 빈도 데이터의 실전 활용

2단계 전략의 빈도 분석 결과는 이론에 그치지 않고, 오픽톡닥의 핵심 모듈 2개에 직접 반영된다.

#### AI 훈련소 — "빈출 주제부터 스크립트 생성"

```
master_questions 510개 질문
  ↓ 데이터랩 빈도 분석 결과 적용
  ↓
AI 훈련소 UI에서:
  ✅ 주제를 빈도 순으로 정렬 — 집(51.7%) > 음악(48.7%) > ... > TV(2.2%)
  ✅ 질문을 빈도 순으로 정렬 — 많이 출제된 질문이 먼저 표시
  → 사용자는 자연스럽게 가장 빈출인 주제/질문부터 스크립트를 생성하게 됨
```

사용자가 별도로 우선순위를 파악할 필요 없이, **시스템이 빈도 순으로 제시**하므로 학습 효율이 자동으로 극대화된다.

#### 모의고사 빈출 모드 — "실전 출제 경향 그대로 재현"

```
데이터랩 시험 후기 제출
  ↓
submission_questions → custom_mode_questions (콤보 자동 생성)
  ↓
모의고사 빈출 모드:
  ✅ 동일한 질문 조합의 출현 횟수(count) 기준 정렬
  ✅ 가장 많이 출제된 콤보부터 우선 선택
  ✅ General 3 + Roleplay 1 + Advance 1 = 5콤보 15문제
  → 사용자는 실제 OPIc과 동일한 출제 경향으로 모의고사를 치르게 됨
```

```
핵심 차이:
  기존 학습법: "어떤 문제가 나올지 모르니 다 준비하자"
  오픽톡닥:   "이 문제가 가장 많이 나오니, 이것부터 연습하자"
```

#### 데이터 → 학습 → 시험의 일관된 흐름

```
데이터랩 (수집)  →  빈도 분석 (전략)  →  AI 훈련소 (학습)  →  모의고사 (실전)
  230건+ 시험 후기     2단계 전략 도출     빈출 순 스크립트      빈출 순 출제
                                          생성                시뮬레이션
```

> **데이터가 전략을 만들고, 전략이 학습을 안내하고, 학습이 실전을 시뮬레이션한다.**
> 이 일관된 데이터 파이프라인이 오픽톡닥의 핵심 경쟁력이다.

## 13. 데이터 플라이휠: 사용자가 쌓을수록 정밀해진다

오픽톡닥의 전략은 **데이터가 쌓일수록 강해지는 구조**다.

### 선순환 구조

```
추천 서베이로 시험 응시
        ↓
데이터랩에 시험 후기 제출
        ↓
같은 서베이 기준의 데이터 축적
        ↓
통계 분석 정밀도 향상
  "음악 48.7%" → 오차 ±1%로 수렴
  "TV 2.2%" → 확실히 안 나오는 주제로 확정
        ↓
학습 범위 더 정밀하게 축소
  10개 → 8개 → 핵심 N개로 수렴
        ↓
더 효율적인 학습 → 더 좋은 성적
        ↓
더 많은 사용자 유입 → 더 많은 데이터
        ↓
(반복)
```

### 서베이 고정이 플라이휠의 전제 조건인 이유

| | 서베이가 다를 때 | 서베이를 고정할 때 |
|---|-----------------|------------------|
| 데이터 | 수십 가지 조합으로 분산 | **한 곳에 집중** |
| 조합별 표본 | 5~10건씩 파편화 | **전체가 같은 조합** |
| 통계 신뢰도 | 낮음 | **높음** |
| 분석 가치 | 제한적 | **정밀한 예측 가능** |

모든 사용자가 **같은 서베이**로 시험을 보고 후기를 제출하면:
- 데이터가 분산되지 않고 **한 곳에 축적**
- 230건이 1,000건, 10,000건으로 쌓이면 **오차가 극적으로 줄어듦**
- "이 주제는 확실히 나온다 / 확실히 안 나온다"가 **데이터로 증명**
- 학습 범위가 점점 더 **정밀하게 좁혀짐**

### 데이터 규모별 예상 효과

| 데이터 규모 | 분석 정밀도 | 학습 효과 |
|------------|-----------|----------|
| **230건** (현재) | 3:2 비율 확인, 빈출 주제 파악 | 핵심 10개 주제 집중 |
| **500건** | ±3% 수준의 정밀 빈도 | 주제별 우선순위 확정 |
| **2,000건** | 난이도별·시기별 패턴 분석 | 시기에 따른 맞춤 전략 |
| **10,000건+** | 출제 패턴 거의 완벽 예측 | **준비한 것이 그대로 시험에 나오는 경험** |

## 14. 오픽톡닥이 존재하는 이유

### 하나의 발견에서 시작했다

> **OPIc의 서베이는 시험 문제의 60%를 결정한다.**

이 한 가지 사실로부터 모든 것이 연쇄적으로 도출된다:

```
서베이가 시험의 60%를 결정한다
        ↓
서베이를 고정하면 학습 범위가 한정된다
        ↓
학습 범위를 한정하려면 최적의 서베이 설계가 필요하다
        ↓
최적 설계를 증명하려면 실제 시험 데이터가 필요하다
        ↓
데이터를 수집하려면 사용자 시험 후기 제출 시스템이 필요하다
        ↓
같은 서베이로 데이터를 모아야 통계적으로 의미있다
        ↓
데이터가 쌓이면 학습 범위를 더 정밀하게 좁힐 수 있다
        ↓
좁혀진 범위에 맞는 맞춤형 학습 콘텐츠가 필요하다
        ↓
AI로 주제별 맞춤 스크립트를 생성한다 (남의 스크립트가 아닌 "내 이야기")
        ↓
그 스크립트로 실전과 동일한 모의고사를 치르고
        ↓
쉐도잉으로 영어다운 톤과 억양을 체화한다
        ↓
AI 튜터가 체크박스 약점을 진단하고 답변 유형별 맞춤 훈련한다
```

### 오픽톡닥의 모든 모듈은 이 논리에서 나왔다

| OPIc의 3대 전략 축 | 오픽톡닥 모듈 | 해결 방식 |
|-------------------|-----------|----------|
| **축 1**: 서베이 주제 선택 | **추천 서베이 + 데이터랩** | 실제 시험 데이터로 출제 빈도 분석 → 최적 서베이 설계 |
| **축 1**: 빈도 기반 학습 | **AI 훈련소** | 빈도 순 주제/질문 정렬 → 빈출 주제부터 자연스럽게 스크립트 생성 |
| **축 1**: 실전 재현 | **모의고사 (빈출 모드)** | 데이터랩 빈도 기반으로 실제 출제 경향과 동일한 콤보 자동 구성 |
| **축 2**: 난이도별 유형 변화 | **모의고사** | 실제 OPIc과 동일한 15문제 콤보 구조 + 난이도별 유형 반영 |
| **축 3**: "내 이야기" 필요 | **AI 훈련소** | 내 경험 입력 → GPT-4o가 OPIc 최적화 스크립트 생성 |
| **축 3**: 등급별 발화량 차이 | **쉐도잉** | 목표 등급에 맞는 길이/난이도의 스크립트로 훈련 |
| **축 3**: 복문/표현력 요구 | **AI 튜터** | 개별 평가 체크박스 약점 진단 → 답변 유형별 우선순위 맞춤 훈련 |
| 피드백 부재 | **AI 평가** | 모의고사 15문제 AI 자동 채점 + 등급 예측 |
| 전 범위 무작정 공부 | **전체 시스템** | 데이터 → 전략 → 학습 → 실전 파이프라인을 서비스에 내장 |

### 아는 사람 vs 모르는 사람

```
모르는 사람:
  "오픽? 영어 말하기 시험이잖아. IM2 받아야 해."
  → 서베이 대충 선택 → 시험 범위 불필요하게 확대
  → 난이도 감으로 선택 → 불리한 유형 배정
  → 남의 스크립트 암기 → 콤보 구조 모름 → 전략 없음 → 낮은 등급

아는 사람 (= 오픽톡닥 사용자):
  "서베이 고정하고, 난이도 5-5 선택하고,
   ~610개 질문을 118개로 줄이고, 빈출 주제부터 집중.
   AI 훈련소에서 빈도 순으로 스크립트 만들고,
   빈출 모드 모의고사로 실전과 같은 시험 연습하자."
  → 데이터 기반 전략 → 효율적 학습 → 시험의 78% 대비 → 상위등급 확률 증가
```

### 한 문장으로

> **"OPIc의 서베이는 시험의 60%를 결정한다. 서베이를 고정하면 ~610개 질문이 118개로 줄고, 빈도 분석으로 핵심 주제부터 집중하면 시험의 78%를 대비할 수 있다. 오픽톡닥은 이 발견을 데이터로 증명하고, AI로 '내 이야기' 기반 학습 콘텐츠를 생성하고, 사용자가 더 많은 데이터를 쌓을수록 더 정밀해지는 플라이휠을 만든다."**
